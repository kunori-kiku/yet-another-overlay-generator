# plan-9 — Default release URLs (agent + mimic) + fix "Assist from release"

## Prerequisites

- **plan-8** (panel version display + "Update all agents → panel version" + refuse-newer guard) — the
  agent-assist tag-pinning convenience THIS plan relies on is driven by the panel version that plan-8
  threads in. plan-9 is **largely independent** of plan-8's code (it touches `settings.go`,
  `release_pins.go`, FE placeholders), but the *narrative* "agent assist pins to a real tag" is only
  fully realized when plan-8's one-click supplies `version=controllerVersion`. plan-9 adds nothing that
  *blocks* on plan-8 merging first; if executed before plan-8, the agent-assist-with-version path
  already works for a manually-typed version (the `resolveReleaseBase` logic at
  `internal/api/release_pins.go:242-252` predates this subject).
- No other plan is a hard prerequisite. Track A / Track B changes do not touch the files here.

## Goal

Make the shipped controller defaults *helpful* and make "Assist from release" actually succeed out of
the box. Add a `DefaultMimicReleaseBase` constant (pointing at the **mimic upstream** —
`hack3ric/mimic` — `releases/latest/download` alias, not a YAOG release) and wire it through
`DefaultSettings()` + `WithDefaults()` exactly as `DefaultAgentReleaseBaseURL` is wired, so a never-edited
controller returns a non-empty `mimic_release_base` on `GET /settings` and the mimic catalog form
pre-fills it — killing the `assistNeedsBase` hard error. Update the two FE placeholders
(`BootstrapSettings.tsx`, `MimicCatalogSettings.tsx`) to the real defaults so they read as truth, not
illustration. The agent assist already pins to a tag whenever a version is supplied against the
`releases/latest/download` alias (`resolveReleaseBase`); this plan documents and tests that contract so
plan-8's one-click cannot regress the moving-`latest` stall warned at `release_pins.go:206-214`. Custody
is **unchanged**: assist still only fills pins the operator reviews and saves through the validated
`/settings` path.

## Implementation steps

### 1. Add `DefaultMimicReleaseBase` and wire it through Default/WithDefaults

`internal/controller/settings.go` — add the const next to `DefaultAgentReleaseBaseURL` (after **line
11**), mirroring its doc style. Point it at the **mimic upstream** `latest/download` alias (the
canonical `hack3ric/mimic` base every test uses is `https://github.com/hack3ric/mimic/releases/...` —
see `settings_test.go:71`, `handler_bootstrap_test.go:149`, `internal/render/signing_test.go:337`). Use
the **`releases/latest/download`** alias (not a pinned `download/v0.1.0`) so the version-pinning path in
`resolveReleaseBase` (which only rewrites a base ending in `releaseLatestSuffix`,
`release_pins.go:244`) can tag-pin a mimic assist exactly as it does the agent base:

```go
// DefaultMimicReleaseBase is the upstream mimic project's "latest release" asset base. mimic is a
// THIRD-PARTY project (github.com/hack3ric/mimic), so this points at its releases, NOT YAOG's. It is
// the default the mimic GitHub-.deb catalog assist fetches .deb .sha256 sidecars from when the
// operator has not set their own MimicReleaseBase; like DefaultAgentReleaseBaseURL it uses the
// "releases/latest/download" alias so a version-bearing assist can pin it to a tag (resolveReleaseBase).
// NON-SECRET (a public URL). Upstream does not currently publish per-distro .deb assets, so the assist
// may legitimately miss rows (handled best-effort, per-row) — the value is still a working, helpful
// pre-fill that removes the assistNeedsBase hard error.
const DefaultMimicReleaseBase = "https://github.com/hack3ric/mimic/releases/latest/download"
```

`DefaultSettings()` (**lines 20-27**) — add the field so a fresh controller carries it:

```go
func DefaultSettings() ControllerSettings {
	return ControllerSettings{
		PublicAgentURL:      "",
		GithubProxy:         "",
		AgentReleaseBaseURL: DefaultAgentReleaseBaseURL,
		MimicReleaseBase:    DefaultMimicReleaseBase,
		Translucency:        boolPtr(true),
	}
}
```

`WithDefaults()` (**lines 32-40**) — fill an empty `MimicReleaseBase` from the default, mirroring the
`AgentReleaseBaseURL` branch (so a legacy saved record predating this default also gets a usable assist
base). Insert the branch *after* the `AgentReleaseBaseURL` fill (line 35), before the `Translucency`
fill:

```go
	if s.MimicReleaseBase == "" {
		s.MimicReleaseBase = DefaultMimicReleaseBase
	}
```

Update the `DefaultSettings`/`WithDefaults` doc comments (settings.go:16-19, :29-31) to mention the
mimic base alongside the agent base (keep it one short clause; don't bloat the comment).

**Back-compat note:** `MimicReleaseBase` is `json:"mimic_release_base,omitempty"` (store.go:335). With
`WithDefaults` now filling it, an old `settings.json` that omitted the field will load and then return
the default on `GET` — *additive, no breaking change*. The only behavioral shift: a controller that
previously returned an empty mimic base (distro-only mimic, no GitHub fallback, D4 in store.go:333) now
returns the upstream `latest` base by default. This is **safe** because the base alone does not enable a
GitHub fallback — `validateMimicCatalog` (handler_bootstrap.go:274) and the renderer only emit a fallback
catalog when `MimicDebs` is **non-empty**. A default base with no debs is inert (a pre-filled assist
source), exactly mirroring how a default `AgentReleaseBaseURL` with no `AgentBins` emits no agent block.
Call this out explicitly in the commit message and verify it with the new test (step 5).

### 2. Confirm `HandleReleasePins` already resolves mimic from the (now non-empty) default base

No code change in `release_pins.go`. The mimic branch at **release_pins.go:300** already reads
`defaultBase = cs.MimicReleaseBase`; with step 1, `cs` (loaded via `loadSettings` → `WithDefaults`) now
carries `DefaultMimicReleaseBase` when unset, so a mimic assist with no explicit `base` resolves against
the default instead of failing the `base == ""` guard at **:319-322** (`CodeAgentReleaseRequestInvalid`,
field `base`). **Verify** `loadSettings` applies `WithDefaults` (it must, for the agent default to work
today). If it does NOT, that is a latent bug — fix it there so both defaults flow identically; do not
special-case the mimic branch in the handler.

`resolveReleaseBase` (release_pins.go:242-252) is **unchanged**: a mimic assist that supplies a
`version` against the now-default `latest/download` base will tag-pin to
`.../releases/download/<tag>` (versionApplied=true), exactly as the agent base does. A mimic assist
*without* a version stays on the moving `latest` alias (versionApplied=false) — acceptable for the mimic
catalog, where the operator sets the version explicitly and reviews pins before save (custody, D6).

## Out of scope for this plan

- **The version-defaulting one-click itself** ("Update all agents → panel version", `versionApplied`
  flow, refuse-newer guard) — that is **plan-8**. plan-9 only makes the *defaults* and the *assist
  resolution* correct so plan-8's one-click pins to a real tag.
- **No new release assets.** This plan does not add YAOG release assets or ask upstream mimic to publish
  `.deb` sidecars. If the upstream mimic asset grammar differs from the `debKeyPattern`/`debAssetPattern`
  expectation, the assist legitimately misses rows (best-effort per-row, MimicCatalogSettings.tsx:137-174)
  and the operator fills them manually — the default base remains a helpful working pre-fill.
- **No change to the signing/verify path.** `verify.go`, the controller-signed `artifacts.json`, and the
  keystone binding are untouched (custody invariant).
- **No change to the SSRF/egress guards.** `blockPrivateAddr`, `isPublicUnicastIP`, `embeddedIPv4`,
  `validateReleaseURL` (release_pins.go:48-266) stay byte-for-byte intact; the default base is just a new
  *value* fed through the same validated path.
- **Mimic UDP fallback** (per-link policy, install-script branch, conditions) — Track B (plans 4-6).
- **Releasing `v2.0.0-beta.9`** — plan-10.
