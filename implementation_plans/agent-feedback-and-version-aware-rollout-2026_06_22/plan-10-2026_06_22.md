7. **Create the ANNOTATED tag from `main`** with the project identity (no secrets, no AI attribution

   Always `-a` (annotated) — `release.yml` and `git describe` depend on it (`RELEASING.md:45`).

8. **Pre-create the release as a PRERELEASE (NOT Latest), then push so `softprops` UPDATES it**
   (the primary path from `RC1-GATE.md:158-162`, inverted for a beta — beta is `--prerelease`, NOT
   `--latest`, per `RELEASING.md:54-56`). This guarantees the not-Latest pointer regardless of the
   `softprops` step:
   ```bash
   # Author release notes (reuse the CHANGELOG beta.9 section body).
   gh release create v2.0.0-beta.9 \
     --title "v2.0.0-beta.9" \
     --notes-file <notes-file> \
     --prerelease                       # NOT --latest; beta stays off Latest (beta.8 keeps it)
   git push origin v2.0.0-beta.9        # triggers release.yml; softprops UPDATES the pre-created release
   ```
   - **Belt:** the pre-create with `--prerelease` (no `--latest`).
   - **Suspenders:** the `make_latest:` expression at `release.yml:432` evaluates **false** for a
     `-beta.` tag, so the `softprops` step also keeps it off Latest. Both must agree.
   - **WRONG (do not do):** a bare `gh release create … --latest` AFTER the push (collides with the
     auto-created release AND would steal Latest from beta.8) — `RC1-GATE.md:164-165`.
