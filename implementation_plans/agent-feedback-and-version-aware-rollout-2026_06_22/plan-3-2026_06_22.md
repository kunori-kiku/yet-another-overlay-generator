## Goal

Make self-update and WireGuard link health **first-class structured conditions** and retire the panel's
free-form `health`-string grepping. The agent emits a `selfupdate` condition whose `reason` is a closed
enum (`Active` | `HealthConfirmedProbationary` | `Updated` | `Abandoned`) carrying the curated meaning the
free-form `State.Health` markers carry today, and a best-effort `wireguard` condition summarizing link
health (`reason` ∈ `AllPeersUp` | `PeerHandshakeStale` | `LinkDown` | `NoInterfaces`) sampled cheaply
from `wg show`. `deriveUpdateState()` **prefers** the structured `selfupdate` condition when present and
falls back to the legacy `lastHealth` substring parsing **only** for old agents that send no conditions —
keeping every shipped agent's chip working while killing the brittle string match for new ones. The legacy
`State.Health` markers stay set (back-compat for old controllers/panels); the structured condition is the
new source of truth. WireGuard sampling is **best-effort and never fails a cycle** (a probe error yields no
condition, never an apply failure).

## Implementation steps

> Convention: every `reason` is CamelCase from a **closed enum**; every `message` is a single
> human line capped at `model.ConditionMessageMax` (160) inside `classify()` — NEVER `err.Error()` of a
> multi-line subprocess (outline Principle D5 / "clear catch"). `Type` values come from plan-1's exported
> `model.ConditionType*` consts (`ConditionTypeSelfUpdate` = `"selfupdate"`, `ConditionTypeWireGuard` =
> `"wireguard"`, `ConditionTypeConfigApply` = `"configapply"`); reference those consts — never redeclare
> a local `condType*` string literal.

### A. Self-update condition (retire the string meaning, keep the string)

1. **Add the `selfupdate` reason enum + classify, agent-side.** New file
   `internal/agent/conditions_selfupdate.go` (keeps `selfupdate.go` focused on the risky swap core):

   ```go
   package agent

   // Condition reasons for the agent self-update lifecycle. These are the STRUCTURED
   // replacement for the free-form State.Health markers (selfupdate.go:321/:348/:384, agent.go:321/:355)
   // — the legacy strings are STILL set (old controllers/panels parse them) but the panel prefers this.
   // The condition TYPE is plan-1's exported const model.ConditionTypeSelfUpdate; do NOT redeclare it.

   const (
       reasonSelfUpdateActive        = "Active"                     // a swap is in flight (PendingUpdate present, not Confirmed)
       reasonSelfUpdateProbationary  = "HealthConfirmedProbationary" // passed the health gate, awaiting one clean cycle
       reasonSelfUpdateUpdated       = "Updated"                    // finalized (floor advanced)
       reasonSelfUpdateAbandoned     = "Abandoned"                  // rolled back at the cap / health gate
   )

   // selfUpdateCondition derives the structured selfupdate condition from the agent's persisted State.
   // The bool result is false when there is nothing to report (no in-flight breadcrumb and no terminal
   // self-update marker) — "no condition" is expressed as (model.Condition{}, false), NEVER a nil pointer.
   // It is PURE (no I/O): the daemon calls it after the reconcile/finalize chain so the breadcrumb + the
   // Health string reflect the latest outcome. The message is a curated one-liner, never a raw error.
   func selfUpdateCondition(st *State, now time.Time) (model.Condition, bool) {
       if st == nil {
           return model.Condition{}, false
       }
       switch {
       case st.PendingUpdate != nil && st.PendingUpdate.Confirmed:
           return classify(model.ConditionTypeSelfUpdate, model.ConditionStatusWarn, reasonSelfUpdateProbationary,
               "self-update to "+st.PendingUpdate.To+" health-confirmed; probationary until one clean cycle", now), true
       case st.PendingUpdate != nil:
           return classify(model.ConditionTypeSelfUpdate, model.ConditionStatusWarn, reasonSelfUpdateActive,
               "self-update to "+st.PendingUpdate.To+" in flight (attempt "+strconv.Itoa(st.PendingUpdate.Attempts)+")", now), true
       case strings.HasPrefix(st.Health, "self-updated to "):
           return classify(model.ConditionTypeSelfUpdate, model.ConditionStatusOK, reasonSelfUpdateUpdated,
               "self-updated to "+strings.TrimPrefix(st.Health, "self-updated to "), now), true
       case strings.Contains(st.Health, " abandoned:"):
           return classify(model.ConditionTypeSelfUpdate, model.ConditionStatusWarn, reasonSelfUpdateAbandoned,
               "self-update abandoned (rolled back); change the target to retry", now), true
       }
       return model.Condition{}, false
   }
   ```

   - `classify` is plan-1's agent-side helper — `classify(condType model.ConditionType, status
     model.ConditionStatus, reason, detail string, since time.Time) model.Condition` (VALUE return,
     caps the message at `model.ConditionMessageMax` internally). `model.ConditionStatus*` are plan-1's
     four status constants — `OK`/`Warn`/`Error`/`Unknown` only (no `Healthy`/`Progressing`). In-flight
     and probationary self-update map to `ConditionStatusWarn`; the settled `Updated` outcome maps to
     `ConditionStatusOK`; `Abandoned` maps to `ConditionStatusWarn`.
   - **No-condition contract:** "nothing to report" is the `(model.Condition{}, false)` zero-value
     sentinel pair, never a `*model.Condition` nil pointer — callers branch on the bool, not on nil.
   - **Custody (Principle: signed self-update):** this is read-only over `State` — it touches NEITHER the
     floor (`selfupdate.go:346`), the breadcrumb, `verify.go`, NOR the swap path. It only *describes* the
     state the reconcile/finalize chain already wrote. Do not move floor/breadcrumb logic here.
   - Note the deliberate **non-1:1** mapping: the breadcrumb (durable) is authoritative for `Active` vs
     `Probationary`; the `Health` string is only consulted for the two *terminal* outcomes (`Updated`,
     `Abandoned`) that clear the breadcrumb. This keeps the structured reason correct even mid-flight when
     the legacy string still says `"applied"`.

2. **Funnel the selfupdate condition into the collector.** In `internal/agent/agent.go` at the
   `persistAndReport` call sites (or plan-1's collector entry point), after the
   reconcile/finalize/record chain has run for the cycle, call `selfUpdateCondition(s, time.Now().UTC())`
   and append the returned `model.Condition` to plan-1's collector when the bool result is `true`. The
   natural seam is inside `recordSuccess`/`recordFailure` (`agent.go:308-376`) — they already load `prev`
   and build the new `State` `s`; append the condition computed over `s` to the collector before
   `persistAndReport(cfg, s)` (`:342`, `:375`). **Re-use plan-1's collection mechanism; do not invent a
   second report path.**
   - Back-compat: when there is no self-update activity, `selfUpdateCondition` returns `(model.Condition{},
     false)` → no condition emitted → the report's `conditions` array simply omits `selfupdate` (it stays
     `omitempty`).

### B. WireGuard link-health condition (best-effort sampler)

3. **Add the cheap link sampler + classify.** New file `internal/agent/conditions_wireguard.go`:

   ```go
   package agent

   // The condition TYPE is plan-1's exported const model.ConditionTypeWireGuard; do NOT redeclare it.

   const (
       reasonWGAllPeersUp        = "AllPeersUp"
       reasonWGPeerHandshakeStale = "PeerHandshakeStale"
       reasonWGLinkDown          = "LinkDown"
       reasonWGNoInterfaces      = "NoInterfaces"
   )

   // wgHandshakeStaleAfter: a peer whose latest handshake is older than this is "stale" (the link may be
   // flapping or the peer gone). 3× a typical Babel/WG keepalive horizon — generous so a healthy idle
   // link is never falsely flagged.
   const wgHandshakeStaleAfter = 5 * time.Minute

   // wgShowFn runs `wg show all dump` and returns its stdout, indirected so a test can inject a fixture
   // without a kernel WireGuard interface. Production shells out to `wg`. A non-nil error (wg absent,
   // not root, no interfaces) is BEST-EFFORT: the caller emits NO condition and NEVER fails the cycle.
   var wgShowFn = func() ([]byte, error) {
       return exec.Command("wg", "show", "all", "dump").Output()
   }

   // sampleWireGuardCondition probes link health cheaply via `wg show all dump` and classifies it into a
   // single wireguard condition. The bool is false on any probe error (best-effort — a status probe must
   // NEVER fail a cycle, Principle: deployable configs / keep-last-good); "no condition" is the
   // (model.Condition{}, false) sentinel, never a nil pointer. The message is a curated count, never a
   // raw dump.
   func sampleWireGuardCondition(now time.Time) (model.Condition, bool) {
       out, err := wgShowFn()
       if err != nil {
           return model.Condition{}, false // best-effort: no probe, no condition, no cycle failure
       }
       reason, status, msg := classifyWGDump(out, now)
       if reason == "" {
           return model.Condition{}, false
       }
       return classify(model.ConditionTypeWireGuard, status, reason, msg, now), true
   }

   // classifyWGDump is the PURE classifier over `wg show all dump` output (TSV: iface, then per-peer
   // rows whose 5th field is latest-handshake epoch seconds, "0" = never). It is table-tested. The
   // status it returns is one of plan-1's model.ConditionStatus values (OK/Warn).
   func classifyWGDump(dump []byte, now time.Time) (reason string, status model.ConditionStatus, msg string) { /* parse + classify */ }
   ```

   - **Parse contract (cite in the file):** `wg show all dump` emits a TSV where the FIRST row per
     interface is the interface line (private/public key, listen-port, fwmark) and subsequent rows are
     peers; a peer row's **field index 4** (0-based) is the latest-handshake UNIX epoch (`0` = never
     handshaked). Classify: zero interfaces → `NoInterfaces` (`model.ConditionStatusWarn`); ≥1 iface but a
     peer never handshaked → `LinkDown` (`model.ConditionStatusWarn`); a peer handshaked but
     `now - hs > wgHandshakeStaleAfter` → `PeerHandshakeStale` (`model.ConditionStatusWarn`); all peers
     fresh → `AllPeersUp` (`model.ConditionStatusOK`). Message = `"N/M peers up"` style count, NEVER the
     raw dump.
   - **Root-script-safety (Principle):** the dump is parsed in Go, never re-shelled; no captured field
     reaches a shell context. `wg show all dump` is read-only.
   - **Client role (`wg0`, no Babel):** `wg show all dump` still works on a single-`wg0` node, so the
     condition is meaningful there too; no role gating needed.

4. **Sample once per cycle (best-effort).** In `internal/agent/cycle.go`, after the APPLY branch's
   `Run` returns successfully (`cycle.go:209-219`, before the final `return appliedGen, true, nil`), call

   IDLE branch (`:172-185`) and rekey branch (`:140-154`) if plan-1's collector is per-cycle and reports
   even on no-apply cycles — pick the seam that matches plan-1's collector lifecycle (cite it).
   - The sample MUST be wrapped so a panic/slow `wg` cannot wedge the cycle: `wgShowFn` already bounds via
     a normal `exec` (no long-poll). Do not add a goroutine; the dump is sub-millisecond. If plan-1's
     collector is funneled exclusively through `recordSuccess`/`recordFailure`, fold the WG sample into the

### C. Retire the panel string-matching (prefer the structured condition)

5. **`deriveUpdateState` reads the structured condition first.** Edit
   `frontend/src/lib/updateStatus.ts:21-43`. Add a structured-condition fast path BEFORE the legacy
   `lastHealth` substring block; keep the legacy block verbatim as the fallback for no-conditions nodes:

   ```ts
   export function deriveUpdateState(
     node: ControllerNode,
     settings: ControllerSettings | null,
     now: number = Date.now(),
   ): UpdateState {
     const target = settings?.targetAgentVersion.trim() ?? '';
     if (target === '') return 'off';
     if (!node.inRollout) return 'not-targeted';

     // PREFERRED: the structured selfupdate condition (plan-2 maps Node.conditions). When a NEW agent
     // sends it, the brittle lastHealth substring parse below is bypassed entirely.
     const su = (node.conditions ?? []).find((c) => c.type === 'selfupdate');
     if (su) {
       switch (su.reason) {
         case 'Abandoned': return 'failed';
         case 'Active':
         case 'HealthConfirmedProbationary': return 'applying';
         case 'Updated': return 'applied';
       }
       // an unknown future reason falls through to the version compare below (forward-compat)
     } else {
       // LEGACY fallback (old agents send no conditions): the historical lastHealth substring parse.
       const health = node.lastHealth || '';
       if (health.includes(' abandoned:')) return 'failed';
       if (health.includes('health-confirmed (probationary)')) return 'applying';
       if (health.startsWith('self-updated to ')) return 'applied';
     }

     // No self-update signal (structured or legacy): the reported running version vs target is authoritative.
     if (node.agentVersion.trim() && compareSemver(node.agentVersion, target) >= 0) return 'applied';
     if (isStale(node.lastSeen, now)) return 'stale';
     return 'pending';
   }
   ```
