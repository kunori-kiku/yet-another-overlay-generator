# plan-6 — Panel: per-link mimic fallback toggle + global default setting + mimic condition chip  (subject agent-feedback-and-version-aware-rollout, Track B)

> Part of [outline.md](./outline.md) — subject: agent-feedback-and-version-aware-rollout.

## Prerequisites

This plan is the **panel face** of the mimic-fallback feature and the **first known-type chip**
hung off plan-2's generic conditions strip. It is a pure-frontend plan; everything it edits is
under `frontend/src/`. It depends on three earlier plans for the data shapes it reads/writes:

- **plan-2 (operator API exposure + generic `<NodeConditions>` render) MUST land first.** This
  plan adds a *known-type renderer* for the `mimic` condition type that plugs into plan-2's
  generic strip. As of this writing the strip does **not** exist (`grep -rn NodeConditions
  frontend/src/` finds only an unrelated comment in `stores/controllerStore.ts:1025`), so this
  plan cannot mount its chip until plan-2 has created `<NodeConditions>` and `ControllerNode.conditions`
  (`frontend/src/types/controller.ts`). If plan-2's strip API differs from what §6 assumes,
  rebase the chip onto the real prop shape (cite plan-2's actual export).
- **plan-4 (model field + global default setting, backend) MUST land first.** This plan writes
  `edge.mimic_fallback` and reads/writes `ControllerSettings.MimicFallbackDefault`; both must
  exist in the Go model + settings wire and in the TS mirrors (`frontend/src/types/topology.ts`
  `Edge`, `frontend/src/api/controllerClient.ts` `ControllerSettings`/`SettingsJSON`) before the
  UI can bind to them. If plan-4 shipped the TS mirror already, this plan only consumes it; if
  plan-4 left the TS mirror to this plan, §3/§4 below add the field (flagged inline).
- **plan-5 (mimic condition emitted by the agent) SHOULD land first** so the chip has real data
  to render, but is not a *build* prerequisite — the chip's reason→label mapping is a pure
  closed-enum table that compiles and unit-tests without any live agent. If plan-5's `reason`
  enum differs from §6's table, the table is the single place to reconcile.

Inherits the Track-B principles from [outline.md](./outline.md) §2 — most relevant here:
**[INFERRED] mimic exists to evade UDP-hostile networks** (a fallback de-cloaks → the chip MUST
read as a loud `warn`, never a quiet "ok"); **[STATED] backward compatibility** (old topologies
have no `mimic_fallback`; old nodes have no `mimic` condition — both render cleanly).

## Goal

Give the operator the controls and the read-back for mimic UDP fallback entirely in the panel:
the **edge inspector** (`EdgeEditor.tsx`) gains a tri-state per-link select (inherit / udp / none)
bound to `edge.mimic_fallback`, shown only when the link is `transport: tcp` (the only transport
mimic touches); the **settings surface** gains the fleet-wide `MimicFallbackDefault` toggle riding
the existing full-replace `/settings` contract (added to `ControllerSettings` + `toSettingsJSON`);
and a **rich, categorized `mimic` condition chip** renders the curated reason (e.g. "Mimic: fell
back to UDP (kernel lacks eBPF)") as a known-type renderer plugged into plan-2's generic
`<NodeConditions>` strip on the node row + detail page. The Subject-2 small-screen read-only canvas
gate is respected **structurally** (the edge inspector is never mounted below `lg` — see §1), so no
new guard is needed. All new strings are EN + ZH. No backend changes; no new condition types.

### Step 4 — Mimic condition chip (known-type renderer for plan-2's strip)

Add a new component `frontend/src/components/deploy/MimicConditionChip.tsx` mirroring
`UpdateStatusChip.tsx`'s closed-enum shape. It takes one `NodeCondition` (plan-2's exported type
from `frontend/src/types/controller.ts` — the canonical interface has **6 fields**: `{ type,
status, reason, message, since, observedAt }`) and renders a colored chip with a curated label +
the curated `message` as the
`title` tooltip. Color is driven by `status` (plan-2's strip already maps status→color generically;
the chip's own class keys off `status` so a `warn` fallback reads loud):

```tsx
import { t, type MessageKey, type UILanguage } from '../../i18n';
import type { NodeCondition } from '../../types/controller';

// Known-type renderer for type==='mimic', plugged into plan-2's generic <NodeConditions> strip.
// reason is a CLOSED enum emitted by plan-5's classify(); message is the curated single line.
// Unknown future reasons fall through to the generic label (never blank) — forward-compatible.
const REASON_LABEL: Record<string, MessageKey> = {
  Active: 'mimicCond.active',
  FellBackToUDP: 'mimicCond.fellBack',
  KernelTooOld: 'mimicCond.kernelTooOld',
  EbpfLoadFailed: 'mimicCond.ebpfFailed',
  InstallFailed: 'mimicCond.installFailed',
};

function statusClass(status: string): string {
  switch (status) {
    case 'ok': return 'bg-green-900/40 text-green-300 border-green-700';
    case 'warn': return 'bg-amber-900/40 text-amber-300 border-amber-700';
    case 'error': return 'bg-red-900/40 text-red-300 border-red-700';
    // 'unknown' is the neutral status value (matching plan-1's ConditionStatusUnknown);
    // the Tailwind `default` branch is the visual fallback bucket it lands in.
    case 'unknown': default: return 'bg-gray-800 text-gray-400 border-gray-600';
  }
}

export function MimicConditionChip({ c, language }: { c: NodeCondition; language: UILanguage }) {
  const labelKey = REASON_LABEL[c.reason];
  const label = labelKey ? t(language, labelKey) : (c.reason || t(language, 'mimicCond.label'));
  // message is the curated, length-capped human line from classify() — NOT raw stderr (D5).
  return (
    <span title={c.message || undefined} className={`px-2 py-0.5 rounded text-xs border ${statusClass(c.status)}`}>
      {t(language, 'mimicCond.label')}: {label}
    </span>
  );
}
```

Wire it into plan-2's `<NodeConditions>` strip: where the strip iterates conditions, branch on
`type === 'mimic'` to render `<MimicConditionChip c={c} language={language} />`, else fall through
to the generic chip plan-2 already renders. (Exact insertion is in plan-2's component — cite its
real `map` callback when editing.) This makes the chip appear on **both** the node row and the
detail page automatically, since plan-2 mounts the same strip in `NodeRegistry.tsx` and
`FleetNodeDetailPage.tsx`. **Do not** add a separate `nodeCells` column — the strip owns the
column set (NodeRegistry's descriptor-spine single-source invariant, `:35-39`).

> Reason-enum sync: `REASON_LABEL`'s keys MUST equal plan-5's classify() reason codes verbatim.
> If plan-5 names them differently (e.g. `UDPFallback`), reconcile here — this table is the one
> place the FE pins the contract. Status strings (`ok`/`warn`/`error`) MUST match plan-1's
> `Condition.Status` enum (re-read plan-1's model before finalizing `statusClass`).
