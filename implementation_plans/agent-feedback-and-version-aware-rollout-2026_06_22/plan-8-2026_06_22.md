## Goal

Give the operator a one-click "Update all agents → the panel's own controller version" capability with
a hard safety floor. The panel displays the controller `BuildVersion` (from plan-7). An "Update all
agents" action with **no version typed** sets `targetAgentVersion = controllerVersion` (FE
`SessionInfo.controllerVersion` / store `controllerVersion`, hydrated from the session wire
`controller_version`), runs the
existing assist with `version=controllerVersion` (so `versionApplied=true` → the release base is pinned
to `releases/download/<tag>` and the operator still REVIEWS the resolved pins), and arms fleet-wide —
then the operator SAVES through the validated `/settings` path (custody unchanged: no auto-trust, no
hash persisted without operator save). The controller `validateAgentRollout` **refuses** any
`TargetAgentVersion` strictly newer than the controller's own version, using the SAME comparison
semantics as the agent's `compareVersions` (empty = minimal sentinel), returning a clear coded
`*apierr.Error`. The guard runs on BOTH the normal `/settings` POST save and (transitively) the
one-click flow, so a misconfiguration can never push an agent target the controller has never built.

### Phase 1 — Backend: refuse-newer guard in `validateAgentRollout`

**Step 1.1 — Add a coded error.** In `internal/apierr/apierr.go` near the rollout/release codes
(`:106-108`), add a dedicated code so the FE + operator get a precise message (NOT a generic field

```go
// than the controller's own BuildVersion. The controller can only certify a self-update target it
// has itself shipped (the release the agent fetches is published by the same release pipeline that
// stamps BuildVersion), so a newer target can only ever fail-closed at verify time — reject it at
// save with a clear message rather than silently arming a rollout that can never converge.
CodeAgentTargetNewerThanController Code = "agent_target_newer_than_controller" // 400
```
```go
CodeAgentTargetNewerThanController: {
    "The agent target version {target} is newer than the controller version {controller}; " +
```

**Step 1.2 — Reuse `compareVersions` semantics.** `compareVersions` is UNEXPORTED in package `agent`
(`version.go:25`); package `internal/api` cannot import it. Two options:

- **(CHOSEN) Promote to a shared internal package.** Move `compareVersions` + its helpers
  (`splitVersion`, `compareReleaseCore`, `comparePrerelease`, `comparePreField`, `atoiOrZero`) into a
  new tiny package `internal/version/version.go` as `func Compare(a, b string) int`, and have
  `internal/agent/version.go` re-export it (`func compareVersions(a, b string) int { return version.Compare(a, b) }`),
  keeping the agent's call sites + tests byte-unchanged. This SINGLE-SOURCES the comparator so the
  guard and the agent's floor/anti-downgrade logic can NEVER diverge (the same single-source rationale
  as `UnsafeOperatorCredentialBindingField`/`validateOperatorCredentialBinding` at
  `handler_bootstrap.go:325-340`). No shim, no copy.
- *(Rejected) Mirror/copy the function into `internal/api`.* Violates "no duplicate logic" — two
  comparators that can silently drift is exactly the failure D5 warns against.

```go
// precedence; "" is below every real version, equal only to itself.)

```

**Step 1.3 — Thread the controller version to the guard.** `validateAgentRollout` is a package-level
function called from the `/settings` POST handler at `handler_bootstrap.go:152`, where `h` (the
`*ControllerHandler` with plan-7's `version` field) is in scope. Change the signature to accept
the controller version explicitly (keeps the function pure + table-testable — preferred over making it
a method) at `handler_bootstrap.go:350`:

```go
// validateAgentRollout enforces D8 on the signed agent self-update pins AND the plan-8 refuse-newer
// floor: a TargetAgentVersion strictly NEWER than controllerVersion is rejected, because the
// controller can only certify a target its own release pipeline shipped. controllerVersion is the
// in-process main.BuildVersion (plan-7); an EMPTY controllerVersion (an unstamped dev build) DISABLES
// only the floor (version.Compare treats "" as minimal, so every target would be "newer" — that would
// wrongly freeze a dev controller, so empty short-circuits the floor instead). The grammar checks are
// unchanged.
func validateAgentRollout(cs controller.ControllerSettings, controllerVersion string) *apierr.Error {
    // ... existing semver/bin grammar checks unchanged (handler_bootstrap.go:351-370) ...
    if cs.TargetAgentVersion != "" && controllerVersion != "" &&
        version.Compare(cs.TargetAgentVersion, controllerVersion) > 0 {
        return apierr.New(apierr.CodeAgentTargetNewerThanController).
            With("target", cs.TargetAgentVersion).
            With("controller", controllerVersion)
    }
    return nil
}
```
Update the lone caller at `handler_bootstrap.go:152`:
```go
if err := validateAgentRollout(cs, h.version); err != nil {
    writeAPIError(w, err)
    return
}
```

> *Backward-compat / empty-version note:* an unstamped local `go run` build has `BuildVersion = "dev"`
> (or `""` depending on plan-7's default). `version.Compare("dev", target)` with a non-semver "dev"
> degrades its release core to `0.0.0` via `atoiOrZero`, which would reject every real target on a
> dev build. **Guard on `controllerVersion != ""` AND skip the floor when `controllerVersion` is not
> a valid semver** — add a `semverPattern.MatchString(controllerVersion)` precondition so a dev/`""`
> build never spuriously rejects (the floor is a production safety net, not a dev blocker). Document
> this explicitly in the function comment.
