# plan-1 — Node Conditions feedback channel: wire + agent collector + controller store (end-to-end, behavior-preserving)

> Part of [outline.md](./outline.md) — subject: agent-feedback-and-version-aware-rollout.

## Prerequisites

None — this is the **foundation** of Track A (and of the whole subject). Every later plan that
emits or renders a condition builds on the primitive this plan introduces:

- **plan-2** consumes `Node.Conditions` (operator API + panel render).
- **plan-3** adds `selfupdate` + `wireguard` conditions via the `classify()` helper this plan defines.
- **plan-5** adds the `mimic` condition via the same helper.

So this plan MUST land first. It deliberately wires exactly **one** condition (`configapply`,
mirroring the existing `State.Health`) so the end-to-end path is proven with **zero behavior
change** before any new diagnostics are introduced.

## Goal

Introduce the reusable, structured agent→panel feedback primitive ("Node Conditions"). Define a
single shared `model.Condition{Type,Status,Reason,Message,Since}` type (Kubernetes-conditions
shape, per outline D5); add an **additive `omitempty`** `conditions` array to the agent→controller
report wire (`reportRequestWire` → `reportRequestJSON`); add an agent-side condition **collector**
plus a `classify()` helper signature the later plans reuse; and store `Node.Conditions` (with a
**server-stamped `ObservedAt`** so a node clock cannot spoof receipt time) in both `Store`
implementations. To prove the path end-to-end with no behavior change, wire exactly **one**
condition: a `configapply` condition that mirrors today's `State.Health` (`applied` /
`degraded: keeping last-good`). The change is backward-compatible in both directions: an old agent
omits `conditions` (controller stores `nil`), and an old controller ignores the unknown field.

## Implementation steps

### 1. Define the shared `Condition` type in `internal/model`

**Where:** `internal/model/topology.go` lives in package `model` and imports **only stdlib**
(verified: `topology.go:1` + no internal imports across `internal/model/*.go`). Both `controller`
(`store.go`, `compile.go`) and `agent` (test code today) can import `model`, and `model` imports
nothing internal — so a `Condition` type here is shared by agent + controller + api with **no
import cycle and no new third-party dep** (PRINCIPLES: minimal deps). This is preferable to a new
`internal/condition` package: it reuses the existing shared-model home and adds zero packages.
Add a new file `internal/model/condition.go` (keeps `topology.go` focused; a sibling of the
existing `artifact.go` shared-type pattern):

```go
package model

// Condition is one structured, curated feedback fact an agent reports about itself, in the
// Kubernetes-conditions shape. It REPLACES brittle free-form-health string-matching (the panel
// greps State.Health today). It is additive and backward-compatible: an old agent sends none, an
// old controller ignores the field. The full set per node is a small slice keyed by Type (one
// active condition per Type; a later report for the same Type supersedes the prior).
//
// Curation invariant (outline D5, HIGH for this subject): Reason is a CLOSED enum (CamelCase code)
// per Type, and Message is a SINGLE length-capped human line produced by a classify() mapping —
// NEVER the raw stderr / LastError dump. message is for an operator tooltip, not a log sink.
type Condition struct {
	// Type is the condition kind ("configapply", "selfupdate", "wireguard", "mimic"). Stable,
	// machine-readable; the panel may render a richer chip for known types and a generic strip for
	// the rest.
	Type string `json:"type"`
	// Status is the coarse state for color-coding: "ok" | "warn" | "error" | "unknown".
	Status string `json:"status"`
	// Reason is a closed, CamelCase enum code scoped to Type (e.g. "Applied", "DegradedKeepingLastGood").
	// Stable across releases so the panel/tests can switch on it.
	Reason string `json:"reason"`
	// Message is a single, length-capped human line (curated by classify(), never raw stderr).
	Message string `json:"message,omitempty"`
	// Since is the AGENT-side wall-clock time the condition last changed, RFC3339. Advisory only —
	// the controller server-stamps an authoritative ObservedAt on receipt (a node clock cannot be
	// trusted for ordering). Empty when the agent did not set it.
	Since string `json:"since,omitempty"`
}

// Condition Status constants — the closed status enum for color-coding.
const (
	ConditionStatusOK      = "ok"
	ConditionStatusWarn    = "warn"
	ConditionStatusError   = "error"
	ConditionStatusUnknown = "unknown"
)

// Condition Type constants — the closed type enum. plan-1 wires only ConditionTypeConfigApply;
// plan-3 adds SelfUpdate + WireGuard, plan-5 adds Mimic. Declared here so every consumer references
// one source of truth.
const (
	ConditionTypeConfigApply = "configapply"
	ConditionTypeSelfUpdate  = "selfupdate"
	ConditionTypeWireGuard   = "wireguard"
	ConditionTypeMimic       = "mimic"
)

// ConditionMessageMax is the hard cap on a curated Message (runes). classify() helpers MUST
// truncate to this. Caps the tooltip to one line and forbids a multi-line stderr dump leaking in.
const ConditionMessageMax = 160
```

> **Custody note (PRINCIPLES — signed-update custody, HIGH):** `Condition` is a plain reporting
> DTO. It is NOT embedded in any bundle, manifest, `artifacts.json`, or signature input. It rides
> ONLY the agent→controller `/report` payload (untouched-by-signing transport) and the controller
> store. `verify.go` and the signed-bundle path are not edited by this plan.

#### 2b. Add the collector + `classify()` helper

**Where:** new file `internal/agent/conditions.go`, package `agent`:

```go
package agent

import (
	"time"

	"github.com/kunorikiku/yet-another-overlay-generator/internal/model"
)

// classify maps a (type, status, reason, detail) tuple into a curated model.Condition with a
// SINGLE, length-capped Message — never a raw multi-line stderr dump (outline D5, HIGH). It is the
// one chokepoint every condition emitter (plan-1 configapply, plan-3 selfupdate/wireguard,
// plan-5 mimic) funnels through, so the curation + message-cap invariant is enforced in ONE place.
// detail is the human line the caller already curated (e.g. "kernel lacks eBPF"); classify never
// reads err.Error() itself — it caps and stamps what it is given.
func classify(condType, status, reason, detail string, since time.Time) model.Condition {
	msg := detail
	if r := []rune(msg); len(r) > model.ConditionMessageMax {
		msg = string(r[:model.ConditionMessageMax])
	}
	return model.Condition{
		Type:    condType,
		Status:  status,
		Reason:  reason,
		Message: msg,
		Since:   since.UTC().Format(time.RFC3339),
	}
}

// collectConditions builds the per-cycle condition set from the apply outcome. plan-1 wires exactly
// ONE condition — configapply — that MIRRORS the existing State.Health with no behavior change:
//   - success  -> status ok,    reason "Applied",                message "configuration applied"
//   - failure  -> status warn,  reason "DegradedKeepingLastGood", message "keeping last-good configuration"
// plan-3/plan-5 extend this to append selfupdate/wireguard/mimic conditions; the signature stays.
func collectConditions(ok bool, now time.Time) []model.Condition {
	if ok {
		return []model.Condition{
			classify(model.ConditionTypeConfigApply, model.ConditionStatusOK,
				"Applied", "configuration applied", now),
		}
	}
	return []model.Condition{
		classify(model.ConditionTypeConfigApply, model.ConditionStatusWarn,
			"DegradedKeepingLastGood", "keeping last-good configuration", now),
	}
}
```

> **Curation rationale (D5):** `collectConditions` passes a Go-emitted constant `detail`, never
> `prev.LastError` / `err.Error()`. The configapply condition deliberately carries no upstream text
> in plan-1 — it only mirrors Health. plan-5 (mimic) is the first plan that classifies a
> failure-category; it will pass a curated category string, still never the raw dump.

### 5. Controller store: persist `Node.Conditions` with server-stamped `ObservedAt`

#### 5a. A store-side condition record (adds `ObservedAt`)

The wire `model.Condition` has an advisory `Since`; the store needs a server-stamped `ObservedAt`.
Add a thin store type in `internal/controller/store.go` (alongside `Node`, after the `Node` struct
at :111):

```go
// NodeCondition is one stored agent condition with a SERVER-stamped ObservedAt. It embeds the
// reported model.Condition verbatim (type/status/reason/message/since) and adds the controller's
// authoritative receipt time, so the panel orders/ages conditions by a clock the node cannot spoof
// (the wire Since is advisory only). Persisted by both Store impls on the whole-Node write.
type NodeCondition struct {
	model.Condition
	// ObservedAt is the controller wall-clock time SetAppliedGeneration recorded this condition.
	ObservedAt time.Time `json:"observed_at"`
}
```

Add to the `Node` struct (after `LastAgentVersion` at :102):

```go

	// ObservedAt on receipt (plan-1). Nil until a conditions-aware agent reports; replaced wholesale
	// each report (the latest report is the truth). Observability only — not custody, not allocation.
	Conditions []NodeCondition
```

(`internal/controller/store.go` already imports `time` and `internal/model`.)

Add one shared helper in `store.go` (so both impls share the server-stamp logic — no duplication):

```go
// stampConditions wraps each reported model.Condition with the controller's authoritative
// ObservedAt. A nil/empty report clears the stored set (the latest report is the truth: an agent
// that no longer reports a condition has it removed). The wire Since is carried through unchanged
// (advisory); ObservedAt is the server clock.
func stampConditions(conditions []model.Condition, observedAt time.Time) []NodeCondition {
	if len(conditions) == 0 {
		return nil
	}
	out := make([]NodeCondition, len(conditions))
	for i, c := range conditions {
		out[i] = NodeCondition{Condition: c, ObservedAt: observedAt}
	}
	return out
}
```

> **Back-compat (MEDIUM):** an old agent sends no `conditions` → `req.Conditions == nil` →
> `stampConditions(nil, _) == nil` → `Node.Conditions` stays `nil`. An old persisted node record
> (no `conditions` key) unmarshals to `nil`. No existing field is perturbed.
