# plan-1 — Telemetry monitoring channel (dedicated `/telemetry` heartbeat)

> Part of [outline.md](./outline.md) — subject: beta9-smoke-hardening. Design verified sound by workflow `w2mslu5jt`.

## Prerequisites
None (first plan). Branch off `main`.

## Goal
Node Conditions are sampled only at apply time and ride `/report`, so the panel freezes a worst-case post-apply snapshot (`wireguard: LinkDown` sampled pre-handshake, `selfupdate: HealthConfirmedProbationary` sampled mid-probation) that never refreshes while the node idles — though the overlay + self-update are healthy. Build a **dedicated, agent-authed `POST /telemetry` heartbeat** that re-samples + re-reports conditions on a configurable interval, as a reusable/extensible monitoring **framework** (conditions = first sampler; latency/metrics pluggable later). Telemetry carries conditions + an extensible metrics map but **NEVER** `applied_generation`/checksum — deploy custody stays on `/report` (sidesteps the no-monotonic-guard `SetAppliedGeneration` / resume-cursor hazard).

## Implementation steps

### 1. Agent sampler framework — `internal/agent/telemetry.go` (NEW)
- `type Sampler interface { Name() string; Sample(now time.Time) ([]model.Condition, map[string]any) }`.
- `conditionSampler{stateDir}`: `Sample` → `LoadState`, `collectConditions(prev, prev.LastResult==LastResultOK, now)`, return conds + nil metrics.
- `Telemetry` aggregator: holds `[]Sampler`; `Collect(now)` runs each under `recover()` (a panicking probe can't crash the daemon), merges conditions by `Type` (last-wins), unions metric maps; returns `([]Condition, map[string]any)`.
- `BuildTelemetry(stateDir) *Telemetry` registers `conditionSampler`.

### 2. Agent client — `internal/agent/controller_client.go`
- `telemetryRequestWire` near `:93`: `Conditions []model.Condition`, `Metrics map[string]any`, `AgentVersion string` (NO generation/checksum).
- `func (c *ControllerClient) Telemetry(conds []model.Condition, metrics map[string]any) error` near `postReport:388`: marshal the wire, POST `c.url("telemetry")` via `authedRequest`, reuse `httpClient` (not `pollClient`), best-effort (return non-2xx error to the caller, which logs+swallows). Reads only immutable fields — document the lock-free invariant.

### 3. Agent daemon — `cmd/agent/main.go`
- `o.telemetryInterval time.Duration` parsed from env `YAOG_TELEMETRY_INTERVAL` (default `30s`; `≤0` disables → skip the goroutine). Parse helper near the other env opts.
- In `runControllerMode` daemon branch, before the `for{}` loop: if interval > 0, `tel := agent.BuildTelemetry(o.stateDir); go runHeartbeat(client, tel, o.telemetryInterval, os.Stderr)`.
- `runHeartbeat(client, tel, interval, stderr)`: one immediate `Collect`+`Telemetry`, then `time.NewTicker(interval)` loop doing the same; log+swallow errors. **No `context`** — the daemon never returns and self-update is `syscall.Exec` (destroys the goroutine).
- Single-shot branch unchanged (keeps apply-time `/report` conditions).

### 4. Controller wire + handler — `internal/api/`
- `wire_controller.go`: `telemetryRequestJSON` near `reportRequestJSON:56` — `conditions` (omitempty), `metrics map[string]json.RawMessage` (omitempty — extension slot), `agent_version` (omitempty).
- `handler_agent.go`: `HandleTelemetry` near `HandleReport`: POST-only (405 else), **identity from the node token** (`identity(r.Context())`, not body), decode, `now := time.Now()`, `h.store.RecordTelemetry(ctx, tenant, node, req.Conditions, req.AgentVersion, now)`, 200 `{"status":"ok"}`. **One-line comment:** telemetry is high-frequency observability — intentionally NOT audited (a 30s heartbeat would flood the hash-chained log) and intentionally does NOT touch applied_generation.
- `routes_controller.go`: `mux.HandleFunc(base+"telemetry", h.requireNode(h.HandleTelemetry))` in `RegisterAgentRoutes:195`, beside `report`.

### 5. Controller storage — `internal/controller/`
- `store.go` interface (~:472): `RecordTelemetry(ctx, t TenantID, nodeID string, conditions []model.Condition, agentVersion string, observedAt time.Time) error` — documented to write ONLY Conditions + LastSeen + (if non-empty) LastAgentVersion.
- `memstore.go` + `filestore.go`: implement near `SetAppliedGeneration`. Load node (ErrNotFound if absent), `n.Conditions = stampConditions(conditions, observedAt)`, `n.LastSeen = observedAt`, `if agentVersion != "" { n.LastAgentVersion = agentVersion }`. MUST NOT touch AppliedGeneration/LastChecksum/LastHealth/DesiredGeneration. (Duplicate the few lines rather than over-factor — FileStore's read-modify-writeJSONAtomic shape differs.)

### 6. Docs — `docs/spec/operations/`
Document the `/telemetry` channel, `YAOG_TELEMETRY_INTERVAL`, the dual-write conditions semantics (telemetry = live, `/report` = apply-time, last-writer-wins, back-compat both directions), and the `Sampler` extension point.
