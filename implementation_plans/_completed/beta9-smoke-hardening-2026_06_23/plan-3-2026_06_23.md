# plan-3 — Off-host signing-handle auto-recovery

> Part of [outline.md](./outline.md). Design verified **sound end-to-end** by workflow `w0s4u9zxq` (signing). Security-sensitive (keystone) — gets an extra security-focused review pass.

## Prerequisites
None (file-disjoint). Branch off `main`.

## Goal
A cleared/new browser shows "this browser holds no local signing key" and Deploy throws the misleading "no operator signing key is enrolled" even though the controller HAS a key pinned — forcing a fleet-stranding rotation. Recover the **non-secret** descriptor (credentialId + alg + rpId + audit-only public PEM) from the server so Deploy just prompts for the authenticator tap. **Off-host guarantee unchanged**: the private key never leaves the authenticator; the PEM is already baked into every node bundle + the bootstrap script; the tap is still required per signature; a compromised controller still can't forge a deploy.

## Implementation steps
1. `wire_controller.go`: add `PublicKeyPEM string \`json:"public_key_pem,omitempty"\`` to operatorCredentialStatusJSON; rewrite the "never the PEM body" doc → "non-secret public PEM (audit-only; already in node bundles + bootstrap)".
2. `handler_keystone.go:116`: set `PublicKeyPEM: cred.PublicKeyPEM`; update the doc comments. Pinned:false branch unchanged (omitempty keeps it absent).
3. `keystone_rotation_test.go`: assert the status carries `public_key_pem` after a WebAuthn pin, absent when not pinned; keep the operator-auth-gating test.
4. `controllerClient.ts`: `OperatorCredentialStatus` gains `publicKeyPEM`; `getOperatorCredentialStatus` parses `public_key_pem` (default `''`). Update the "never the PEM body" doc.
5. `controllerStore.ts`:
   - (A) `hydrateKeystoneStatus`: when `st.pinned` AND `st.alg ∈ {webauthn-es256, webauthn-eddsa}` AND `st.credentialId` AND `st.publicKeyPEM`, recover `operatorCredentialId/Alg/RpId/PublicKeyPEM` **into empty slots only** (read `get()` first — never clobber a freshly-enrolled local cache; enrollOperator sets local fields BEFORE hydrate, so the guard holds). Leave raw-ed25519 CLI keystones untouched.
   - (B) `deploy()` belt: before the throw, `await get().hydrateKeystoneStatus()` (now fills local fields) wrapped in try/catch, then re-read the local consts.
   - (C) split the throw: `serverOperatorPinned===false` → "controller has none pinned — enroll one"; pinned-but-unrecovered → "pinned but this browser couldn't recover the descriptor — connect the authenticator / re-enroll on this device". Use `tLocal`.
6. `frontend/src/i18n/messages/en.ts` (canonical, REQUIRED) + `zh.ts` (parity): two new `controllerStore.*` keys.
