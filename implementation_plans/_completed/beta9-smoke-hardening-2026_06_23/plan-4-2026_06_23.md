## Prerequisites
None (file-disjoint). Branch off `main`.

## Goal
The mimic `.deb` "Assist from release" silently no-ops with zero rows and forces hand-typing exact upstream filenames. Add a discover-and-pick flow: a new SSRF-guarded endpoint lists the release's `.deb` assets; the FE shows a checklist; the operator picks + labels `<codename>-<arch>` and the existing per-row Assist fills SHA-256.

## Implementation steps
1. **apierr codes** — EITHER add `CodeReleaseAssetsRequestInvalid`(400)/`CodeReleaseAssetsFetchFailed`(502) at THREE sites (const + `registry` + `allCodes`; `apierr.New` panics on an unregistered code and `TestRegistryBijection` asserts the three sets match) OR **reuse `CodeAgentReleaseRequestInvalid`/`CodeAgentReleaseFetchFailed`** (identical statuses/params, zero bijection churn — the FE localizes by code). **Recommended: reuse.**
2. `internal/api/release_assets.go` (NEW) — methods on `*ControllerHandler`:
   - `deriveReleasesAPIURL(base)`: `releases/latest/download` → `https://api.github.com/repos/<o>/<r>/releases/latest`; `/releases/download/<tag>` → `.../releases/tags/<tag>`; **pin host to api.github.com**; strict `^[A-Za-z0-9._-]+$` on owner/repo/tag; hard-fail a non-github base.
   - `HandleReleaseAssets` (POST, operator-gated): body `{kind, base?}`; `h.loadSettings` for gh-proxy + default base (`kind=mimic`→cs.MimicReleaseBase); `base = req.Base || default`, require non-empty + `validateReleaseURL`; `apiURL = deriveReleasesAPIURL(base)`; gh-proxy prefix; `validateReleaseURL` the composite; `fetchReleaseAssetNames`. Response `{assets, base, proxy_applied}`.
   - `fetchReleaseAssetNames(ctx, url)`: GET via `h.releaseClient` (Accept: application/vnd.github+json), `io.LimitReader` cap (≈256KB), non-200 → fetch-failed (HTTP code in detail, no IP), JSON unmarshal (HTML/rate-limit page → fetch-failed, NOT empty-success), filter `*.deb` excluding `dbgsym`/`.ddeb`, dedupe, cap (≈200). **Return dkms names too** (don't hide assets — the FE labels them blank).
3. `routes_controller.go`: register `release-assets` under the operator mux beside `release-pins`.
4. `controllerClient.ts` + `controllerStore.ts`: `fetchReleaseAssets(body)` (mirror fetchPins, map `proxy_applied`).
5. `MimicCatalogSettings.tsx`:
   - **Discover** button (requires releaseBase) → `fetchReleaseAssets({kind:'mimic', base})` → checklist of `{asset, label, checked}`. `deriveKey(asset)`: match `^([a-z0-9]+)_mimic_.*_([a-z0-9]+)\.deb$` (PLAIN package) → `${codename}-${arch}`; **return `''` for `-dkms`/unmatched** (operator must label). Validate the label with `DEB_KEY_RE`.
   - **Block two checked rows yielding the SAME key** (the dkms collision: `filledDebs()` keys by codename-arch and silently keeps the last → pin loss) with an inline "duplicate key — edit the label" hint.
   - "Add selected" appends `DebRow{key, asset, sha256:''}` (empty SHA — custody: operator then runs the existing Assist to fill SHA-256 + reviews + Saves; never auto-fill a hash here). Empty discovery → a note; thrown fetch → discoverError (tError-localized).
6. `frontend/src/i18n/messages/en.ts` (REQUIRED) + `zh.ts` (parity): `mimicCatalog.discover*` keys.
