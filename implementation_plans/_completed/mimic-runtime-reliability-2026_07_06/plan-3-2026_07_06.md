# plan-3 — pre-deploy "can this node run mimic" capability probe + panel warning

**Goal:** warn BEFORE deploy when a node can't run mimic (the exact hkg14 case: kernel module can't be
built because `linux-headers-$(uname -r)` isn't installable / DKMS not `installed`), so an operator
sees it up front rather than after a failed apply. Extends the beta.10 Sampler + the rc.2 native-XDP
probe pattern (agent → opaque telemetry metric → panel reads by key; ZERO controller change).

Reads from specs: `docs/spec/artifacts/mimic.md`; the telemetry Sampler framework.

## Changes
- **Fix the rc.2 native-XDP display gate (owner-reported 2026-07-06).** The native-XDP capability is
  fully wired (agent `nativeXDPSampler` registered → `telemetry.native_xdp` → `Node.nativeXDP` →
  NodeEditor), but `NodeEditor.tsx:252` only DISPLAYS the hint when `xdp_mode === 'native'` is already
  selected — so an operator can't see whether a node supports native BEFORE choosing it. Make the
  capability an **always-visible per-node indicator** (a small chip/line, independent of the current
  xdp_mode), so it's a real pre-decision aid: supported / conditional (virtio) / unsupported / unknown
  + the driver. Keep the extra "you selected native but this NIC is unsupported" warning styling when
  native IS selected. Note the live-only caveat (stripped from the offline cache; needs a live poll +
  an rc.2+ agent heartbeat) in the hint's empty state if helpful.
- **Agent** (`internal/agent/telemetry_mimic_capability.go`, NEW): a pure, read-only Sampler →
  `metrics["mimic_capability"] = {capability, kernel, reason}` where capability ∈
  `ready` (module `installed` for `uname -r` OR already loaded) / `buildable` (headers present, dkms
  `added`/`built` but not yet installed) / `unbuildable` (headers for `uname -r` not installed AND not
  installable-known) / `unknown`. Signals via indirected readers (testable): `dkms status` parse (or
  `/var/lib/dkms/mimic`), `modinfo mimic`/`/lib/modules/$(uname -r)/updates/dkms`, `uname -r`. NO
  package installs, NO builds, NO shelling to apt — inspection only. Node-global (egress-independent,
  so unaffected by plan-2). Register in `internal/agent/telemetry.go`.
- **Panel:** `controllerClient.ts` map `telemetry.mimic_capability → Node.mimicCapability` (live-only;
  add to `stripLiveTelemetry` like `nativeXDP`); `types/controller.ts` the type; `NodeEditor.tsx` a
  hint when a node has a tcp link and capability is `unbuildable` (warning: "this kernel can't build
  the mimic module — reboot into the current kernel or use mimic_fallback=udp"), muted for `buildable`,
  success for `ready`. i18n (en+zh).
- Tests: agent classify/parse unit tests (indirected readers); the FE wire-mapping test (like
  `mapNativeXDP`). No conformance impact (telemetry is runtime).
