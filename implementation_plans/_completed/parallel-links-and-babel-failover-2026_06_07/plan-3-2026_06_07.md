## Implementation steps

1. **Types/i18n** — `Edge.role?: 'primary' | 'backup'`; STRINGS tuples: addBackupLink, roleChip
   labels, duplicateWarnChip, backupEndpointNudge.
2. **Add-backup button** — RightPanel edge section: visible when an edge is selected; creates
   `{...pair, id: crypto.randomUUID(), role: 'backup', endpoint_host: <target endpoint>}`,
   selects it, and shows the path-diversity nudge ("point the backup at a distinct endpoint").
   Store action `addBackupEdge(primaryEdgeId)` to keep the logic out of the component.
3. **Role chips** — CustomEdge label pill gains a role chip: ★ primary (the pair's primary-class
   link), `bN` for backups (ordinal by parallelIndex among backups), ⚠ duplicate? for roleless
   same-direction extras (mirrors the plan-2 D71 warning semantics). Chip fields join the
   keyed-sync equality list.
3b. **Show-interfaces chips become edge-aware (Decisions #12)** — the node-card chips currently
   derive `peerName` by stripping `wg-` from the interface name; a backup interface
   (`wg-<clean8><hash4>`, e.g. `wg-betaa3f2`) would render a garbage chip `betaa3f2:51823`.
   Replace the string-strip with per-edge resolution: match each compiled interface to its edge
   by pinned port (ports unique per node), then render `<peerName>:<port>` with the SAME role
   marker as the edge fan (★ / bN) and the real interface name in the tooltip. One resolver
   shared with step 4 (extract a small lib helper, e.g.
   `frontend/src/lib/compiledInterfaces.ts`, so RightPanel and CustomNode cannot drift apart).
4. **Per-edge compiled values** — replace RightPanel:766-804 target-name matching with
   pinned-port matching: find the wireguard_configs entry for the from-node whose
   `ListenPort == selectedEdge.pinned_from_port` (ports unique per node ⇒ deterministic). Also
   closes the carried-forward PR #6 prefix-match note.
5. **Focus-transparency (Decisions #11)** — de-emphasis is TRANSPARENCY, never hiding: every
   de-emphasized element stays mounted, rendered, and **clickable** (opacity does not affect
   hit-testing, so clicking a transparent node/edge moves focus to it), at very low opacity
   (`~0.15` — faint but present, tune by eye in smoke) with the existing 150ms transitions so
   focus changes fade rather than snap. Local `connectingFrom` state via React Flow
   `onConnectStart`/`onConnectEnd`; compute per-element `deemphasized` in the two sync effects:
   node selected → everything except it + its incident edges; edge selected → everything except
   it + its two endpoint nodes; connecting → ALL edges (nodes stay fully opaque — they are the
   drop targets). **Background click returns to normal**: `onPaneClick` already clears all
   selection, and `deemphasized` derives from selection state, so a pane click fades everything
   back to full opacity — verify explicitly (it must also hold after a connection drag is
   cancelled, i.e. `onConnectEnd` always resets `connectingFrom` even on an aborted drag).
   `deemphasized` joins edgeRenderEqual keys and node data. No `display:none`, no conditional
   unmount, no pointer-events suppression.
6. **Arc separation** — bump `offsetStep` 30→48 and label offset accordingly (fleet note: 18px
   label gap reads as a glitch).
7. **Role editing** — small primary/backup select next to priority/weight editors; helper text
   explains preset cost mapping (96-default vs 384) and that explicit priority/weight overrides.
