## Implementation steps (ORDER IS LOAD-BEARING)

1. **Per-CIDR transit index (D12):** replace global `transitPairIndex` with
   `transitPairIndexByCIDR map[string]int` keyed by resolved transit CIDR; each pool consumes
   independently from 0. Tests BEFORE proceeding.
2. **Pool hygiene (D48/D70):** allocateTransitPair excludes network/broadcast for the mask;
   allocateLinkLocalPair formats hex (`fe80::%x`). Document the /24 → 126-pair capacity.
3. **SNAT parameterization (D38/D39):** `InstallScriptConfig.TransitCIDRs []string` (the node's
   distinct resolved transit CIDRs, default 10.10.0.0/24); template emits one SNAT rule per CIDR
   in install rule, nft/iptables variants, AND overlay-snat.service ExecStart/ExecStop.
4. **Model:** Edge gains `PinnedFromPort, PinnedToPort int` + `PinnedFromTransitIP,
   PinnedToTransitIP, PinnedFromLinkLocal, PinnedToLinkLocal string` (all omitempty); Topology gains
   `AllocSchemaVersion int `json:"alloc_schema_version,omitempty"``. Mirror in TS Edge.
5. **Reserve-then-gap-fill (Pass 1 rewrite):** `pinKey(a,b string)` = sorted `min|max` (also
   replaces the ad-hoc pair-dedup key); FIRST loop reserves every non-zero pinned value into
   `usedPorts map[nodeID]map[int]bool`, `usedTransitIdx map[cidr]map[int]bool` (reverse-map IP →
   index via the 2i+1 formula), LL set; SECOND loop allocates: pinned pairs use pins verbatim,
   unpinned gap-fill via `nextFreePort(nodeID, base)` / `nextFreeTransitIndex(cidr)` seeded from
   `H(pinKey)` (hash-seeded re-add idempotence). *Stop-loss:* reverse-mapping fragile → store
   verbatim transit IPs in the used-set (skip index math) and draft plan-7.5.
6. **Write-back:** compiler.go stamps chosen values onto all six Pinned* fields for every
   enabled edge (not just CompiledPort) + sets AllocSchemaVersion=1.
7. **Key reuse:** generateKeys default branch — node with non-empty WireGuardPublicKey routes
   through the fixed-key path (reuse); only empty-key nodes get fresh keys; chosen public keys
   written back so they round-trip. (Private keys: unchanged handling — only persisted when
   FixedPrivateKey, per security spec.)
8. **validateAllocationPins:** duplicate pinned ports per node / transit / LL across edges →
   error; pinned transit outside domain's TransitCIDR → error; pinned port base-mismatch after
   ListenPort change → warning + auto-clear; pins on client-role edges → warning + ignore.
9. **Frontend:** read-only pin display in edge panel + "Unpin (renumber on next compile)" button
   clearing the six fields; store/persistence need no change (edges already round-trip).
10. Tests, commit, push, PR #7. Commit template: `feat(allocation): <step> — Spec B I1-I10 (T13/D12/D38/D39/D48/D70)`
