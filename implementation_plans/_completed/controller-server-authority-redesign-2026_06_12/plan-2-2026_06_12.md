# plan-2 — Backend: topology version history
<!-- drafted: 2026-06-12 -->
Outline: [`outline.md`](outline.md) · Prerequisites: plan-1 (custody rejection shapes what enters history)

## Goal

Replace the single overwritten `TopologyRecord` with a bounded history: every `PutTopology`
appends a version and prunes to the last 10 (D7). Expose `GET /topology?version=N` and a
version list, so a bad overwrite (empty-canvas deploy, botched import) is recoverable without
filesystem backups. Rollback UI is out of scope (D10) — this is the API substrate.

Reads from specs: controller-store, controller-stage-promote

## Implementation steps

1. **Store interface** additions (keep PutTopology signature unchanged — Version continues to
   increment as today):
   ```go
   // ListTopologyVersions returns the retained versions, newest first (≤ TopologyHistoryLimit).
   ListTopologyVersions(ctx context.Context, t TenantID) ([]TopologyVersionInfo, error)
   // GetTopologyVersion returns a specific retained version, or ErrNotFound.
   GetTopologyVersion(ctx context.Context, t TenantID, version uint64) (TopologyRecord, error)
   ```
   `TopologyVersionInfo{Version uint64; UpdatedAt time.Time; Bytes int}` (no JSON payload in the
   list — cheap to serve).
2. **FileStore**: keep `topology.json` as the CURRENT record (unchanged readers stay working);
   add `topology-history/<version>.json` written in the same PutTopology call (atomic
   temp+rename each, 0600), prune oldest beyond `TopologyHistoryLimit = 10`. Crash-safety: write
   history file BEFORE flipping topology.json; a crash between leaves a harmless extra history
   entry. GetTopology unchanged.
3. **MemStore**: ring of the last 10 records alongside the current pointer.
4. **API**: `HandleTopology` gains `?version=N` (operator-auth as today; 404 on unknown version)
   and a sibling `GET /topology/versions` route registered in RegisterOperatorRoutes returning
   the info list. Audit: no new entries (reads).
5. **Docs touch**: `docs/spec/controller/persistence.md` stored-shapes section gains the history
   layout (full doc pass remains plan-7).
6. Commits per substep on branch `plan-2-topology-history`; PR.

pre-deploy design; prune verified by 11 successive PUTs in the store test.
