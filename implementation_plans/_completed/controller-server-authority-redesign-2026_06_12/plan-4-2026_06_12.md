# plan-4 — Panel: full-page login gate + hydrate-on-login
<!-- drafted: 2026-06-12 -->
Outline: [`outline.md`](outline.md) · Prerequisites: plan-1 merged (panel speaks the new operator-prefix env semantics); plan-2 merged (history protects against bad first deploys during rollout)

## Goal

Entering the panel with persisted controller mode shows a **full-viewport login page** before
any shell chrome (D2); a successful login or cookie session-restore **hydrates the canvas from
the server** (`GET /topology` → `loadTopology`), overwriting the local cache (D1), with a
one-time export stash of a differing non-empty local design (D9). The login form moves out of
`ConnectionSettings`; Settings keeps connection/prefix fields only.

Reads from specs: panel-auth, panel-shell, panel-deploy-fleet, panel-design

## Implementation steps

1. **`components/auth/LoginPage.tsx`** (new): full-viewport layout (brand mark, app name,
   username/password, conditional TOTP leg, "Sign in with passkey", collapsed break-glass
   disclosure, language toggle, theme-correct in dark/light). Reuses controllerStore actions
   verbatim; form logic extracted from ConnectionSettings (move, don't fork). Apple-minimal
   styling consistent with the appshell redesign (`.app-chrome` tokens).
2. **Gate in Shell** (`Shell.tsx`): while `mode==='controller'`: render LoginPage instead of the
   sidebar+topbar+`<Outlet/>` whenever `!loggedIn`; show a quiet "checking session…" splash
   until the mount `checkSession()` resolves (no canvas flash). Deep links: the requested path
   stays in the router; after login the gate unmounts and the route renders (verify
   `/fleet/nodes/:id`). Local mode: completely unaffected.
3. **Hydration**: new controllerStore action `hydrateFromServer()`:
   `getTopology(configOf(get()))` → on 200, `useTopologyStore.getState().loadTopology(parsed)`;
   on 404 (no topology yet) leave the local canvas as-is (first-ever-deploy case). Called from
   the two success paths: `login()` and `checkSession()`-restores-session. Overwrite always (D1).
4. **One-time export stash (D9)**: before the FIRST overwriting hydration where the local design
   is non-empty AND differs from the server copy (compare canonical JSON), trigger
   `exportProject()` download named `pre-hydration-backup-<date>.json` + a dismissible notice
   ("local design replaced by the server copy; backup downloaded"). Persist a
   `hydrationStashDone` flag in controllerStore's allowlist so it fires once per browser.
5. **ConnectionSettings cleanup**: login/identity sections removed (now LoginPage); keep
   baseURL/pathPrefix/agentBaseURL + sign-out moved to UserMenu (`UserMenu.tsx` placeholder gets
   its first real content: identity + sign-out).
6. **i18n + a11y sweep** over every new string/control (focus order, labels, skip-link still
   first).
7. Branch `plan-4-login-gate-hydration`; commits per substep
   (`feat(panel): full-page controller login gate`, `feat(panel): hydrate canvas from server on login`,
   `feat(panel): one-time pre-hydration export stash`, `refactor(panel): move login out of ConnectionSettings`); PR.
8. **Rollout step**: PR body notes operator-visible change (everyone re-logs-in; first login
   downloads the stash file if they had local work).

## Verification

```bash
cd frontend && npm run lint && npm run build
```
Manual (dev controller + real deployment): cache-clear → open panel → login page (no chrome
flash) → login → canvas shows the server design; refresh → session cookie restores + hydrates;
deep link while logged out lands on the gate then the target page; local mode untouched;
stash file downloads exactly once when local differed.
