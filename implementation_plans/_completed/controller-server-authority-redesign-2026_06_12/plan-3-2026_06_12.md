# plan-3 — Backend: safety bugs (agent backoff, staged-bundle purge, audit gap)
<!-- drafted: 2026-06-12 -->
Outline: [`outline.md`](outline.md) · Prerequisites: none (independent of 1–2; ordered here to keep backend work contiguous)

## Goal

Fix the three audit-confirmed safety bugs: (a) an orphaned non-keystone agent (fleet row whose
node-ID left the design) busy-loops re-fetching and re-running `install.sh` as root with no
delay; (b) promote flips ALL staged bundles, so a stale staged bundle for a since-removed node
goes live on a later promote; (c) a stage that stages zero nodes leaves no audit trace.

Reads from specs: agent, controller-stage-promote, keystone-trustlist

## Read first

1. `cmd/agent/main.go:370-383` (daemon loop: `errBackoff` on error path ONLY; success path has
   no sleep), `internal/agent/cycle.go:101-181` (poll/resume cursor semantics; rekey-wake
   watermark — do NOT regress it), `internal/agent/state.go:142-146` (idempotent re-apply allowance).
2. `internal/controller/filestore.go:513-584` (StageBundle/PromoteStaged),
   `memstore.go:266-276`, `internal/controller/compile.go:160-196` (stage set, early return).
3. Audit finding nuances: keystone orphans already back off 5s via the error path
   (`verify.go:320-322` refusal → error); the hot loop is ONLY the non-keystone orphan whose
   re-apply "succeeds".

## Implementation steps

1. **Agent idle/no-op backoff** (`cmd/agent/main.go` + `cycle.go`): detect the no-op cycle —
   applied checksum equals the last-applied checksum AND generation didn't advance — and treat
   it as idle: skip re-running install.sh (short-circuit before apply) and sleep `errBackoff`
   (5s) before the next poll. Preserve: rekey-wake watermark advance; genuinely new bundles
   apply immediately; `--once` semantics unchanged. Signature sketch:
   `func (c *Cycle) shouldApply(fetched Bundle) bool` consulting `State.LastChecksum`.
2. **Promote scoping** (both stores): `PromoteStaged` flips only bundles whose staged generation
   matches the stage being promoted; `CompileAndStage` deletes staged bundles for node-IDs NOT
   in the current stage set (purge-on-stage). Wire the StageResult to carry the staged set (it
   already does — `compile.go:105-118`).
3. **Empty-stage audit** (`compile.go:164-166`): before the early return, append a `stage`
   audit entry with `staged=0` + the skipped list, so a design-destroying deploy is visible.
4. Commits per substep on branch `plan-3-safety-bugs`; PR. Each fix independently revertable.

## Verification

CI. Unit: simulated orphan cycle (frozen generation) performs zero applies and sleeps; promote
test: staged bundle from gen N is NOT flipped by promote of gen N+1's stage; purge test; audit
entry asserted for the empty stage. Manual (optional, two-node smoke owed at closure anyway).
