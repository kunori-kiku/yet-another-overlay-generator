## Goal
`cmd/agent/main.go` (983 LOC) fuses CLI dispatch with the entire controller-mode daemon (the run loop,
the heartbeat, the coalescing kick) as free functions — the subsystem framework-refactor never touched.
The **root self-replacing control loop is untestable today** (it lives in `func main`-adjacent free
functions with process-level side effects). Decompose behavior-preservingly so the loop can be tested.

## Implementation steps
1. Extract the controller-mode daemon (`runControllerMode` + `runHeartbeat` + `tryKick` + the run-loop
   state) into a testable type in `internal/agent` (e.g. `agent.Daemon` with injectable clock / poster /
   applier seams), leaving `cmd/agent/main.go` a thin CLI dispatcher that constructs + runs it.
2. Move the per-subcommand bodies (`runKeygen`/`runKit`/`runKitVerify`/`runEnroll`/
   `runReprovisionKeystone`) into per-command files (`cmd/agent/cmd_*.go`) or `internal/agent` helpers —
   whichever keeps `main.go` a dispatcher without inflating `internal/agent`'s public surface.
3. Add tests the god-`main` made impossible: the heartbeat coalescing (lost-wakeup / no-leak on
   shutdown), the run-loop apply→kick→reconcile path, and the self-replace breadcrumb interaction under
   `-race`. Address any real race the gap-pass sampler concern surfaces (context cancellation / shared
   state) — but do NOT churn the sound breadcrumb/rollback design.
