## Goal
Close the T6 controller-store correctness cluster + the gap-pass custody finding:
1. **(MED, custody)** `HandleTrustListSignature` is **not** serialized under `lockTenantOps` (unlike
   Enroll / Rekey / Stage / Promote), so a concurrent re-stage can leave a stale signed manifest
   (`M_old`) paired with freshly-staged bundles (`B_new`); the promote gate checks only sig-vs-pin and
   silently promotes the desynced pair → nodes fetch a `B2` bundle bound to `M1`, `VerifyMembership`
   mismatches, and the fleet strands (fail-closed, recoverable).
2. **(MED)** Promote crash-atomicity window (a torn served/staged pair on an ill-timed crash).
3. **The telemetry-durable overlay-leak** partition — PRESERVE FileStore's deliberate volatile overlay
   (do NOT force a durable write; the framework-refactor decision-6 correction).
4. **(LOW)** `writeJSONL` requeues already-durably-written samples on a post-`Write` `Close()` error →
   duplicate JSONL lines double-counting one aggregation bucket (observability-only).

## Implementation steps
1. Wrap `HandleTrustListSignature`'s manifest-mutation under `lockTenantOps` (same tenant-scoped lock as
   stage/promote), so a signature submission and a re-stage cannot interleave into an `(M_old, B_new)`
   pair. **Audit lock ordering first** (stop-loss): confirm no nested acquisition against the existing
   holders (deadlock) — if the handler already calls a store method that takes the lock, restructure to
   a single acquisition, do not double-lock.
2. Add the promote crash-atomicity fix (atomic served/staged snapshot on promote — model on the existing
   `GetServedConfig` atomicity from beta.5).
3. Confirm the telemetry overlay stays volatile (a test that MemStore's overlay is not durably written);
   do not "fix" it to durable.
4. Fix `writeJSONL`: on a `Close()` error after a successful `Write`, do NOT requeue the already-written
   batch (it is durably appended) — treat the samples as flushed; log the close error. Add a test that a
   post-write close error does not duplicate lines on the next flush.
5. Regression tests: the concurrent sign-vs-restage desync (proves promote refuses / serializes); the
   `writeJSONL` no-dup.
