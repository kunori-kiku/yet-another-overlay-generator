# plan-5 — Mimic UDP fallback mechanism: install-script branch + agent breadcrumb detection + `mimic` condition

## Prerequisites

- **plan-1** (Track A) — the `model.Condition{Type,Status,Reason,Message,Since}` shape, the report-wire
  `conditions` field (additive/omitempty), the agent-side condition collector + `classify` pattern, and
  `Node.Conditions` in the controller store. This plan adds a **new producer** (`mimic`) on top of that
  collector; it does not re-define the channel.
- **plan-4** (Track B) — the per-link `Edge.MimicFallback` tri-state (`""`/`udp`/`none`),
  `ControllerSettings.MimicFallbackDefault`, and the **effective per-link policy** resolved into
  `PeerInfo` (the field this plan reads to emit the script branch). This plan consumes that resolved
  policy; it does not re-resolve inheritance.

If either is not yet merged, STOP and surface it — do not stub the channel or the policy field locally.

## Goal

When a link's effective mimic policy is `udp` and mimic provisioning fails (kernel too old / eBPF load
fails / `.deb` install fails / `systemctl enable --now mimic@<egress>` fails), the rendered `install.sh`
DETECTS the failure category with explicit checks, SKIPS mimic, brings the WireGuard link up as plain
UDP (the link must come up fully — endpoint/port are already mimic-independent and the mimic-reduced
MTU is conservative-safe for UDP), and writes a small **Go-constant-keyed JSON breadcrumb** (never raw
stderr). When the effective policy is `none`, today's fail-closed behavior (`set -euo pipefail` abort)
is preserved unchanged, but the script STILL writes a categorized breadcrumb before aborting so the
panel can show why. The agent reads the breadcrumb each cycle, `classify()`-es it into a closed-enum
`mimic` condition (`KernelTooOld` | `EbpfLoadFailed` | `InstallFailed` | `FellBackToUDP` | `Active`)
with a SHORT curated message, and emits it through the plan-1 channel.

## Implementation steps

### Step 1 — Define the breadcrumb path + the closed reason enum as Go constants (single source of truth)

The breadcrumb is keyed by **Go-emitted constants** so the script writer and the agent reader cannot
drift, and so no upstream/user text becomes a key. Put them in a new small file
`internal/agent/mimic_condition.go` (the agent owns the read + classify; the renderer imports the path
constant from here to avoid a back-edge — renderer already imports `model`, and the agent imports
`renderer` indirectly via cmd, so place the **path** constant in `model` and the **reason** constants in
`agent`).

Add to `internal/model/artifact.go` (next to `InstallFetch`, `:27`):

```go
// MimicBreadcrumbPath is the absolute path install.sh writes the mimic-provisioning outcome
// breadcrumb to (a small JSON marker keyed by Go constants — never raw stderr). The agent reads
// it each cycle to emit the `mimic` Node Condition. It lives outside the bundle (host-local mutable
// state) so it survives re-applies, mirroring the agent state dir. The directory is created 0700 by
// install.sh under root. See docs/spec/artifacts/mimic.md (UDP fallback) + docs/spec/agent.
const MimicBreadcrumbPath = "/var/lib/yaog-agent/mimic-status.json"

// MimicOutcome* are the closed-enum outcome codes install.sh writes into the breadcrumb's "outcome"
// field. They are the ONLY values the script emits; the agent's classify() maps each to a Condition
// reason. Adding a value here is a coordinated script+agent change (the contract is closed).
const (
	MimicOutcomeActive        = "active"          // mimic provisioned + mimic@<egress> started
	MimicOutcomeKernelTooOld  = "kernel_too_old"  // eBPF/bpffs absent → mimic cannot load
	MimicOutcomeEbpfLoad      = "ebpf_load_failed" // module present but XDP/TC attach failed
	MimicOutcomeInstallFailed = "install_failed"  // distro pkg + pinned .deb both failed
	MimicOutcomeFellBackToUDP = "fell_back_to_udp" // policy=udp: skipped mimic, link up as plain UDP
)
```

Backward-compat: a node with no `tcp` link never writes the breadcrumb → the agent reads ENOENT → emits
no `mimic` condition (the absence is the zero value, not an error).

> **Per-node resolution rule:** mimic provisioning is per-NODE (one `mimic@<egress>` unit for all this
> node's mimic ports). The policy is per-LINK in plan-4. Resolve to a single node-level boolean as
> **"udp if ALL mimic links on this node resolve to udp, else none (fail-closed)"** — fail-closed wins on
> a mixed node so a `none` link is never silently de-cloaked by a sibling `udp` link. Document this in the
> field comment and in `docs/spec/artifacts/mimic.md`. Implement it in a pure helper:

```go

// True only when EVERY mimic link (p.Mimic) resolves to the "udp" policy (plan-4); a single "none"
// (or inherit-off) mimic link forces fail-closed for the whole node, since one shared mimic@<egress>
// unit serves all ports and partial fallback is not representable. Nodes with no mimic link return
// false (no fallback branch rendered).
func resolveMimicFallbackUDP(peers []compiler.PeerInfo) bool
```

### Step 3 — Rewrite the Phase-3 mimic provisioning block to branch on policy (per-peer template)

Replace the current hard `systemctl enable --now "mimic@${MIMIC_EGRESS_IF}"` (`script.go:721`) with a
deterministic, root-safe detect-and-branch block. The new template fragment (inside the existing
`{{ if .HasMimic -}}` … `{{ end -}}` at `:694-723`):

```bash
echo "Provisioning mimic TCP-shaping transport..."
MIMIC_EGRESS_IF="$(ip route show default 2>/dev/null | awk '{for(i=1;i<=NF;i++) if($i=="dev"){print $(i+1); exit}}')"
MIMIC_EGRESS_IP="$(ip route get 1.1.1.1 2>/dev/null | awk '{for(i=1;i<=NF;i++) if($i=="src"){print $(i+1); exit}}')"
if [ -z "$MIMIC_EGRESS_IF" ] || [ -z "$MIMIC_EGRESS_IP" ]; then
    echo "ERROR: could not detect egress interface/IP for mimic (no default route?)" >&2
    exit 1
fi
echo "  mimic egress: $MIMIC_EGRESS_IF ($MIMIC_EGRESS_IP)"
mkdir -p /etc/mimic /var/lib/yaog-agent

# _mimic_breadcrumb <outcome>: write a small Go-constant-keyed JSON marker the agent reads to emit the
# `mimic` Node Condition. Only the OUTCOME token (a fixed shell literal from a Go constant) and the
# detected egress IF (a kernel-derived NIC name) are interpolated — never captured stderr.
_mimic_breadcrumb() {
    printf '{"outcome":"%s","egress":"%s","ts":"%s"}\n' \
        "$1" "$MIMIC_EGRESS_IF" "$(date -u +%Y-%m-%dT%H:%M:%SZ)" \
        > {{ shq .MimicBreadcrumbPath }}
}

# eBPF/bpffs sanity gate — mimic is an eBPF (TC/XDP) program; an old kernel without BPF cannot run it.
if [ ! -d /sys/fs/bpf ] && ! grep -qw bpf /proc/filesystems 2>/dev/null; then
{{ if .MimicFallbackUDP -}}
    echo "WARNING: kernel lacks eBPF/bpffs; mimic unavailable — falling back to plain UDP (policy=udp)" >&2
    _mimic_breadcrumb {{ shq .MimicOutcomeKernelTooOld }}
{{ else -}}
    echo "ERROR: kernel lacks eBPF/bpffs; mimic required by policy (no fallback)" >&2
    _mimic_breadcrumb {{ shq .MimicOutcomeKernelTooOld }}
    exit 1
{{ end -}}
else
    # Write the egress config and start the unit.
    {
        {{ range .MimicPorts -}}
        echo "filter = local=${MIMIC_EGRESS_IP}:{{ . }}"
        {{ end -}}
        echo "xdp_mode = {{ .MimicXDPMode }}"
    } > "/etc/mimic/${MIMIC_EGRESS_IF}.conf"
    echo "  Wrote /etc/mimic/${MIMIC_EGRESS_IF}.conf"
    if systemctl enable --now "mimic@${MIMIC_EGRESS_IF}"; then
        echo "  Started mimic@${MIMIC_EGRESS_IF}"
        _mimic_breadcrumb {{ shq .MimicOutcomeActive }}
    else
{{ if .MimicFallbackUDP -}}
        echo "WARNING: mimic@${MIMIC_EGRESS_IF} failed to start; falling back to plain UDP (policy=udp)" >&2
        # De-provision the half-applied filter so no orphaned mimic shaping survives on a UDP link.
        systemctl disable --now "mimic@${MIMIC_EGRESS_IF}" 2>/dev/null || true
        rm -f "/etc/mimic/${MIMIC_EGRESS_IF}.conf"
        _mimic_breadcrumb {{ shq .MimicOutcomeEbpfLoad }}
{{ else -}}
        echo "ERROR: mimic@${MIMIC_EGRESS_IF} failed to start; mimic required by policy (no fallback)" >&2
        _mimic_breadcrumb {{ shq .MimicOutcomeEbpfLoad }}
        exit 1
{{ end -}}
    fi
fi
```

### Step 5 — Agent: read the breadcrumb + `classify()` → `mimic` condition

New file `internal/agent/mimic_condition.go`:

```go
// MimicReason is a closed enum of `mimic` Node Condition reasons (CamelCase codes). It is the curated
// classification of install.sh's breadcrumb outcome — NEVER a raw error string. (D5, PRINCIPLES curated.)
type MimicReason string

const (
	MimicActive        MimicReason = "Active"
	MimicKernelTooOld  MimicReason = "KernelTooOld"
	MimicEbpfLoadFailed MimicReason = "EbpfLoadFailed"
	MimicInstallFailed MimicReason = "InstallFailed"
	MimicFellBackToUDP MimicReason = "FellBackToUDP"
)

// mimicBreadcrumb is the on-disk JSON install.sh writes (model.MimicBreadcrumbPath). Only the closed
// outcome token is trusted; egress/ts are advisory and never interpolated anywhere.
type mimicBreadcrumb struct {
	Outcome string `json:"outcome"`
	Egress  string `json:"egress"`
	TS      string `json:"ts"`
}

// classifyMimic maps a breadcrumb outcome token to a Condition reason, status, and a SHORT curated
// message (single line, never raw stderr). The 160-rune cap is NOT applied here — the message is
// routed through plan-1's classify(...) (the single cap chokepoint), so this returns the curated
// strings as-is. Unknown tokens map to a generic warn with a safe message so a future script value
// never crashes an old agent. Active maps to model.ConditionStatusOK; every failure/fallback maps to
// model.ConditionStatusWarn (a UDP fallback de-cloaks the link and must be surfaced loudly).
func classifyMimic(outcome string) (reason MimicReason, status model.ConditionStatus, message string)

// readMimicCondition reads model.MimicBreadcrumbPath and returns the model.Condition the report should
// carry, or (Condition{}, false) when the breadcrumb is absent (no tcp link / never provisioned).
// It builds the Condition via plan-1's classify(model.ConditionTypeMimic, status, reason, message,
// since) so the 160-rune cap (model.ConditionMessageMax) is applied at the single chokepoint.
// breadcrumbPath is injectable so tests never touch /var/lib.
func readMimicCondition(breadcrumbPath string) (model.Condition, bool)
```

`classifyMimic` table (the curated map — this is the load-bearing contract):

| outcome token (Go const) | reason | status | message |
| --- | --- | --- | --- |
| `active` | `Active` | `ok` | `mimic TCP-shaping active` |
| `kernel_too_old` | `KernelTooOld` | `warn` | `Mimic unavailable: kernel lacks eBPF` |
| `ebpf_load_failed` | `EbpfLoadFailed` | `warn` | `Mimic eBPF load failed` |
| `install_failed` | `InstallFailed` | `warn` | `Mimic install failed` |
| `fell_back_to_udp` | `FellBackToUDP` | `warn` | `Mimic: fell back to plain UDP` |
| *(unknown)* | `Active` is NOT assumed → return a generic `warn` reason `"Unknown"`, message `"Mimic status unrecognized"` | `warn` | — |

`Condition.Type` is `model.ConditionTypeMimic` (plan-1's exported const — never a local `condTypeMimic`
or a bare `"mimic"` literal); `Since` is set by `classify` (plan-1) to the breadcrumb `ts` when
parseable, else the read time. The message is routed through plan-1's `classify(...)` — the single cap
chokepoint, which caps to `model.ConditionMessageMax` (160) INSIDE `classify`; there is no separate
`capMessage` helper and no local 80-rune cap.

**Wire it into the collector:** call `readMimicCondition(model.MimicBreadcrumbPath)` from the plan-1
agent condition collector (the function that assembles `[]model.Condition` for the report). Anchor: the
collector plan-1 added near `recordSuccess`/`recordFailure` (`agent.go:312-…`) and that
`RunControllerCycle` (`cycle.go:88-220`) drains into the report. The breadcrumb path is injectable in
tests via the same StateDir-style config the agent already threads (`Config.StateDir`, `state.go:21`);
if plan-1's collector takes no path arg, add an optional `MimicBreadcrumbPath string` to the collector
config defaulting to `model.MimicBreadcrumbPath` (omitempty/zero-value safe).
