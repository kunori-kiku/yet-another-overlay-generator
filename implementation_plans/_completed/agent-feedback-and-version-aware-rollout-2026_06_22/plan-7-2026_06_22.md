# Plan 7 — Controller version self-awareness (backend)

## Prerequisites

None. This is the **Track C** start (outline §7). It depends on no other plan in this
subject. It is, however, a **prerequisite for plan-8** (panel version display +
"Update all agents → panel version" + the refuse-newer guard): the controller version
surfaced here is the source of truth plan-8 reads to default the rollout target and to
reject a target `> controllerVersion`. Keep that downstream consumer in mind but build
nothing of it here (see *Out of scope*).

## Goal

`cmd/server/main.go:79` declares `var BuildVersion = "dev"` (overwritten at release link
time via `-ldflags "-X main.BuildVersion=<tag>"`), but the value is consumed only by the
`version` subcommand (`main.go:83-86`) and is **never threaded into the request handler**
nor exposed to any API surface. This plan threads `BuildVersion` into the controller
handler via a **constructor argument** (immutable, preferred over a setter) and exposes
it on the **authenticated operator `/session` response** (`sessionResponseJSON`) so the
panel can read the controller's own build version after login. The non-release default
stays `"dev"`. No anonymous surface beyond `/api/health` is touched; no FE display, no
rollout/guard logic, no agent-side change (all plan-8).

## Implementation steps

The threading is: `main.BuildVersion` → `NewControllerHandler(... , buildVersion)` → a new
`ControllerHandler.version` field → `HandleSession` writes it into `sessionResponseJSON`.

### Step 1 — Add the `version` field to `ControllerHandler`

`internal/api/routes_controller.go`, in the struct (`:18-73`), add after `releaseClient`
(`:72`), before the closing brace (`:73`):

```go
	// version is the controller's build version (cmd/server main.BuildVersion, stamped at
	// release link time via -ldflags -X; "dev" for a non-release build). It is threaded in
	// at construction (immutable for the handler's lifetime, never a runtime setter) and
	// surfaced ONLY on the AUTHENTICATED operator /session response — never on an anonymous
	// surface beyond /api/health. The panel reads it to display the controller version and
	// (plan-8) to default the agent-rollout target and reject a target newer than the
	// controller. Empty is normalized to "dev" by the constructor.
	version string
```

### Step 2 — Extend the constructor signature

`internal/api/routes_controller.go:80`, change the signature and body. New signature:

```go
func NewControllerHandler(store controller.Store, tenant controller.TenantID, operatorTokenHash, operatorName, buildVersion string) *ControllerHandler {
	if operatorName == "" {
		operatorName = DefaultOperatorName
	}
	if buildVersion == "" {
		buildVersion = "dev"
	}
	return &ControllerHandler{
		store:             store,
		tenant:            tenant,
		operatorTokenHash: operatorTokenHash,
		operatorName:      operatorName,
		version:           buildVersion,
		// ... existing fields unchanged (loginLimiter, enrollLimiter, sessionTTL,
		//     pollDeadline, secureCookie, releaseClient) ...
	}
}
```

Notes:
- `buildVersion` is appended **last** in the arg list to keep the existing positional args
  stable for readers, and is normalized to `"dev"` when empty (mirrors the `main.go:79`
  default; protects test callers and any future caller that passes `""`).
- Update the doc comment above the ctor (`routes_controller.go:75-79`) to mention
  `buildVersion` ("…and the controller build version surfaced on the operator session").

### Step 3 — Update the single production call site

`cmd/server/main.go:217`, pass `BuildVersion` through:

```go
	ch := api.NewControllerHandler(store, controller.TenantID(tenant), opTokenHash, api.DefaultOperatorName, BuildVersion)
```

`BuildVersion` is the package-level var in `cmd/server` (`main.go:79`) — directly in scope
here. No other production call site exists (grep confirms `NewControllerHandler` is called
only at `main.go:217`); **every test caller** in `internal/api/*_test.go` that constructs a
handler directly must add the new trailing arg — see Step 6.

### Step 4 — Add `controller_version` to the session DTO

`internal/api/cookie_session.go`, in `sessionResponseJSON` (`:134-138`), add a field. Keep
it `omitempty` so the wire shape is additive (an old panel ignores an unknown field; a
future caller that constructs the struct without a version emits nothing rather than an
empty `"controller_version":""`):

```go
type sessionResponseJSON struct {
	Operator  string `json:"operator"`
	ExpiresAt string `json:"expires_at"`
	CSRFToken string `json:"csrf_token"`
	// ControllerVersion is the controller's build version (handler.version; "dev" for a
	// non-release build). Surfaced ONLY here on the AUTHENTICATED operator session — never
	// on /api/health or any anonymous surface. The panel displays it and (plan-8) uses it
	// to default the agent-rollout target and reject a target newer than the controller.
	ControllerVersion string `json:"controller_version,omitempty"`
}
```

The `controller_version` snake_case name matches the convention the FE `SessionResponseJSON`
(`controllerClient.ts:606-610`) will mirror in plan-8.

### Step 5 — Write the version into the session response

`internal/api/cookie_session.go:167`, change the `writeJSON` call in `HandleSession` to
populate the new field:

```go
	writeJSON(w, http.StatusOK, sessionResponseJSON{
```

`HandleSession` is already gated by `operatorAuth` (`routes_controller.go:198,207`: `op :=

version is only emitted to an authenticated operator. No new route is needed — the outline's
"dedicated `/version` route" is the explicit *stop-loss* fallback (see *Out of scope*), not
the chosen path.

## Tests produced by this plan

### `internal/api/session_version_test.go` (new)

A focused handler test for the version-on-session contract.

- **Cases:**
  1. `TestSessionReportsControllerVersion` — construct a `ControllerHandler` with
     `buildVersion = "v2.0.0-beta.9"`, log in (reuse `loginForCookies` from
     `cookie_session_test.go:45-77`), GET `/session` with the session cookie, decode the
     body, assert `controller_version == "v2.0.0-beta.9"`.
  2. `TestSessionVersionDefaultsToDev` — construct the handler with `buildVersion = ""`,
     authenticate, GET `/session`, assert `controller_version == "dev"` (the ctor's
     empty→"dev" normalization, mirroring the `main.go:79` default).
  3. (Same test, negative leak check) — assert `GET /api/health` does **not** carry the
     controller version (parse its body / assert the field absent), proving the value stays
     on the authenticated surface only.
- **Lifetime:** **subject-scoped** (`tests/`-equivalent: lives in `internal/api/` alongside
  its sibling tests; tracked as subject-scoped per the outline's per-plan test discipline).
- **Guards:** the controller build version is threaded into the handler and surfaced on the
  **authenticated** operator session (and only there), with `"dev"` as the unstamped default.
- **Retirement trigger:** the version-on-session contract is removed or relocated (e.g. a
  later plan moves it to a dedicated `/version` route and drops it from `/session`), OR this
  subject closes and the contract is folded into a perpetual session-contract test.
- **Retirement destination:** `tests/legacy/agent-feedback-and-version-aware-rollout-2026_06_22/`
  at subject closure (per the close-phase test-migration ritual), unless promoted — see below.

> **Promotion note:** the "version never leaks on `/api/health`" assertion is a small
> security-floor guard. If, at subject closure, the team wants a standing regression floor
> against re-leaking the version on an anonymous surface, *that single assertion* may be
> promoted into a **perpetual** test (e.g. an existing `airgap_routes`/`handler_test`
> health-surface test). Default for the file as a whole remains subject-scoped.
