# plan-6 — Apple-minimal polish + i18n + a11y

See `outline.md` (M6). One session (can ship per-section). **Risk: low.** Prereq: plan-1..4 (5 optional).

## Goal
Apply the refined **Apple-minimal** design language across the shell, complete EN/ZH localization of every new surface, and make the shell accessible. This is the "make it beautiful, not coarse" pass.

(none — presentation.)

## Read first
- `src/index.css` (the `@theme` + accent/hairline/surface vars from plan-1).
- `src/components/shell/*` (Shell/Sidebar/Topbar/UserMenu/ThemeToggle), the aside components (plan-3), section pages (plan-2).
- `src/i18n.ts` (`txt()` + the `STRINGS` table); grep new hardcoded strings.

## Implementation steps
1. **Design language:** generous padding/air; **hairline** dividers (`border-[color:var(--hairline)]`) instead of bordered gray boxes; soft elevation; `rounded-2xl`; one restrained accent (active nav, primary button, focus ring); muted neutral surfaces.
2. **Vibrancy:** translucent `backdrop-blur` sidebar/topbar with a solid fallback gated by the `.no-translucency` class (from plan-4); honor `prefers-reduced-transparency` if set.
3. **Typography:** SF-style stack (`-apple-system, system-ui, "SF Pro Text", Inter`); hierarchy by weight/size/color.
4. **Motion:** quiet 150–200ms transitions on hover / sidebar fold / aside open; honor `prefers-reduced-motion`.
5. **Light + dark parity:** verify every surface in both themes (graphite-dark / teal-light accent); fix any contrast gaps.
6. **i18n sweep:** EN/ZH for every new label (nav, aside, user menu, settings, appearance, empty/loading/error states); zero hardcoded strings (grep audit).
7. **a11y:** sidebar keyboard navigation + focus states; `aria-current` on active nav; landmark roles; visible focus rings.
8. **States:** empty/loading/error for Overview, Fleet, Deploy, Security.

## Tests produced by this plan
- Gate `npm run lint && npm run build`. Manual: both themes; reduced-motion/transparency; keyboard-only sidebar nav; i18n toggle shows no English leakage in ZH.

- [ ] Sidebar keyboard-navigable; focus visible; `aria-current` set.
- [ ] Empty/loading/error states present; lint/build green.
