# plan-4 — Persisted mode + non-secret caches + appearance

See `outline.md` (M4). One session. **Risk: low.** Prereq: plan-1,2 (3 helpful).

## Goal
Persist Local/Controller **Mode** (Settings → Mode); paint fleet/settings instantly from **non-secret** caches; wire the **Appearance** settings (theme client; translucency server-read + local fallback); move **Refresh** to the bottom of the Connection section as a submit-style action. **No token persisted.**

## Reads from docs/spec
`docs/spec/controller/controller-api.md` (settings shape), `docs/spec/controller/bootstrap.md` (`ControllerSettings`).

## Read first
- `src/stores/controllerStore.ts` — state shape (~47–140), persist `partialize` (~691–704), `refresh()` (~245–269), the memory-only `sessionToken`/`operatorToken` comment (~58–62).
- `src/components/deploy/DeployPanel.tsx` — the Local/Controller `useState` (~52) + connection form + Refresh button.
- `src/api/controllerClient.ts` — `getSettings`, `ControllerSettings` type.
- `src/theme/ThemeProvider.tsx` (from plan-1).

## Implementation steps
1. **Lift Mode into the store:** add `mode: 'local'|'controller'` to `controllerStore` + to its `partialize` (additive). Drive nav visibility + landing from it. Surface in **Settings → Mode**. Remove the `DeployPanel` `useState`.
2. **Non-secret caches in `partialize`:** add `nodes`, `settings`, `lastSyncedAt`, `sidebarCollapsed`, `theme` (the last two may already live in plan-1's store — keep one home). Caches carry **no key material** (node IDs/status/generations/timestamps only). Add a comment: caches are **paint-only / advisory** — never gate a Deploy decision; gating uses freshly-`refresh()`ed state.
3. **NEVER persist** `sessionToken`/`operatorToken`/CSRF — extend the existing do-not-persist comment block.
4. **Appearance settings UI** (Settings → Appearance): theme (System/Light/Dark — mirrors the top-right toggle, client-persisted); **translucency** toggle bound to `ControllerSettings.Translucency` via `GET/POST /settings` in controller mode, with a **localStorage fallback** in Local mode. Apply via a `.no-translucency` html class. (Backend field added in plan-5; until then, read-only/local-only — or sequence the settings struct field early; see note.)
5. **Refresh → bottom of Connection** as a full-width submit-style button (label "Connect / Refresh"); keep the loading/last-synced affordance.
6. i18n parity.

## Note on translucency ordering
The `ControllerSettings.Translucency` backend field is specified in plan-5's change surface but is independent of the cookie work; it can be split into a tiny precursor PR if plan-4 needs the server field before plan-5 lands. Local-mode translucency works client-side regardless.
