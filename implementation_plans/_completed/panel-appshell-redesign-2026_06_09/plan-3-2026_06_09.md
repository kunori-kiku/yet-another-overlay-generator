# plan-3 — Right aside + canvas toolbar

See `outline.md` (M3). One session (may split → `plan-3.5`). **Risk: med** (`RightPanel` is 1157 ln).

## Goal
Demote node manipulation from the default full scene to a **selection-driven right aside** inside `/design`. Canvas is full-width when nothing is selected; selecting a node/edge/domain opens its editor in the aside. The docked `LeftPanel` collapses into a canvas toolbar. **Zero topology-editing feature may be lost.**

## Reads from docs/spec
(none — frontend; but cross-check `docs/spec/data-model/{node,edge,domain}.md` to confirm no editor field is dropped.)

## Read first
- `src/components/layout/RightPanel.tsx` — **READ WHOLE FILE (1157 ln)**; catalog every editor block: Actions (~169–206), Domain (~211–293), Node (~297–700: incl. public_endpoints, extra_prefixes, SSH details), Edge (~703–1020: type/endpoint/transport/role/priority/backup/pinned-alloc), Compile-result preview (~1023–1146).

- `src/components/deploy/NodeRegistry.tsx` (for the Fleet node-detail pattern).

## Implementation steps
1. **Aside container** in the `/design` element: renders the selected entity's editor; hidden (canvas full-width) when nothing selected. Drive from existing `selectedNodeId/selectedEdgeId/selectedDomainId`.
2. **Extract editors** from `RightPanel` into focused components (e.g. `aside/DomainEditor`, `NodeEditor`, `EdgeEditor`) — **move, don't rewrite**; preserve every field, handler, validation, and the `reconcileEdgeEndpoints` coupling. Do a field-by-field parity checklist vs the catalog from Read-first.
3. **Canvas toolbar** replacing the docked LeftPanel: `[+ Domain] [+ Node]` (open create forms in the aside or a popover); domain/node **lists** move to a collapsible aside tab or the toolbar.
4. **Compile-result preview** → stable home under `/deploy` (no longer hidden behind "nothing selected").
5. **Validation** → a Design footer strip or an aside "Validation" tab (from `BottomBar`).
6. **Fleet node detail** → `/fleet/nodes/:id` (status/actions) as a right-side detail, mirroring the aside pattern.
7. i18n parity for any moved/added labels.

## Tests produced by this plan
- Gate `npm run lint && npm run build`. **Manual parity pass:** every editor field + action from the RightPanel catalog is reachable and functional in the aside.
