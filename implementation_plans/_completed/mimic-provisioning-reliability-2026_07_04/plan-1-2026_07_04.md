# plan-1 — Two-package catalog + install both debs + robust fallback

**Prerequisites:** none (first plan). Branch `feat/mimic-two-package-install` off `main`.

**Goal:** each `<codename>-<arch>` catalog row pins both the `mimic` and `mimic-dkms` `.deb`; the
install fetches, verifies (`sha256sum -c`), and installs BOTH together so `Depends: mimic-modules`
resolves; and every failure in the provisioning block degrades per the link's `mimic_fallback` policy
instead of a `set -e` abort. Land the model, artifacts.json, both install-script blocks, the TS

## Implementation steps

### 1. Model — the new mimic pin type (P1: do NOT touch `model.Artifact`)
In `internal/model/artifact.go`, add beside `Artifact`:
```go
// MimicDebPin is ONE catalog row's pinned pair: the userspace `mimic` .deb (Asset/SHA256, the
// legacy fields — an old catalog with only these still binds) and the companion `mimic-dkms` .deb
// (DKMSAsset/DKMSSHA256), which Provides the `mimic-modules` the mimic pkg Depends on. Flat +
// additive so encoding/json round-trips an old {asset,sha256}-only catalog into a zero DKMS pair.
// NOT a reuse of Artifact — that type is shared by agent-bins + release-pins and must not gain a
// mimic-only field.
type MimicDebPin struct {
    Asset      string `json:"asset"`
    SHA256     string `json:"sha256"`
    DKMSAsset  string `json:"dkms_asset,omitempty"`
    DKMSSHA256 string `json:"dkms_sha256,omitempty"`
}
```
Change **only the mimic map's value type** everywhere it is the mimic map (leave `AgentBins`
`map[string]Artifact` alone):
- `render.FetchSettings.MimicDebs` (`render.go:251`) → `map[string]model.MimicDebPin`.
- `controller.ControllerSettings.MimicDebs` (`store.go:387`) → same; update `Clone` (`:430-436`) —
  the value is still a value type, so the shallow map copy stays correct; reword the doc comment
  (`:426-427`) and the `memstore.go:867,872` comments.
- `artifactsMimic.Debs` (`artifacts_json.go:33`) → same (this defines the emitted `.mimic.debs[k]`
  JSON: now `{asset,sha256,dkms_asset,dkms_sha256}`).
- `BuildFetchSettings` (`compile.go:157`) is a transparent copy — no code change beyond the type.
- `LoadFetchSettings` (`fetchsettings_env.go:58`) transparent — no code change beyond the type.

### 3. Install script — fetch+install BOTH debs, robust fallback (node block, `script.go:504-581`)
Restructure the provisioning block so it CANNOT `set -e`-abort under policy=udp. Introduce a function
that returns non-zero on any failure and a single policy-aware call site. Sketch (keep the existing
comments/idiom; `shq`/`.MimicFallbackUDP`/breadcrumb constants unchanged):
```bash
_mimic_provision_debs() {   # returns non-zero on ANY failure; no bare exit inside
    command -v mimic >/dev/null 2>&1 && return 0
    [ -n "$YAOG_PM" ] && _pm_install mimic || true
    command -v mimic >/dev/null 2>&1 && return 0
    [ "$YAOG_PM" = "apt-get" ] && command -v dpkg >/dev/null 2>&1 || return 1
    [ -f "$SCRIPT_DIR/artifacts.json" ] || return 1
    # ... codename/arch/key, jq install, read release_url + asset + sha + dkms_asset + dkms_sha ...
    _mimic_dkms_asset="$(jq -r --arg k "$_mimic_key" '.mimic.debs[$k].dkms_asset // ""' "$SCRIPT_DIR/artifacts.json")"
    _mimic_dkms_sha="$(jq -r --arg k "$_mimic_key" '.mimic.debs[$k].dkms_sha256 // ""' "$SCRIPT_DIR/artifacts.json")"
    [ -n "$_mimic_rel" ] && [ -n "$_mimic_asset" ] && [ -n "$_mimic_sha" ] || return 1
    # DKMS toolchain (best-effort; already present at :558-561)
    _pm_install "linux-headers-$(uname -r)" || _pm_install "linux-headers-$(dpkg --print-architecture 2>/dev/null)" || _pm_install linux-headers-generic || true
    _pm_install dkms || true; _pm_install gcc || true
    _dl() { curl -fL --retry 3 --proto '=https,http' "${GH_PROXY}${_mimic_rel}/$1" -o "$2" && echo "$3  $2" | sha256sum -c -; }
    _mimic_deb="$(mktemp --suffix=.deb)"; _dl "$_mimic_asset" "$_mimic_deb" "$_mimic_sha" || { rm -f "$_mimic_deb"; return 1; }
    _install_list="$_mimic_deb"
    if [ -n "$_mimic_dkms_asset" ] && [ -n "$_mimic_dkms_sha" ]; then
        _dkms_deb="$(mktemp --suffix=.deb)"; _dl "$_mimic_dkms_asset" "$_dkms_deb" "$_mimic_dkms_sha" || { rm -f "$_mimic_deb" "$_dkms_deb"; return 1; }
        _install_list="$_install_list $_dkms_deb"
    fi
    # install both together so `mimic Depends mimic-modules` resolves from the local dkms .deb
    DEBIAN_FRONTEND=noninteractive apt-get install -y $_install_list; _rc=$?
    rm -f $_install_list
    [ "$_rc" -eq 0 ] && command -v mimic >/dev/null 2>&1
}
if ! _mimic_provision_debs; then
{{ if .MimicFallbackUDP -}}
    echo "WARNING: mimic could not be provisioned; falling back to plain UDP (policy=udp)" >&2
    _mimic_breadcrumb {{ shq .MimicBreadcrumb.InstallFailed }}
    _MIMIC_SKIP=1
{{ else -}}
    echo "ERROR: mimic could not be provisioned and this link's policy is fail-closed (no fallback)" >&2
    _mimic_breadcrumb {{ shq .MimicBreadcrumb.InstallFailed }}
    exit 1
{{ end -}}
fi
```
Notes: the function form makes `set -e` inert *inside* it (a failing command becomes the return
value at the `if !` boundary — the standard bash pattern already used for wg-quick at `:879-884`).
Keep the `/sys/fs/bpf` sanity warn after. Mirror the SAME restructure in the **client block**
(`script.go:1489-1542`, guarding the `apt-get install` at `:1524`).
