# Plan 1 — Phase 0: sign the existing bundle path

Outline: [outline.md](outline.md) · Prerequisites: none · Branch: stacked on the plan-PR branch
`plan/controller-panel-2026-06`.

## Goal

Add end-to-end **authenticity** to the per-node bundle, standalone — no DB, no agent, no custody
change. Define a canonical bundle serialization, Ed25519-sign it (stdlib), and have `install.sh`
verify the signature against a Go-emitted pinned public key before the existing `sha256sum -c`. This
strengthens the air-gap path *and* lays the signing primitive every later phase reuses. Ships as a
normal release (candidate v1.4.0).

Reads from specs: artifacts/export-bundle, artifacts/install-script, security/security

## Implementation steps

1. **NEW `internal/bundlesig/bundlesig.go`** (leaf, stdlib only). Canonical serialization:
   `Canonicalize(files map[string]string) []byte` → emit `"<sha256hex>  <path>\n"` for every bundle
   file, **sorted by path**, LF-only, trailing newline — i.e. a deterministic normalization of the
   existing `checksums.sha256` content (it already lists every file incl. install.sh). Plus
   `Sign(canonical []byte, priv ed25519.PrivateKey) []byte` and
   `Verify(canonical, sig []byte, pub ed25519.PublicKey) bool`. Document: signs the canonical bundle
   digest, never `compiler.go computeChecksum`.
2. **Export path** (`export.go`): after writing `checksums.sha256`, compute `Canonicalize` over the
   same file set, sign with an operator-configured Ed25519 key (loaded from a path/env; if absent,
   skip signing — opt-in stop-loss), and write `bundle.sig` (base64) + embed the verifying public key
   into the install script as a Go-emitted constant.
3. **install.sh** (`script.go`): before the existing `sha256sum -c`, if a signature + pinned pubkey
   are present, verify `bundle.sig` over a freshly-recomputed canonical digest using a tiny embedded
   verifier (openssl if available, else a vendored check) and abort on mismatch. Keep hash-only
   behavior when unsigned (opt-in). Mirror the `EXPECTED_PAYLOAD_SHA256` failure style.
4. **Self-extracting installer** (`handler.go`): extend so the wrapper also carries+verifies the
   signature alongside `EXPECTED_PAYLOAD_SHA256`.
5. **Spec** (`export-bundle.md`, `install-script.md`, `security.md`): extend the integrity-anchor
   section from tamper-detection to authenticity; document the canonical serialization + signing key.
6. **NEW spec** `docs/spec/controller/signing.md`: the canonical-serialization contract + Ed25519
   signing/verify (referenced by all later phases).

## Tests produced by this plan

- `internal/bundlesig/bundlesig_test.go`
  - **Lifetime:** perpetual (`internal/bundlesig`).
  - **Guards:** canonical serialization is deterministic + order-independent over the file map; sign
    → verify round-trips; tamper (one byte) fails verify; signing is over canonical bytes, not
    `computeChecksum`.
  - **Retirement:** never (pins the authenticity primitive every phase depends on).
- `internal/artifacts/export_signing_test.go`
  - **Lifetime:** subject-scoped.
  - **Guards:** a signed export produces `bundle.sig` verifying against the embedded pubkey; an
    unsigned export (no key) still produces a valid hash-only bundle (back-compat).
  - **Retirement:** at subject close (fold into export tests if cheap).
- `internal/renderer/script_signature_test.go`
  - **Lifetime:** subject-scoped.
  - **Guards:** install.sh contains the signature-verify step + pinned pubkey constant; verify
    precedes `sha256sum -c`.
