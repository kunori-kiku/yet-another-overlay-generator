# Plan 2 — Phase 1a: split-render + custody mode

Outline: [outline.md](outline.md) · Prerequisites: Plan 1 merged (signing primitive) · Branch stacked
on Plan 1's.

## Goal

Let the compiler render a whole fleet **from public keys only**, emitting a `PRIVATEKEY_PLACEHOLDER`
in each node's own `[Interface]` so the controller never needs (or sees) a private key. The agent
(Plan 3) splices its locally-held key into the placeholder. The air-gap path is untouched and
byte-identical. This is the code half of the zero-knowledge custody decision.

Reads from specs: compiler/peer-derivation, data-model/node, security/security, controller/signing

## Implementation steps

> **Refinement (verified during Phase 1a investigation, 2026-06-08).** The custody decision lives in
> `GenerateKeys` ALONE, not `render.All`, and needs **no `wireguard.go` change**. Verified facts:
> the compiler never parses the private key — `DerivePeers` uses only `PublicKey`
> (`peers.go:487/580/668`) and `DeriveClientConfigs` copies `clientKey.PrivateKey` verbatim
> (`peers.go:1072`); the only private-key parse anywhere is `render.go:48` (AirGap case-a); and the
> validators (`schema.go`/`semantic.go`/`nat.go`) do **no** key-format validation. Therefore a
> placeholder string placed in the `keys` map propagates untouched: `GenerateKeys → keys map →
> compiler → both WG renderers (`PrivateKey = {{ .PrivateKey }}` verbatim, per-peer at
> `wireguard.go:57`, client wg0 at `:94` fed from `ClientPeerInfo.PrivateKey`) → install.sh`. So the
> placeholder is the sentinel string itself; no renderer/compiler/validator edits.

1. **Custody type + sentinel** (`render.go`): add `type KeyCustody int` with `AirGap` (iota, default)
   and `AgentHeld`, and `const PrivateKeyPlaceholder = "PRIVATEKEY_PLACEHOLDER"` (intentionally NOT
   valid base64, so no key parser mistakes it for a key).
2. **`GenerateKeys(topo, custody KeyCustody)`** — explicit required parameter (self-documenting; the
   security-sensitive dimension is visible at every call site). Update the existing call sites to pass
   `render.AirGap` (`handler.go:139/187/252`, `cmd/compiler/main.go:47`, `equivalence_test.go`, the
   `renderAll` helper in `signing_test.go`) — behaviorally and byte-for-byte unchanged.
   - **AirGap**: today's switch verbatim (case-a derive+reuse `render.go:44`; case-b public-only =
     hard error `:55`; case-c generate `:60`). Byte-identical.
   - **AgentHeld**: never emit a real private key. Use `node.WireGuardPublicKey`; if absent but a
     private key is present (e.g. an air-gap topology imported into the controller), derive the public
     half and **discard** the private one; if neither is present, hard error (the agent must register
     a public key first). Emit `KeyPair{PrivateKey: PrivateKeyPlaceholder, PublicKey: pub}` and zero
     `node.WireGuardPrivateKey` so the controller's topology never carries a private key.
3. **(removed — no `wireguard.go` change; the placeholder propagates verbatim, see Refinement.)**
4. **No-private-key CI guard** (`internal/render/custody_guard_test.go`): render a multi-node
   public-only topology in `AgentHeld` mode and assert **no** emitted `PrivateKey =` line carries a
   value other than `PrivateKeyPlaceholder` (and that the placeholder does not parse as a wgtypes
   key). Perpetual guard against a split-render regression reintroducing the key vault.
5. **Byte-diff parity** (`internal/render/custody_diff_test.go`): with a topology carrying FULL
   keypairs, the `AgentHeld` output equals the `AirGap` output line-for-line *except* each node's own
   `PrivateKey` line (placeholder vs real). Proves the only difference is the intended one.
6. **Spec** (`security.md`, `data-model/node.md`, new `docs/spec/controller/key-custody.md`): document
   AgentHeld custody, the placeholder contract, and that I5's guarantee holds via public-key persistence.
