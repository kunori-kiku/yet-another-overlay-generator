## Goal (milestone)

Turn the static single-tenant pull of Plan 3 into a real controller with durable state, node
identity, and a deploy workflow — still single-tenant (tenant_id wired but constant). Operators get
a Deploy button and per-node status; nodes enroll with a token and authenticate with per-node mTLS.

## Scope (to be decomposed when detailed)

- **Persistence** (`internal/controller`, NEW; Postgres; new deps quarantined here): per-(future-)
  tenant registry — node_id, agent mTLS cert fingerprint, agent WG **public** key, last_seen,
  desired/applied generation; topology JSON **public-keys-only**; staged/current signed bundles;
  **hash-chained append-only audit log**. Single data-access chokepoint with `tenant_id` as a
  mandatory predicate (constant for now, real in Plan 5).
- **Enrollment:** single-use, short-TTL, node-scoped tokens; agent submits CSR + WG public key with
  **proof-of-possession** (signs a challenge); controller burns the token atomically, issues a
  per-node mTLS client cert, records the public key, logs for operator review. Pending→approved
  gate (admin approval or pre-created slot).
- **Agent identity:** per-node **mTLS** on all (outbound) agent traffic; **anti-rollback** enforced;
  short-lived certs (no CRL/OCSP — overlay-eviction is the fast revoke).
- **Controller transport:** real **TLS 1.3** on the controller (not punted to a proxy); reuse
  `server.go` wrap/recover/timeout scaffolding; routes beside the untouched air-gap endpoints.
- **Deploy state:** long-poll generation channel (`/poll`), `/config` returns only the caller's node,
  `/report`; operator Compile/Stage + Deploy.
- **Frontend:** Deploy button, per-node status (enrolled/online/current generation/drift),
  enrollment UX. Reuse the existing React/Zustand designer.

## Hazards / gates (milestone)

Tenant-scope chokepoint correctness (even single-tenant, build the predicate right); enrollment-token
theft (single-use + TTL + scope + audit); never store a private key; keep-last-good/fail-closed
preserved. Gate: enrollment→mTLS→pull→apply integration test + real-host smoke. Stop-loss: if
Postgres/mTLS scope balloons, split persistence and enrollment into separate sub-plans.

## Out of scope (deferred to Plan 5)

Multi-tenant isolation enforcement, per-tenant KMS signing, hardware-signed membership, stage→promote
approval, OIDC operator auth, supply-chain hardening.

## Decomposition (detailed 2026-06-08, after the refine-investigation + user decisions)

Phase 2 is split into four stacked sub-plans, each a reviewed PR through the pipeline:

- **[plan-4.1](plan-4.1-2026_06_08.md) — persistence foundation.** `internal/controller` Store
  interface + stdlib MemStore/FileStore + hash-chained audit + tenant chokepoint. CI-green, no HTTP.
- **plan-4.2 — enrollment + per-node mTLS cert issuance.** Single-use TTL tokens (hashed, atomic
  burn), CSR + WG-pubkey + proof-of-possession, dev self-signed controller-CA (stdlib `crypto/x509`),
  pending→approved (auto-approve in v1; manual approval is a 4.4 UI affordance). Agent `enroll`
  subcommand. Extends the Store with token methods.
- **plan-4.3 — controller HTTP surface + TLS/mTLS + deploy state + agent integration.**
  `/api/v1/controller/*` (`/enroll`, `/config` caller-only, `/poll` long-poll, `/report`, operator
  `/update-topology` + `/stage` + `/promote`); TLS 1.3 + mTLS on an env-gated controller mode of
  `cmd/server`; tenant+node from the mTLS cert CN at a middleware chokepoint; **generation-based
  anti-rollback** (the generation is bound into the signed bundle, closing the Phase-1b unsigned-
  manifest gap); agent mTLS pull/poll/report. In-process httptest+TLS+dev-CA+MemStore integration
  test (mock the install.sh apply).
- **plan-4.4 — frontend controller panel.** New `/controller` route + separate `controllerStore`;
  NodeRegistry (per-node enrolled/online/current/drift + per-edge readiness), EnrollmentFlow modal,
  Deploy (stage→promote) + audit display; `controllerApiClient`; EN/ZH i18n. Does NOT pollute the
  `Node` topology type. CI: tsc + vite build + eslint.

### Adopted decisions (from the investigation; defaults taken, documented for course-correction)

- **Binary layout:** extend `cmd/server` (env-gated controller mode), same mux, `/api/v1/controller/*`
  prefix, air-gap routes untouched. No separate `cmd/controller`.
- **Persistence:** stdlib-only for v1 (MemStore for CI + FileStore for durability). **No new go.mod
  dep** — Go isn't installed locally so a Postgres driver can't be vendored (`go.sum` can't be
  generated; CI would break). The Postgres adapter is a documented future `Store` impl; the interface
  makes the swap drop-in (file-backed is appropriate for single-tenant v1; Postgres is a
  scale/multi-tenant concern, Plan 5).
- **Tenant scope:** structural — `TenantID` is a mandatory predicate on every Store method; derived
  from the principal (mTLS cert CN) at the middleware, never a URL/body field. Constant
  (`YAOG_TENANT_ID`) in single-tenant v1. Perpetual cross-tenant CI gate.
- **Enrollment:** single-use, short-TTL, node-scoped tokens, stored **hashed**, atomically burned;
  CSR + WG pubkey + proof-of-possession; dev self-signed controller-CA (ephemeral key bounds breach
  surface); mTLS cert CN = `tenant:node`. Auto-approve in v1; manual approval is a 4.4 affordance.
- **mTLS / TLS:** `http.Server.TLSConfig` MinVersion TLS 1.3; `VerifyClientCertIfGiven` (operator
  routes may be certless+OIDC later, agent routes require a client cert). Revocation = overlay
  eviction (remove from topology + re-deploy), no CRL/OCSP.
- **Deploy state:** monotonic **generation** counter; `/poll` is a blocking long-poll (~55 s) on
  `Store.WaitForGeneration`; `/config` returns only the caller's bundle; `/report` records
  applied-generation/health. `/stage` is all-or-nothing.
- **Partial-fleet deploy (user decision 2026-06-08): RENDER WHAT'S READY.** Compile renders each
  node's bundle including only peers whose public keys are registered; an edge whose far end is not
  enrolled is omitted and activates on a later deploy. Implemented controller-side by filtering the
  topology to the **enrolled subgraph** before calling the (unchanged) `Compile`/`render.All`; the
  frontend shows per-node/per-edge readiness + desired-vs-applied drift. Not all-or-nothing.
- **Audit:** SHA-256 sequential hash chain; tamper-**evident** for operational visibility only
  (honest scope — an attacker with store write access can recompute the chain; cryptographic
  anti-tamper is Plan 5), matching the agent anti-rollback honesty framing.
- **Frontend:** new `/controller` route, separate `controllerStore`, long-poll status, modal
  enrollment, one-click promote + loud audit display, `/api/v1/controller/*` versioned, `Node`
  topology type unchanged.
