## Prerequisites

## Goal
The deferred self-update **retry** path must verify keystone membership (`VerifyMembership`) before any
binary swap, exactly as the apply path does — so a keystone-ON node cannot be made to swap its agent
binary by any party that can merely serve it a `VerifyBundle`-passing bundle.

## Implementation steps

### Step 1 — a membership-verifying fetch for the retry
In `cmd/agent/main.go`, add a `membershipVerifiedFetch` closure beside `verifiedFetch` (`:523`) that
does `client.Fetch` → `VerifyBundle(files, pinned)` → `VerifyMembership(files, MembershipConfig{NodeID:
o.nodeID, OperatorCredPEM: operatorCred, OperatorCredAlg: o.operatorCredAlg, OperatorRPID:
o.operatorRPID, OperatorOrigin: o.operatorOrigin}, <prevEpoch>)`, returning `files` only when BOTH
pass. Keep the existing bundle-only `verifiedFetch` for the post-swap health-gate
(`ReconcileSelfUpdatePromote`, `:537`) — that runs after a swap already gated by the apply/retry
membership check, so bundle-only reachability is correct there.

`prevEpoch`: load `State` (`LoadState(o.stateDir)`) and pass `st.MembershipEpoch` (mirror the apply
path's `prev.MembershipEpoch`) so the retry honors the anti-rollback floor. When keystone is OFF
(`operatorCred` empty) `VerifyMembership` is a no-op → identical to today (dev unaffected).

### Step 2 — wire it into the retry
Pass `membershipVerifiedFetch` (not `verifiedFetch`) to `RetryDeferredSelfUpdate` at `main.go:655`.
Prefer NOT widening `RetryDeferredSelfUpdate`'s signature (it already takes a `verifiedFetch func()`) —
the membership binding lives entirely in the closure, keeping the retry function agnostic and the fix
localized. Update the `selfupdate_retry.go:16-19` doc + the false `:11-12` comment to state the retry
now verifies membership (keystone binding restored).

### Step 3 — spec
Update `docs/spec/controller/agent-selfupdate.md`'s deferred-retry section: the retry re-verifies
BOTH tier-1 integrity AND keystone membership before a swap (matches the apply path); keystone-OFF is
a no-op.
