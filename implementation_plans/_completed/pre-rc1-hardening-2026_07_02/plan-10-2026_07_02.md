## Implementation steps

### Step 1 — new agent Sampler
Add `internal/agent/telemetry_resource.go`: `resourceSampler` implementing `Sampler`, returning
`(nil, map[string]any{"resource": …})` from PURE Go file reads (no shell) — `/proc/loadavg`
(load1/5/15) and `/proc/meminfo` (`MemTotal`/`MemAvailable`). Best-effort: read error → `(nil,nil)`
(mirror `wireguardPeersSampler.Sample`). Inject the read fn (like `wgShowFn`) for testability. Define
`const resourceMetricKey = "resource"`.

### Step 2 — register it
Append `resourceSampler{}` to the slice in `BuildTelemetry` (`telemetry.go:105-108`). That is the ENTIRE
backend change (transport/wire/persist/serve already carry arbitrary metric keys).

### Step 3 — panel surfacing (mirror `wireguard_peers`)
- `controllerClient.ts:291-293`: add `resource?: ResourceMetricWire` to the telemetry type; add a
  `mapResource(n.telemetry?.resource)` projection in `mapNode` beside `mapWireGuardPeers`, defensive
  against garbled input (like `:704-713`).
- `types/controller.ts`: add `resource?: NodeResource` to `ControllerNode`.
- Pure math in `frontend/src/lib/resource.ts` (mirror `wgPeers.ts`) for formatting (load coloring,
  mem %), rendered by a small `ResourcePanel.tsx` (mirror `WireGuardPeersPanel.tsx`) added to
  `FleetNodeDetailPage.tsx:113`.

### Step 4 — custody (live-only)
Add `resource: undefined` to `stripLiveTelemetry` (`custody.ts:91`) and do NOT touch the `leakOracle`
allowlist. Rationale: same honesty argument as `wireguardPeers` (a load average frozen at persist time
is stale) + minimal persisted surface. Host load/mem carry no endpoint/IP/key material, so the
`leakOracle` e2e is the backstop (reds if `resource` ever reaches localStorage un-stripped).
