# plan-8 — `agent kit verify`

> See `outline.md` (§7 plan-8). Gives a manual (hand-installed) node the same fail-closed
> VerifyBundle+VerifyMembership gate a managed agent gets, to run before `sudo bash install.sh`.

## Prerequisites
Ships toward rc.1. File-disjoint (CLI-only).

## Goal
A `kit verify` subcommand that verifies an already-downloaded manual-node bundle (tier-1 signature +
per-file checksums + keystone membership) so a breached controller's tampered `install.sh` is caught for
manual nodes as it is for managed ones.

## Implementation steps

### Step 1 — sub-dispatch
In `main.go` `case "kit":`, if `args[0]=="verify"` → `runKitVerify(args[1:])`, else `runKit(args)`.
Update `usage`.

### Step 2 — `runKitVerify` flags
- `--bundle PATH` (required) — a directory of extracted files OR a `.zip`.
- `--node-id ID` (required) — for `VerifyMembership` self-membership + digest binding.
- `--operator-cred FILE|-` + `--operator-cred-alg` (optional; keystone gate; reuse `readOperatorCred`),
  `--operator-rpid`/`--operator-origin` (WebAuthn).
- `--pubkey PEM` (optional pinned signing key; absent → TOFU exactly like the agent, `verify.go:170-176`).

### Step 3 — loader + verify
New small helper (since `DirSource` needs a nodeID subdir): build `map[string][]byte` keyed by
slash-relative path — for a dir, `filepath.Walk` + `filepath.ToSlash` (mirror `readBundleDir`); for a
`.zip`, `archive/zip`. Then `VerifyBundle(files, pinned)` → on success `VerifyMembership(files,
MembershipConfig{NodeID, OperatorCredPEM, OperatorCredAlg, OperatorRPID, OperatorOrigin}, prevEpoch=0)`.
`prevEpoch=0` is correct (a pre-install human check is stateless; 0 accepts any `epoch≥0`,
`verify.go:416`) — document that kit verify does not track rollback.

### Step 4 — output + exit codes
Human summary to STDERR (signed bool, files N, membership verified, epoch, operator-cred fingerprint via
`CredFingerprintShort`); machine JSON to STDOUT (`{ok, signed, file_count, epoch, node_is_member}`).
Exit **0** verified; **1** verification failed (surface the primitive's message — it already names the
tamper/rotation remedy); **2** usage/IO error.

## Definition of done
- [ ] `kit verify` runs `VerifyBundle` + `VerifyMembership` over a dir or zip; correct exit codes.
- [ ] Zero-knowledge preserved (reads public material only; no controller contact; no private key).
- [ ] No validator/apierr code; no drift/i18n/golden.
- [ ] Tests green; independent review → fixed → re-reviewed → CI green.

## Out of scope
- Fetching from the controller (verifies an already-downloaded bundle — matches the zero-contact manual
  model).
- Persisting epoch/state; changing `VerifyBundle`/`VerifyMembership` (reuse verbatim).
