# plan-4 — WireGuard public-key format validation

> See `outline.md` (§2 Principle "Generated scripts/configs run as root; no unescaped user text",
> §7 plan-4). Closes config-injection into other nodes' root-parsed `wg` confs.

## Prerequisites
Ships toward rc.1 (after beta.17). File-disjoint from plans 5–10.

## Goal
Reject a malformed WireGuard public key at every ingress so a key string (rendered raw via non-escaping
`text/template` into other nodes' `[Peer]` blocks) can never carry a newline/injection.

`model-validation, render-keys, controller-stage-promote`

## Implementation steps

### Step 1 — a shared validator helper
Add `validWGPublicKey(s string) bool` = base64-std decode ok AND `len==32` (reuse `decodeKey` semantics;
Go: in `internal/validator` or a shared leaf; TS: `atob` + `Uint8Array` length 32, base64-std not url).

### Step 2 — schema (Go + TS), only-when-non-empty
In `validateNodesSchema` (Go `schema.go` + TS `validator.ts`), when `node.wireguard_public_key != ""`
and `!validWGPublicKey(...)`, add an error with a NEW code `CodeNodeWGPublicKeyInvalid =
"validation_node_wg_public_key_invalid"`. Only-when-non-empty (managed nodes legitimately carry an
empty topology key). Add the const + registry message (Go) and the Code + registry + check (TS).

### Step 3 — controller ingress (network path; schema never sees these)
- `validateManualNodes` (`compile.go:511`): add a format branch reusing the EXISTING
  `apierr.CodeManualNodeInvalid` (`.With("detail", "WireGuard public key is not valid base64/32-byte
  Curve25519")`). No new apierr code.
- `Enroll` + `Rekey` (`enrollment.go`): validate `req.WGPublicKey` at the top of `Enroll` (BEFORE
  `ConsumeEnrollmentToken` — a pure format check, so validating pre-burn is safe + better UX) and
  `newPubKey` at the top of `Rekey`. Return a new sentinel `ErrInvalidWGKey`; map it in the enroll/rekey
  HTTP handlers (mirror the existing `ErrDuplicateWGKey` mapping) to the EXISTING
  `apierr.CodeReqFieldInvalid` with `field=wireguard_public_key`. No new apierr code.

### Step 4 — CI-gate ritual (the new validator code)
`go test ./internal/conformance/ -run TestDrift -update` (regen `drift_manifest.json` — the new code
lands in `validator_codes`); add `error.validation_node_wg_public_key_invalid` to `en.ts` + `zh.ts`;
confirm the TS `validator.ts` carries the code + message + check. Run
`go test ./internal/conformance/ ./internal/localcompile/ -update` and confirm the ONLY diff is the
manifest (existing goldens carry valid derived keys → no byte churn). Add one fail-fixture (a manual
node with a bad pubkey) under `internal/conformance/testdata/fail/` (intentional new golden).

### Verify + commit

```
$HOME/.local/go/bin/go test -race ./internal/validator/ ./internal/controller/ ./internal/conformance/
cd frontend && npm run build && npx vitest run
```
Commit `fix(validation): reject malformed WireGuard public keys at every ingress (plan-4)` +
