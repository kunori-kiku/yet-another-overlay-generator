# plan-9 — reliable, distinguishable "update failed" state

> See `outline.md` (§7 plan-9). Investigation reshaped this: a DURABLE `selfupdate:Abandoned` condition
> already flows end-to-end. The real gaps are (1) `Abandoned` and `Blocked` are both `Warn`
> (indistinguishable), (2) the failure reason is dropped, (3) the panel's best-effort caveat is now
> dishonest. This is a small precise fix riding the existing channel — NOT a new channel.

## Prerequisites
Build AFTER plan-5 (shares `/telemetry` + conditions). File-adjacent to plan-10 (both touch the panel +
telemetry) but disjoint fields.

## Goal
The panel's "update failed" signal is authoritative and distinguishable from the transient "Blocked",
carrying the curated failure reason, via the existing durable `selfupdate` condition — zero new
apierr/validator codes.

## Implementation steps (Option A — chosen; see rejected alternatives)

### Step 1 — elevate the coarse status
`conditions_selfupdate.go:84`: change `Abandoned` from `model.ConditionStatusWarn` to
`model.ConditionStatusError` (a closed-enum value that already exists — zero new code). `Blocked`
(`:94`) STAYS `Warn` (it is the transient one). This is the positive distinguishing signal at the
status level.

### Step 2 — carry the curated reason durably
Add `AbandonedReason string \`json:"abandoned_reason,omitempty"\`` to `State` (`state.go`, beside
`:76`). Set it in `rollbackAndAbandon` (`selfupdate.go:427`) from the internal `reason` mapped through a
curator (reuse `classifySelfUpdateBlock`-style curation so raw stderr never leaks — the curation
invariant). Preserve it wherever `AbandonedAgentVersion` is preserved (`agent.go:357,:400`); clear it in
`FinalizeSelfUpdate` beside `:386`. Feed it into the `Abandoned` message in `selfUpdateCondition`
(`:85`), falling back to the current generic line when empty (legacy state).

### Step 3 — honest panel caveat
`UpdateStatusChip.tsx:48-51`: gate the `failedBestEffort` suffix on the ABSENCE of a structured
`selfupdate` condition (`su` is already computed at `:46`) — show it only on the legacy `lastHealth`
path; for a structured `Abandoned` (now `status:"error"`) show the curated message alone. No
`updateStatus.ts` change (it already maps `Abandoned→failed` by reason regardless of status).

### Rejected alternatives (record in the PR)
- **B — a dedicated wire field** (`Node.LastUpdateOutcome`): redundant with the durable condition; adds
  operator-wire surface, a `leakOracle` allowlist decision, and `fe_field_lists` drift churn. Rejected.
- **C — a new `ConditionType`**: `ConditionTypeSelfUpdate` already models the terminal `Abandoned`; a
  second type fragments `deriveUpdateState` + the panel derivation for no gain. Rejected.
