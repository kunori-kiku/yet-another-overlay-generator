## Goal
Charset-validate node IDs at the compile root so a malicious ID cannot reach a shell/filesystem sink
(the operator-run deploy script's scp/ssh/rm paths; the manual-bundle Content-Disposition).

## Implementation steps

### Step 1 — node-ID charset at schema (Go + TS), only-when-non-empty
Home = `validateNodesSchema` (the single conformance-mirrored authority; every ID sink is downstream of
a successful compile, so rejecting here gates them all). Add `nodeIDCharset = ^[A-Za-z0-9._-]+$` (no `/`
→ no path traversal; no whitespace/shell metachars) and check it right after `:285` when `node.ID != ""`.
New code `CodeNodeIDIllegalChars = "validation_node_id_illegal_chars"`. Mirror in `validator.ts`
(Code + registry + check).

### Step 2 — CI-gate ritual
`go test ./internal/conformance/ -run TestDrift -update` (new code → `validator_codes`); add
`error.validation_node_id_illegal_chars` to `en.ts`+`zh.ts`; TS mirror carries it. Run
`go test ./internal/conformance/ ./internal/localcompile/ -update`, confirm only the manifest changed
(FE IDs are charset-clean → no golden churn). Add one fail-fixture (intentional new golden).

### Step 3 — trusted-proxy client-IP (ONLY if not done in plan-5 F4)
If plan-5 F4 has NOT landed, implement it here per plan-5 F4's spec (`trustedProxies` field +
`SetTrustedProxies` + `h.clientIP` method + `YAOG_TRUSTED_PROXIES`). Otherwise skip — record in
`outline.md` §6 that F4 shipped in plan-5.
