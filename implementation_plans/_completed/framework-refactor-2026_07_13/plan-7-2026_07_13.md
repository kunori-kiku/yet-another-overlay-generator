# plan-7 — Error framework (handler adapter + sentinel→code table)

**Goal:** Delete the copy-pasted handler preamble and the duplicated `errors.Is` ladder, and make
"which sentinel maps to which HTTP code" auditable in ONE table instead of 28 independent judgments.

**Prerequisites:** plan-2 (handlers in their real homes).

## Design (assessment recap)
40 handlers hand-roll the same method-guard + `identity()` + storage-relay preamble (e.g.
`handler_deploy.go:326-340 == :508-522`); a sentinel→code `errors.Is` ladder is duplicated across 7
handler files (28 branches, no central mapper) — the same `ErrNotFound` maps to six codes ad hoc. The
deep layers barely adopt "coded at source": `controller` has 7 `apierr` refs vs ~92 `fmt.Errorf`; the
agent 0 vs ~143 — so 71 relay sites (`writeCodedOr(..., CodeInternal, ...)`) degrade real errors to a
generic `internal_500` the panel cannot localize. `apierr` also carries 4 redundant internal-500 codes +
a dead `WithStatus`.

## Changes
- New `internal/api/adapter.go` — a typed handler adapter: method-guard + auto-`identity()` + a
  coded-error return signature, so `identity()` becomes structurally UN-skippable (not a per-handler
  convention). Migrate the 40 handlers onto it.
- New `internal/api/errmap.go` — ONE central `sentinel → apierr.Code` table (with HTTP status), replacing
  the 28-branch ladder across the 7 files.
- Migrate the highest-value `controller`/`agent` relay sites onto coded-at-source errors the FE i18n
  joins on; retire the 71 generic-500 relays where a real code exists; remove the 4 redundant
  internal-500 codes + dead `WithStatus`.

## Verify + branch
Full Go suite (`-race`); each migrated handler independently green; the FE i18n error-key join unchanged
(the coded-error envelope is the contract — do not change its shape); `gofmt -l`. Branch
`refactor/plan-7-error-framework`.

## Tests produced
- `adapter` unit test — **perpetual** — asserts the method + `identity()` guard is enforced by the
  adapter (backend-sole-authority).
- `errmap` table test — **perpetual** — every sentinel maps to exactly one code+status.

## Invariants at risk
- **[4] backend sole authority for status/identity:** the `identity()` guard must be structurally enforced
  by the adapter, never skippable; no change to the auth chokepoint.
- The coded-error envelope the FE i18n catalog joins on is a CONTRACT — its shape stays fixed.

## Stop-loss
If a handler cannot adopt the adapter without changing its auth/identity semantics, that handler has a
latent divergence — surface it; do NOT weaken the adapter to fit an outlier.

## Out of scope
The FE regex-based error framework stays semantically distinct (only the Go `apierr` central table +
typed handler land here); the Store core (plan-8).
