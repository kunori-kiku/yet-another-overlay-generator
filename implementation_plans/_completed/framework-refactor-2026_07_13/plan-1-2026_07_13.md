# plan-1 — Pre-WASM contract hardening

**Goal:** Make `localcompile.Compile` the single heal-inclusive compile entry and restore
`internal/model` to a pure leaf, so the contract is CORRECT before Phase 3 makes it the WASM entry
point. Closes a real correctness divergence and an unsigned-file custody surface along the way.

**Prerequisites:** plan-0 (the arch-test catches the `runtimecontract` eviction not re-leaking).

## Design (assessment recap)
- **Heal is silently missing on the airgap path.** `normalize.HealCollidingPins` is a caller-applied
  decorator: `internal/controller/compile.go` heals BEFORE compiling, but `internal/api/handler_airgap.go`
  and the `localcompile` façade do NOT — so a colliding-pin topology compiles DIFFERENTLY on the
  anonymous vs controller path. The 25-topology conformance corpus has no colliding-pin fixture to catch
  it.
- **`internal/model` (a pure zero-import leaf) houses a stateful contract** — `model.Condition` is
  consumed ONLY by the agent/controller/api (never the pure compiler/renderer), a reverse
  type-placement that dodges an import cycle. **NOTE (review):** `MimicOutcome*` + `MimicBreadcrumbPath`
  are DIFFERENT — the pure-core renderer bakes them into `install.sh` (`renderer/script.go:126-134`), so
  they are legitimately model-resident and MUST stay pure-core-reachable; only `Condition` moves.
- **`51820` (the WG listen-port base) is un-hoisted** — duplicated at `peers.go` (≈1129/1361) and
  `semantic.go` (≈272), the exact comment-synced drift hazard `allocconst` exists to kill.
- **`artifacts/export.go` triplicates the bundle file-set** — `BundleFiles` parses the config keys, the
  disk-write loop re-parses them, and `allFiles` is a third hand-built list feeding the manifest; a file
  written-but-unlisted ships UNSIGNED/UNCHECKSUMMED.

## Changes
- **Close the air-gap silent-heal gap by pre-healing in the air-gap HANDLER**
  (`internal/api/handler_airgap.go` `airGapRequest`, shared by `/compile`, `/export`, `/deploy-script`),
  mirroring the controller's `CompileAndStage` pre-heal — so the anonymous routes converge on the SAME
  healed compile the controller produces. **CORRECTION (caught during execution — a Phase-3.5 HIGH-principle
  stop):** the plan's original "fold heal INTO `localcompile`" was WRONG — it would remove a DELIBERATE
  safety net. The shipped C2 design (`docs/spec/rc1/3.4-findings.md`; `internal/edgecase/c2_reenable_test.go`)
  applies heal at the KNOWN entry points (controller stage / update-topology save / canvas load — and now
  the air-gap handler) while `localcompile`'s semantic validator STAYS the LOUD safety net that rejects a
  raw collision reaching the compiler by any other path (`validation_pin_*_duplicate_cross_link` — the
  `heal-collision-reenable` conformance FAIL fixture enforces it). Folding heal into the façade would
  SILENTLY heal a raw collision, defeating that net (the failing fixture surfaced it). So heal lands at the
  handler; **`localcompile`, the goldens, and the fail fixture are byte-UNCHANGED.** No TS change: the
  browser heals at LOAD (store sanitize) before compile, so the TS compile keeps the same loud safety net.
  The regression test locking the air-gap pre-heal is **plan-1.6**.
- New `internal/runtimecontract` — move **ONLY `model.Condition`** there; imported by `controller`+`agent`,
  NEVER by the pure core. **Leave `MimicOutcome*` + `MimicBreadcrumbPath` in `model`** (the renderer
  consumes them — moving them would force a pure→stateful `renderer→runtimecontract` edge and break
  invariant 7). Add `runtimecontract` to plan-0's arch-test FORBIDDEN set so a future pure-core→runtimecontract
  edge is a red build.
- Hoist `51820` → `internal/allocconst.WGListenPortBase`; `peers.go` + `semantic.go` read the constant.
- Single-source `artifacts/export.go` → **DEFERRED to plan-1.5** (split during execution): it is a
  distinct custody concern (bundle file-set robustness, not the heal/purity/port contract hardening) and
  has a rendered-output nuance (`manifest.json`'s `files` order becomes deterministically sorted) that
  deserves its own reviewed byte-diff. It ships as plan-1.5, not dropped.

## Verify + branch
Full Go suite (default + airgap, `-race` for controller/api) + gofmt; **ZERO golden diff** — the heal is
added at the air-gap HANDLER, not the compiler, so the `localcompile`/conformance goldens AND the
`heal-collision-reenable` fail fixture are byte-UNCHANGED (that fixture still asserts `localcompile`'s
LOUD rejection of a raw collision — the safety net stays live). FE build + vitest + lint (TS unchanged).
Branch `refactor/plan-1-contract-hardening`.

## Tests produced
- None new in plan-1 itself: the existing contract + conformance goldens (byte-unchanged) prove the
  heal-fold is a no-op on the collision-free corpus. The colliding-pin regression fixture is **plan-1.6**;
  the `BundleFileSet` unit test is **plan-1.5**.

## Invariants at risk
- **[2] allocation stability / deployable configs + the loud safety net:** heal lands at the air-gap
  HANDLER, not the compiler, so the compiler keeps its loud-fail on a raw collision (the C2 safety net);
  goldens byte-UNCHANGED. Do NOT fold heal into `localcompile` (it would silently accept corruption).
- **[7] quarantine:** evict ONLY `model.Condition`; keeping `MimicOutcome*`/`MimicBreadcrumbPath` in
  `model` is REQUIRED (the renderer consumes them — a move would create the pure→stateful edge invariant 7
  forbids). `runtimecontract` must be in the arch-test forbidden set so no pure-core edge re-forms.
- **[5] custody:** the export single-source (written==listed==signed==checksummed) is handled in
  plan-1.5.

## Stop-loss
If heal is NOT idempotent on the already-healed controller path (re-healing changes bytes), the fold is
unsafe as written — STOP; the controller call site must stop pre-healing in the SAME change, proven by
the fixture.

## Out of scope
Splitting `peers.go`/`semantic.go` (deferred to plan-5); any WASM work (plan-3).
