# plan-2 — Stateful god-file splits (no logic change)

**Goal:** Navigability + merge-conflict relief on the stateful half by splitting the god-files along
seams that ALREADY exist — a pure mechanical motion, all suites green byte-identically. This is the
prerequisite that makes the harder Store core (plan-8) tractable.

**Prerequisites:** plan-0, plan-1.

## Design (assessment recap)
Wherever state or orchestration lives, one unit fuses 6–10 concerns along seams the sweep already
identified: `filestore.go` (2307) folds path-safety + fsync + audit-JSONL + telemetry-overlay into CRUD;
`controller/compile.go` (990) holds compile/stage/promote/preview/manual-node under a header naming only
"compile-and-stage"; `controllerStore.ts` (2086) fuses ~8 concerns in one `create()` literal;
`controllerClient.ts` (1410) spans 7 route domains. None of these needs a logic change — only file moves.

## Changes
- `internal/controller/filestore.go` → `filestore.go` (thin CRUD) + `filestore_io.go` (path-safety +
  fsync primitive) + `filestore_audit.go` (audit JSONL) + `filestore_telemetry.go` (the volatile overlay).
- `internal/controller/compile.go` → `compile_stage.go` / `compile_promote.go` / `compile_subgraph.go` /
  `compile_preview.go` / `compile_manualnode.go` (same package; move functions, no signature change).
- `internal/api` — the handlers plan-0 re-homed by name are now in their real files; finish any residual.
- `frontend/src/stores/controllerStore.ts` → `stores/controller/{auth,fleet,deploy,keystone,settings,sync}.ts`
  slices composed behind ONE `create()`; `stores/controller/persist.ts` keeps the `partialize` allowlist
  as ONE auditable custody gate (do NOT fragment it across slices).
- `frontend/src/api/controllerClient.ts` → `services/{auth,fleet,deploy,keystone,settings,release,telemetry}.ts`
  per-domain modules over a shared transport helper.
- Delete the plan-0 arch-test / eslint allow-list entries these splits resolve.

## Verify + branch
Full Go + FE suites green with **zero behavior change** (this is a move; if any golden or test byte moves,
STOP — a split changed logic). `gofmt -l`; `npm run build` (tsc -b) + vitest + lint. Branch
`refactor/plan-2-godfile-splits` (may be several stacked PRs, one per file, each independently green).

## Tests produced
- A `persist.ts` allowlist unit test asserting the ONE custody gate survives slicing — **perpetual**
  (guards zero-knowledge custody boundary). Otherwise no new tests (a move).

## Invariants at risk
- **[6] zero-knowledge custody:** the FE `partialize` allowlist MUST stay ONE gate — the highest-risk
  detail of this otherwise-mechanical phase.
- No-logic-change is itself the guard: any behavioral delta is a bug.

## Stop-loss
If a "move" cannot be done without touching logic (a hidden shared closure, an ordering dependency),
that seam is not clean — split at a different boundary or surface it; do NOT smuggle a logic change into
a "split" PR.

## Out of scope
The pure-core files `semantic.go` / `peers.go` (deferred to plan-5 — splitting them now incurs the 1:1
TS-mirror parity tax); the Store behavioral-core unification (plan-8); the `Field` primitive (plan-10).
