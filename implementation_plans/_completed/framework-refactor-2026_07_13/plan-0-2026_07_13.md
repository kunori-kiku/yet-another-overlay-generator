# plan-0 — Ratchet + zero-behavior hygiene

**Goal:** Install the enforcement spine (an import-rule ratchet) and clear actively-harmful debt, with
**zero behavior change** — nothing executes differently; every gate stays green day one. This is the
prerequisite that makes every later boundary claim a red build.

**Prerequisites:** none (first plan).

## Design (assessment recap)
The macro boundaries (pure/stateful quarantine, airgap split) are real but enforced only by reviewer
vigilance today. This phase converts the *current* reality into a machine-checked baseline (an
allow-list that may only shrink), and removes debt that is not just cosmetic but actively wrong:
`model/artifact.go` carries a FALSE `package renderer` doc header; `handler_deploy.go` holds 13 `Handle*`
methods, **~9 unrelated to deploy** (only `HandleStage`/`HandleDeployPreview`/`HandleCompilePreview`/
`HandlePromote` are deploy-flow); the vitest `include` is a directory allow-list, so a misplaced
`*.test.ts` is silently never collected (a recurrence recorded in memory).

## Changes
- **Arch-test ratchet** — new `internal/arch/layers_test.go` using `go/packages` to assert the pure core
  (`model,validator,allocator,compiler,renderer,artifacts,localcompile,normalize,render,naming,linkid,allocconst`)
  imports NOTHING stateful (`controller,api,agent`, and — once plan-1 creates it — `runtimecontract`).
  The forward-edge seed allow-list is **EMPTY today** (verified: no pure-core package forward-imports a
  stateful one; `internal/model` imports nothing); a CI check FAILS if it ever GROWS. Note the `model`
  debt is a REVERSE type-placement (stateful types DEFINED in `model`), which an import-direction test
  cannot see — plan-1 fixes it by moving `model.Condition` out, and adding `runtimecontract` to the
  forbidden set keeps it from re-forming; a distinct placement check (a pure leaf must not define
  stateful-only types) is the complete guard.
- **FE import ratchet** — eslint `no-restricted-import` encoding today's FE layering (stores/services/
  components) with the same shrink-only allow-list.
- **Fix lying names** — `internal/api/handler_deploy.go` → move its non-deploy handlers into real files
  (`handler_topology.go` / `handler_stage_promote.go` / `handler_enrollment.go` / `handler_audit.go` /
  `handler_rekey.go`); registration in `routes_controller.go` follows.
- **Delete false / harmful comments** — the `internal/model/artifact.go` "package renderer" header + its
  nonexistent-cycle reasoning; the `internal/controller/compile.go` header's stale call-chain narration.
  (Both are Go doc/header comments — NOT rendered output — so zero golden diff.) **The `sysctl.go` CJK
  comment fragments are DEFERRED to plan-6:** they are in the RENDERED sysctl output (golden-frozen in
  BOTH corpora) and have a TS `sysctl.ts` mirror, so cleaning them is a cross-language change + a
  both-goldens regen — which plan-6's Go-only renderer phase (post-TS-deletion) makes cheap and
  single-golden; they are inert `/etc/sysctl.d` comments (no host behavior), so no urgency, and keeping
  them out preserves plan-0's zero-golden-diff character.
- **`lib/deployMode.ts`** — one validated descriptor collapsing `VITE_LOCAL_ONLY` +
  `VITE_YAOG_LOCAL_ENGINE` into a single source of truth for engine/mode selection. (Correction: there is
  NO dead `CompileResponse` mirror — `CompileResponse` is defined once in `types/topology.ts` and is LIVE
  on both engine paths; the only related code is the deliberately-retained `backend` escape-hatch
  `/api/compile` fetch branch, whose removal is **plan-9's** job per invariant [9] — do NOT touch it here.)
- **Test hygiene** — switch `frontend/vitest.config.ts` `include` from the dir-allow-list to a recursive
  `**/*.test.ts` glob. (Correction: the Go golden dirs are NOT duplicated — the INPUT topology corpus is
  ALREADY single-sourced [`conformance/golden_test.go` reads `../localcompile/testdata/contract/topologies`],
  and `contract/golden` (rendered artifacts) vs `conformance/golden` (verdict manifests + a `fail/` subdir)
  are DISTINCT oracles, not two freezes of the same thing; no merge. `conformance`'s `fail/` corpus is
  preserved and re-homed by plan-5.)
