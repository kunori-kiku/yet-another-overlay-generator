## Goal

Add the **fleet-lifecycle scenario layer** of Subject 3 on top of the 3.1 E2E scaffold: a set of Playwright specs (plus the Go agent-fixture invocations they drive) that walk a real-or-mock `yaog-agent` and an operator through the full node lifecycle against the **controller-mode** stack — enroll → poll → report → config, then the operational transitions that have repeatedly bricked live fleets: **rekey** (the two-way matrix the panel actually drives — rekey-all + per-node clear-rekey; there is no third "cancel-rekey" path, only the panel button *label* "Cancel rekey" on a clear-rekey button), **revoke** (the beta.8 S4/S5 fleet-lifecycle delta on top of 3.2's basic touchpoint), **keystone rotation + node reprovision** (pin/TOFU → deploy-with-sign → acked-rotate → node fail-closed-refuses → reprovision → node adopts), and **pin-collision heal-on-load/deploy** (user-visible behavior only). These are smokes, not unit tests: the byte-level invariants are already covered by the Go suites; the gap this plan closes is the cross-stack, real-browser, real-process assertion that each lifecycle transition is correctly wired end to end — the panel surfaces the right state, the agent reacts to the right signal, the controller serves the right bundle, and the fail-closed guards actually fail closed when driven through the deployed product rather than an in-process `httptest.Server`. This plan ships **scenario specs only** (plus TEST-ONLY agent-fixture sibling-file modes); it adds zero bring-up infrastructure and zero production-code behavior.
