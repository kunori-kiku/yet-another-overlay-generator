# plan-14 — Operator-flow smokes  (subject S3, milestone 3.2)

> Part of the pre-rc.1 program — see outline.md for mission, principles, and full sequencing.

## Prerequisites

- **plan-13 (3.1 E2E-harness-foundation) — HARD.** plan-13 owns and provisions the
  Playwright runner + Chromium-in-CI, the bootable controller-mode `cmd/server`/test-mode
  fixture (with a `create-operator` account, cookie config, per-test FileStore reset), a
  seeded enrolled-node store, the single `seed-topology.json`, and any agreed test-only
  source seam. This plan authors `*.spec.ts` files + the shared fixtures module on top of
  it; it does NOT re-solve bring-up. Per the harness-FIRST spine, plan-13 lands before
  plans 14–18.
- **plan-13 cross-plan amendments REQUIRED by this plan (file in plan-13, honor here):**
  1. **Host = `localhost`, never `127.0.0.1`.** plan-13's runtime diagram navigates
     `http://127.0.0.1:<panelPort>/` and defaults `baseURL` to `http://localhost:8080` —
     the two disagree. Because `assertRegistrableRpId` (`frontend/src/lib/webauthn.ts:89`)
     rejects IP-literal RP-IDs, every passkey path REQUIRES the panel base URL be
     `http://localhost:PORT` (OS-assigned `localhost:0` is fine; `127.0.0.1:0` is not).
  2. **Two keystone tenants as a first-class fixture property:** (i) a keystone-OFF
     controller/operator with NO pinned operator credential (so
     `selectServerOperatorPinned()===false` and `getTrustlist` 404→null), and (ii) a
     keystone-ON controller/operator WITH a pinned operator credential matching a fixed
     test keypair pinned into the virtual authenticator (so the deploy-signing end-to-end
     leg can assert the Go verifier ACCEPTS the signature).
- **beta.8 already shipped (reference as DONE, do not rebuild):** the **F1** fix
  (`getTrustlist` through `request()`, `frontend/src/api/controllerClient.ts:971-986`) — this
  suite is its regression gate; and **S4/S5/S6** (enroll identity-hijack guard,
  revoke-resurrection token purge, TTL cap) — the revoke touchpoint asserts only the UI side;
  the Go regression suite owns the server-side anti-resurrection/TTL contract.

## Goal

Stand up the **operator-journey Playwright spec suite** on top of plan-13's E2E harness:
exercise the controller operator's end-to-end flows (login matrix, the full
`design → validate → compile-preview → stage → (keystone sign) → promote → deploy`
journey, export/import round-trip, session-survives-refresh, and logout revocation)
against a real bootable controller-mode backend, and — the load-bearing contribution
beyond "does the button work" — assert the **fleet-secret custody invariants** the
investigation flagged as a *zero-knowledge SECURITY invariant with ZERO automated tests*
today. The reusable `leakOracle` becomes the first runner-level gate on the custody guards,
run at three checkpoints (post-deploy, post-refresh, post-logout). This plan authors spec
files + a single shared fixtures module ONLY; it consumes plan-13's scaffold and never
edits production source beyond a plan-13-approved test seam.

## Implementation steps

> All paths under `frontend/`. New spec files live in `frontend/e2e/` (per plan-13's
> convention); shared helpers live in **one** `frontend/e2e/fixtures/` module, re-exported
> from plan-13's harness and single-sourced with plan-13's `seed-topology.json`. No
> production source is edited except a minimal, plan-13-approved test-only seam.
>
> **Hygiene constraint (HARD, single-source-of-truth — pulls toward plans 1.1/1.2):** specs
> assert ONLY via DOM + network observation + `localStorage` reads. **No spec imports
> `frontend/src/api/controllerClient.ts` (or `controllerStore` internals) directly** — that
> would re-create the report §5 layer leak ("5 deploy components import controllerClient
> directly") at the test layer and break when plan 1.2 decomposes the god-files
> (`controllerStore.ts` 1637 ln, `controllerClient.ts` 1063 ln). To *trigger* an action, drive
> the UI; only if no UI path exists, call a store action exposed via plan-13's test seam —
> never a `controllerClient` function.

**Phase 0 — Primitives (in plan-13's shared `frontend/e2e/fixtures/` module).**

1. `0.1` Add `totpNow(secretBase32, atUnix?)` reproducing `internal/controller/totp.go`:
   base32-decode → HMAC-SHA1 (matching `crypto/sha1`, `totp.go:21`) over the big-endian 30s
   counter (`totpPeriod=30`, `:34`) → RFC-4226 dynamic truncation → 6 digits
   (`totpDigits=6`, `:33`). **Self-test it against a known RFC-6238 vector** so the helper
   cannot silently drift from the Go side. No clock mocking — use real server time; the
   `totpSkewSteps=1` window (`totp.go:36`, applied in the loop at `:89`) absorbs test latency.
2. `0.2` Add `virtualAuthenticator(context)` wrapping CDP `WebAuthn.enable` /
   `addVirtualAuthenticator` (CTAP2, internal transport, `hasResidentKey`+
   `hasUserVerification` true, `isUserVerified` forced true so `userVerification:'required'`
   at `webauthn.ts:254`/`:390` passes without UI) + `addCredential`/`getCredentials`. Honor
   the `localhost` RP-ID constraint (`webauthn.ts:89`). For the deploy-signing end-to-end
   leg, accept a **fixed test keypair** so plan-13 can pin the matching operator credential
   server-side.
3. `0.3` Add `leakOracle` = `readPersisted(page)` (JSON.parse `localStorage` for
   `topology-storage`/`controller-storage`/`ui-storage`) + `assertNoFleetSecrets(stores,
   sentinels)` doing THREE checks:
   - **Structural** — in controller mode after deploy/logout, `topology-storage.state` must
     match the `partialize` server-held branch (`topologyStore.ts:868-875`):
     `project===defaultProject`, `domains===makeDefaultDomains()`-shaped, `nodes:[]`,
     `edges:[]`, `allocSchemaVersion:0`, `canvasFromServer:true`.
   - **Value** — none of `FIXTURE_SENTINELS` (public IPs / `user@host` / WG endpoint
     strings) and no key shapes (`-----BEGIN`, base64 WG-key shape) appear anywhere in the
     three serialized stores.
   - **Custody + persist-allowlist** — `controller-storage.state` keys are a subset of the
     verified allowlist (`controllerStore.ts:1622-1633`:
     `baseURL`/`pathPrefix`/`agentBaseURL`/`operatorCredentialId`/`operatorCredentialAlg`/
     `operatorRpId`/`operatorPublicKeyPEM`/`mode`/`nodes`/`settings`/`lastSyncedAt`); it
     **never** contains `sessionToken`/`csrfToken`/`operatorToken`; and **each persisted
     `controller-storage.nodes` entry carries ONLY `ControllerNode` non-secret fields**
     (`controller.ts:12`: `nodeId`/`status`/`hasWGPublicKey`(boolean, not bytes)/
     `desiredGeneration`/`appliedGeneration`/`lastChecksum`/`lastHealth`/`agentVersion`/
     `lastSeen`/`enrolledAt`/`rekeyRequested`/`inRollout`) — explicitly **forbid**
     `endpoint`/`public_endpoints`/`private*`/`preshared*`/WG-key-byte fields, while
     **allowing** `nodeId`/`status`/timestamps/generations. Document that `nodeId` is an
     allowed value.
4. `0.4` Define `FIXTURE_SENTINELS` (the seed topology's public IPs / SSH targets / WG
   endpoints) sourced from plan-13's single `seed-topology.json` — never duplicated — so the
   value-grep is precise and **CANNOT collide with the seeded `nodeId`** (which is often
   hostname-derived).

**Phase 1 — Login matrix (`frontend/e2e/login.spec.ts`).**

5. `1.1` Password happy-path: fill `#login-username` / `#login-password`
   (`LoginPage.tsx:100`/`:119`), submit; assert the `login` success branch
   (`controllerStore.ts:756-769`) → panel renders, LoginPage gone.

   (`LoginPage.tsx:205-209`, `role="alert"`) shows the 401 message and the form stays.

   `controllerStore.ts:741-754`) and the in-test code completes login. Negative: a wrong
   code re-prompts with `totpNotAccepted` (`controllerStore.ts:749`).
8. `1.4` Passkey enroll: from PasskeySettings, run `enrollOperatorCredential`
   (`webauthn.ts:210`) against the virtual authenticator; assert `POST /operator-credential`
   and that `serverOperatorPinned` flips true (field `controllerStore.ts:315`, set from the
   server probe at `:1145`). **Isolation note:** this enrolls a credential on its tenant —
   it MUST NOT run against the keystone-OFF tenant of Phase 3 (use the distinct seeded
   keystone-ON controller, or strict ordering — see Risks).
9. `1.5` Passwordless passkey login: `loginWithPasskey` (`controllerStore.ts:966`) →
   `assertLogin` (`webauthn.ts:347`, raw challenge `:354-355`) → assert end-to-end success +

   unregistered username → decoy `allow_credentials` → `noPasskeyRegistered`
   (`controllerStore.ts:973`, decoy path `:971-975`).
10. `1.6` Password+passkey 2FA leg: password account with a registered passkey → assert the

**Phase 2 — Session lifecycle (`frontend/e2e/session.spec.ts`).**

11. `2.1` Session-survives-refresh: log in (password), `page.reload()`, assert `checkSession`

12. `2.2` Logout revocation: invoke logout (`controllerStore.ts:832`); assert `POST /logout`

13. `2.3` Break-glass-is-not-a-login (light): enter a break-glass token via

**Phase 3 — Deploy journey (`frontend/e2e/deploy.spec.ts`).**

14. `3.1` Build/seed a minimal valid topology in the canvas (plan-13's
    `seed-topology.json`); run validate; open compile preview (the store's compile/preview

15. `3.2` keystone-OFF deploy: **precondition (load-bearing)** — assert
    `selectServerOperatorPinned() === false` (`controllerStore.ts:527-531`,

16. `3.3` keystone-ON deploy (the **F1 regression**): on a tenant WITH a pinned operator

    `sessionToken`); deploy → assert the `getTrustlist` call (`controllerStore.ts:1298`,

    `/trustlist-signature` verify **ACCEPTS** the browser-produced signature (end-to-end);
    for any well-formedness-only leg assert the `SignedTrustList` has six base64url fields,

17. `3.4` Post-deploy leak check: run `leakOracle` — after the first deploy

## Tests produced by this plan

- **`frontend/e2e/fixtures/` shared module** (`totpNow`, `virtualAuthenticator`,
  `leakOracle` = `readPersisted` + `assertNoFleetSecrets`, `FIXTURE_SENTINELS`, + the
  module's own self-tests).
  **Lifetime:** perpetual.
  **Guards:** the zero-knowledge fleet-secret custody invariant (no fleet IPs / SSH targets
  / WG endpoints / key material in any persisted store; `controller-storage` persists only
  the verified allowlist and never `sessionToken`/`csrfToken`/`operatorToken`; each
  `nodes`-cache entry carries only `ControllerNode` non-secret fields) + the in-test RFC-6238
  derivation staying byte-faithful to `internal/controller/totp.go`.
  **Retirement trigger:** the custody model is removed or fully replaced by a different
  persistence mechanism, OR `leakOracle` is superseded by a sibling vitest unit suite that
  the program elects as the canonical custody gate.
  **Retirement destination:** tests/legacy/rc1/operator-flow-fixtures/.
- **`frontend/e2e/login.spec.ts`** (password, password+TOTP, password+passkey 2FA,
  passwordless passkey; + wrong-password, wrong/expired-TOTP, unregistered-passkey
  negatives).
  **Lifetime:** perpetual.
  **Guards:** the operator authentication contract — the `login` `passkey_required`/
  `totp_required`/success branches (`controllerStore.ts:695`/`:741`/`:756`) and the
  passwordless `loginWithPasskey` path stay functional end-to-end (virtual authenticator
  produces assertions the Go controller accepts).
  **Retirement trigger:** the auth model (password/TOTP/passkey) is removed or replaced
  (e.g. OIDC migration).
  **Retirement destination:** tests/legacy/rc1/operator-flow-login/.
- **`frontend/e2e/session.spec.ts`** (session-survives-refresh, logout revocation +
  `leakOracle`, break-glass-is-not-a-login).
  **Lifetime:** perpetual.
  **Guards:** cookie-session restore on refresh gates on a non-empty `csrf_token`
  (`controllerStore.ts:900`); logout revokes the server session AND flushes the server-held
  canvas from memory + `localStorage` (`clearServerCanvasAtGate`, `:876`); break-glass never
  marks a cookie login.
  **Retirement trigger:** the cookie/CSRF session lifecycle is replaced.
  **Retirement destination:** tests/legacy/rc1/operator-flow-session/.
