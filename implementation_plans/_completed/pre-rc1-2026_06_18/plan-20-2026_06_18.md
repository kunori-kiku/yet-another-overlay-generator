## Goal

Charter the FINAL pre-rc.1 adversarial security audit — the gate between "all work done"
and "tag rc.1" — against the system **as it will actually ship after the refactor** (TS
browser compiler, controller-only backend, phone UX, new TS crypto/ZIP deps), not against
the unchanged round-1 tree. This plan authors ONE spec document
(`docs/spec/rc1/plans/4.1-reaudit-charter.md`, replacing its current stub) that defines: (1) the
**surface inventory** — the ORIGINAL 7 surfaces re-verified PLUS 4 NEW surfaces the
refactor created, each with file:line anchors and an exploit-first adversarial question;
(2) the **workflow shape** — finders (fan-out, ≤4 per the MEMORY concurrency cap) →
two-lens adversarial verify (exploit lens + reachability skeptic lens) → synthesize into a
findings JSON in the round-1 `investigation-findings.json` `.security` schema + refuted
list + disposition map + GO/NO-GO verdict; (3) the **baseline disposition map** — the 14
round-1 confirmed findings + the B1–B4 completeness items become a *regression baseline*
to re-verify (not rediscover), with the beta.8-shipped fixes (B1/F1/C1/S4/S5/S6) marked
done; and (4) the **exit bar** that gates the rc.1 tag. **The fix this plan applies:** the
exit bar MUST include `plan-18` (3.6) real-tunnel green-and-required AND `plan-5` (1.5)
conformance green-and-required (the prior stub's bar listed "build/test/conformance/E2E
gates green" but omitted `realtunnel` — flagged in plan-22 / 4.3 Risks R8 / line 112 as a
contradiction to fix here), plus an explicit owner sign-off path for any accepted residual.
A sibling plan (plan-21 / 4.2) *executes* this charter; this plan only writes it.
