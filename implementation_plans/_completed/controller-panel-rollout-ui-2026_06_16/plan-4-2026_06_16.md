# plan-4 — Mimic catalog config card (Settings)

See `outline.md`. Milestone: mimic config UI (symmetry). **Prerequisites:** plan-1 (`done`),
plan-2 (`done`); plan-3 (`done`) recommended (reuse its field/editor patterns).

## Goal
Build `MimicCatalogSettings.tsx` on the Settings page, symmetric to the agent card: configure the mimic
GitHub-`.deb` catalog (version, release base, per-`<codename>-<arch>` `.deb` pins) with a **best-effort**
"Assist from release" button (manual entry is the guaranteed default), custody-perception copy, and EN/ZH
keys.

## Implementation steps

1. **`MimicCatalogSettings.tsx`** mounted in `SettingsPage.tsx` controller-only fragment after the agent
   card. Same parent-loads / keyed-remount controlled-form pattern.

2. **Fields:** `mimic_version` (semver hint); `mimic_release_base` (absolute http(s) URL); a
   **per-`<codename>-<arch>` debs editor** — a dynamic list of rows `{key, asset, sha256}` (add/remove
   rows, since codename-arch combos are operator-defined, unlike the agent's fixed two arches). Surface
   the read-only proxy echo (single-sourced from settings).

3. **Best-effort assist:** an "Assist from release" button that calls `fetchPins({kind:'mimic',
   base: mimic_release_base || undefined, assets: <current rows' {key,asset}>})` to pre-fill the
   `sha256` of rows whose `asset` is filled. Because the mimic release base is **external** and may not
   publish `.sha256` sidecars, treat failures as expected: on a fetch/parse failure show an informational
   "couldn't fetch a sidecar for <asset> — enter the SHA-256 manually" (not a hard error), and leave the
   row for manual entry. Manual entry is always available and is the documented default.

4. **Custody copy** (same as plan-3): the fetched sidecar is convenience-only; trust is the signed
   `artifacts.json` + the install-time `sha256sum -c`. Do not auto-save fetched hashes.

5. **Client-side pre-validation** mirroring `validateMimicCatalog` (incl. debs-require-release-base);
   server authoritative. Local error banner via `localizeError`.

6. **i18n:** `mimicCatalog.*` keys (labels, row add/remove, assist/best-effort states, validation hints)
   in BOTH `en.ts` and `zh.ts`, lockstep.
