# plan-1 — Backend: assisted release-pin-fetch endpoint + `in_rollout` flag

See `outline.md`. Milestone: backend foundation. **Prerequisites:** none (first plan).

## Goal
Add a controller-mode **operator** endpoint that, server-side, fetches per-asset `.sha256` sidecars
through the persisted gh-proxy and returns `renderer.Artifact`-shaped pins — serving both the agent
(per-arch) and mimic (per-`<codename>-<arch>`) assisted-fill flows — and enrich `nodeJSON` with a
server-computed `in_rollout` bool so canary membership is authoritative.
A browser cannot do this (raw GitHub release sidecars send no CORS headers, and the proxy must be applied
server-side), so this backend rail is required before the frontend assist (plan-3/4) and status chip
(plan-5) can exist.

## Implementation steps

1. **New apierr codes** (`internal/apierr/apierr.go`): add `CodeAgentReleaseFetchFailed`
   (`"agent_release_fetch_failed"`, status 502), `CodeAgentReleaseSidecarInvalid`
   (`"agent_release_sidecar_invalid"`, status 502), `CodeAgentReleaseRequestInvalid`
   (`"agent_release_request_invalid"`, status 400). Each code must be declared in BOTH the `const` block
   AND the registry map (English message template + status int) — `apierr.New` panics on an unregistered
   code. **Also add all three to the `allCodes` slice in `internal/apierr/apierr_test.go`** — `TestRegistryBijection`
   asserts an exact bijection and fails otherwise. Add matching EN+ZH `error.agent_release_*` keys to
   BOTH `frontend/src/i18n/messages/en.ts` and `zh.ts` (bijection). All Go message strings English. (Note:
   `internal/apierr` is NOT scanned by `i18n_gate_test`, so English templates here are a discipline point,
   not a test-enforced one.)

2. **Bounded fetch client** (new `internal/api/release_pins.go`): a handler-injected `*http.Client` with
   `Timeout` ~10s, a `CheckRedirect` that refuses any non-`http`/`https` scheme and caps redirect count,
   AND a custom `DialContext` that **rejects loopback / link-local (169.254.0.0/16, fe80::/10) / RFC1918
   / ULA private resolved IPs** — this is the only way to also defeat DNS-rebind (a hostname that
   resolves to `169.254.169.254`/`127.0.0.1` on the dial). Read body through `io.LimitReader(resp.Body,
   512)` (a sidecar is one hex line). Parse the first whitespace-delimited token; validate against
   `sha256HexPattern`; else `CodeAgentReleaseSidecarInvalid`.

3. **Resolve base + proxy** (helper): `base := override or loadSettings().AgentReleaseBaseURL`
   (mimic uses `MimicReleaseBase`). **Version→tag:** when a `version` is supplied AND `base` ends with
   `releases/latest/download`, rewrite to `releases/download/<tag>` where `<tag>` is the version **with a
   leading `v` prepended if absent** (git tags are `v`-prefixed; `semverPattern` accepts the bare form,
   so `2.0.0-beta.3` must become `v2.0.0-beta.3` or it 404s). If `base` does NOT end in
   `releases/latest/download`, use it as-is and IGNORE `version` (custom/mirror), returning a flag in the
   response so the UI can warn. Prepend `loadSettings().GithubProxy` (if set) to form the resolved URL.
   **SSRF — note `validateAbsoluteHTTPURL` is a FORMAT check, not an SSRF control:** run the resolved URL
   through it + the shell-byte reject (mirror `validateMimicCatalog`) BEFORE fetching, and rely on the
   step-2 `DialContext` private-IP reject for the actual egress guard. Validate every asset name against
   `agentBinAssetPattern` (agent) / `debAssetPattern` (mimic) before use. Return the exact resolved URL(s)
   fetched in the response so a 404 cause is visible to the operator.

4. **Handler** `HandleReleasePins` (`POST {operatorBase}release-pins`): body
   `{kind: "agent"|"mimic", version?: string, base?: string, assets: [{key, asset}]}`.
   - `kind=agent`: keys must match `agentBinKeyPattern` (`linux-<arch>`); if `assets` empty, derive the
     two certified arches' asset names (`yaog-agent-<key>`) from the release.yml contract.
   - `kind=mimic`: keys must match `debKeyPattern` (`<codename>-<arch>`); assets are operator-supplied.
   - For each `{key, asset}`: fetch `<resolvedBase>/<asset>.sha256`, parse, collect.
   - Respond `{pins: {key: {asset, sha256}}, base, version, version_applied: bool, proxy_applied: bool,
     resolved: {key: url}}` (pin values exactly `renderer.Artifact` JSON `{asset, sha256}`;
     `version_applied=false` when a custom base ignored `version`; `resolved` carries the exact fetched
     URLs so a 404 cause is visible). Register in `RegisterOperatorRoutes` beside `settings`:
     `mux.HandleFunc(base+"release-pins", op(h.HandleReleasePins))`. POST-only (405 otherwise); the
     `cors(operatorAuth)` wrapper enforces auth + CSRF on the cookie path automatically.

5. **`in_rollout` on nodeJSON** (`internal/api/handler_controller.go`): in `HandleNodes` load settings
   for the tenant and compute `rollout := controller.AgentRolloutNodeIDs(cs, nodes)` ONCE before the node
   loop; in `nodeJSON` add `"in_rollout": rollout[n.NodeID]` (the construction loop var is `n` and the
   map is keyed by `Node.NodeID`, not `ID`). Do NOT add a per-node `target_agent_version` echo — the chip
   single-sources the target from `settings.targetAgentVersion` (decisions log). Keep it cheap (one
   settings load + one membership pass per list call).

6. **Custody comment**: a doc comment on `HandleReleasePins` stating the fetched sidecar is
   convenience-only and trust remains the keystone-signed `artifacts.json` (PRINCIPLES.md:26-38).
