# plan-4 — Recharts reusable `TimeSeriesChart` + node-detail charts + granularity picker

**Goal:** CPU/RAM history as diagrams with user-selectable granularity — via a REUSABLE chart
component ("one day it could carry ping data", owner decision 1).

## Changes
- Dependency: `recharts` (`npm install --legacy-peer-deps`); the npm SCA CI gate covers it. If the
  scan flags it or bundle growth is unacceptable → plan-4.5 (fall back to uPlot; reusability is the
  requirement, not the lib).
- **`frontend/src/components/charts/TimeSeriesChart.tsx` — series-GENERIC:** props =
  `{ series: {key, label, color?: token, unit, data: {t, avg, min?, max?}[] }[], yDomain?, height }`.
  Theme via CSS custom properties ONLY (`var(--content)`, `--hairline`, `--surface`, the
  accent/danger/success tokens — the beta.13 lesson: no hardcoded colors); min/max as a soft band
  around avg when present; gaps render as gaps (recharts `connectNulls={false}`); tooltip + axes
  localized; `data-testid="timeseries-chart"` (+ per-series testids). NOTHING resource-specific in
  this file.
- **Node detail wiring** (`FleetNodeDetailPage.tsx`, beside/under `ResourcePanel`): a history section
  with range presets (1h / 6h / 24h / 7d) × granularity picker (auto / 30s / 5m / 30m / 1h — "auto"
  asks the server's clamped step), fetching the plan-3 endpoint on view/param change (live-only:
  NEVER persisted to localStorage — `stripLiveTelemetry` custody rules unchanged). Charts: CPU %
  (cpu_pct), load (1/5/15 as three series), memory used % (derived). Loading/empty/disabled states
  (history off ⇒ hint linking the fleet setting).
- Fleet-settings card for the plan-2 cap knob if not landed there (one numeric field + validation).
- i18n en+zh for every string; ESLint/tsc clean.

## Tests
- vitest: pure logic ONLY — `src/lib/telemetryHistory.ts` (param-to-query wiring, series shaping,
  gap handling, disabled/empty states) — `src/lib/**` IS in the include globs. NO component vitest
  for TimeSeriesChart (`src/components/**` is not globbed and the runner is `environment: 'node'`
  with no jsdom); the rendered chart is covered by the e2e below.
- Playwright e2e: node detail shows the chart section; granularity change refetches (mock API);
  `data-testid` locators only.
