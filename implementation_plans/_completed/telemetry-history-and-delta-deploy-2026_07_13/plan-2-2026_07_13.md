# plan-2 — controller telemetry-history store (ring + periodic flush + configurable hard cap)

**Goal:** raw resource samples persist server-side, bounded, WITHOUT violating the RecordTelemetry
memory-only DoS invariant (`filestore.go:751-758`: heartbeats never force durable writes).

## Design (decisions 2 + invariants)
- New `internal/controller/telemetry_history.go`: `type telemetryHistory` — per-node ring of
  `resourceSample{TS time.Time; CpuPct *float64; Load1,Load5,Load15 float64; MemTotalKB,MemAvailKB uint64}`.
  Its OWN mutex (never the store-wide `mu`; mirror `telemetryMu` isolation).
- **Append path:** `RecordTelemetry` (FileStore AND MemStore) additionally parses
  `metrics["resource"]` (tolerant: absent/malformed → skip silently) and appends to the ring.
  In-memory only at this point — O(1), no IO.
- **Flush path (FileStore only):** a background flusher (started with the store, stopped on Close)
  writes ring deltas every `flushInterval` (5 min) OR when a node's unflushed count exceeds a
  threshold — append-only JSONL per node at `<data>/telemetry-history/<nodeID>.jsonl` (one compact
  object per line; greppable, versioned by an optional header line). Flush failures are logged +
  retried next tick, NEVER propagated to heartbeats. Best-effort flush on graceful shutdown.
- **Boot:** lazily load a node's tail on first history query (not at startup — avoids boot IO spikes);
  cache in the ring.
- **Hard cap (configurable):** fleet-settings field `telemetry_history_max_samples_per_node`
  (default 20160 ≈ 7 days at 30 s beats; 0 = disable history). Enforced in-ring (ring size) and at
  flush time (rewrite-truncate the JSONL when it exceeds cap × slack — amortized, never per-append).
  Settings surface: the existing fleet-settings API/panel card pattern (mirror how agent/mimic
  config cards store fields); validate ≥ 0 with an upper sanity bound.
- MemStore: ring only (dev parity; queries work, nothing durable).

## Tests
- Ring append/evict; cap enforcement; flush writes deltas + truncates over cap (temp dir);
  malformed metrics skipped; a failing flush never surfaces to RecordTelemetry; lazy-load
  round-trip (write → new store instance → query loads tail); settings validation.
- Concurrency: `-race` append-while-flush.
