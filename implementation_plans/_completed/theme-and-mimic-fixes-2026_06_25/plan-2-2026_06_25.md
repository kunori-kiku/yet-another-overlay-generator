# plan-2 — Mimic `local=` correctness + guards + Go test ladder

**Subject:** theme-and-mimic-fixes-2026_06_25 · **Track:** backend + compiler + TS mirror ·
**Depends on:** none · **Parallel with:** plan-1.

## Goal

Fix the root cause of "mimic filter seems to be using local, and it did not work": the mimic eBPF
filter is pinned to a single egress IP from `ip route get 1.1.1.1` (`internal/renderer/script.go:788`),
written as `filter = local=${MIMIC_EGRESS_IP}:<listenport>` (`script.go:811`), and matched by an
**exact eBPF hash lookup with no fallback**. When that IP isn't the source IP WireGuard actually puts
on the wire for a given peer (multi-homing / secondary or floating IPs / policy routing) — or
literally resolves to `lo`/`127.0.0.1` when `1.1.1.1` is null-routed — the link silently drops to
plain UDP. Fix it at the root, with the same byte-identical change in the Go renderer AND the TS
in-browser compiler, pinned by Go golden tests; the live confirmation is the owner real-host smoke
(plan-3) since a faithful data-plane mimic test is not feasible in this sandbox.

## Step 0 — Confirm upstream mimic filter semantics (de-risks the design)

A faithful smoke is impossible here, so the design MUST be grounded in upstream `hack3ric/mimic`
docs, not a guess. WebFetch the mimic README / `mimic.1` man page and confirm:
1. Does a filter accept a **port-only** or **`0.0.0.0` wildcard-IP** form (would let us match on the
   stable listen port regardless of local source IP)?
2. Is the **`remote=IP:port`** filter form supported, and are **multiple filter lines OR-combined**
   (not AND)?
3. Does `local=`/`remote=` match the on-the-wire 4-tuple post-routing?

Record the findings in the PR description + `mimic.md`. The exact fix shape (Step 2) is chosen from
these answers. If upstream supports a port-only/wildcard match, that is the simplest root fix and
supersedes the per-peer derivation below.

## Step 1 — Reject a loopback/empty egress IP (certain fix, root cause #3)

In BOTH renderers' egress-detection block (`script.go:787-792` and `script.ts:758`), after deriving
`MIMIC_EGRESS_IP`, reject a loopback or empty result instead of writing a guaranteed-dead filter:
- If `MIMIC_EGRESS_IP` is empty OR matches `127.` / `::1` / resolves via `lo`, do NOT write
  `local=127.0.0.1:...`. Instead write a new breadcrumb outcome `egress_ip_unresolved` and apply the
  node's fallback policy exactly like the existing eBPF-gate failure (`script.go:822-831`):
  policy=udp → de-provision + `_MIMIC_SKIP`/UDP fallback; policy=none → ERROR + `exit 1`.
- Add the closed-enum token `MimicOutcomeEgressUnresolved` to `internal/model/artifact.go:46-50` and
  map it in `internal/agent/mimic_condition.go` `classifyMimic` to a `warn` `mimic` Node Condition
  with a curated message ("mimic could not determine a routable egress IP"). Mirror the TS
  classifier (`frontend/src/lib/mimicCond.ts`) + `nodeConditions.conformance.test.ts`.

This alone fixes the literal "using local" (loopback) failure and can never make a working node worse.

## Step 2 — Make the filter route-correct (robust fix, root cause #1)

Implement the shape Step 0 confirms. Preferred design (additive, cannot remove matches a working node
relies on):

**Per-peer `remote=` filters keyed on the known, stable peer endpoint.** For every mimic peer that
has a known endpoint (`edge.EndpointHost`/`PublicEndpoints` → `PeerInfo` already carries the dial
target), emit an additional `filter = remote=<peer_endpoint_ip>:<peer_endpoint_port>` line. The peer
endpoint is route-independent and known at config time, so it matches the outbound dial regardless of
which local source IP the kernel picks for a multi-homed node. Keep the existing
`local=<egress_ip>:<listenport>` line(s) for the inbound/listen direction (belt-and-suspenders;
strictly additive). To do this the renderer needs each mimic peer's `{endpointHost, endpointPort}`
threaded into `InstallScriptConfig` (extend the per-peer collection beside `collectMimicPorts`,
`script.go:1030-1042`) — resolve hostnames to IPs at install time in-script (`getent hosts` / the
existing endpoint), since the eBPF map needs an IP.

If Step 0 instead confirms a **port-only/wildcard** form, prefer that: replace the single guessed
`local=<egress_ip>:<port>` with the port-keyed match (no local-IP guess at all) — the cleanest root
fix. Choose ONE coherent approach; do not ship both a guess and a wildcard.

Document the residual limitation (root cause #4: a peer whose route egresses a NON-default interface
isn't covered because mimic attaches one unit per default-route interface) in `mimic.md` as a known
constraint + future (per-peer egress-interface attach).

## Step 3 — Compile-time validation guard

Extend `internal/validator/semantic.go` (near `validateMimicTransport:498` /
`validateEdgeEndpointConsistency:765`): a mimic (`transport: tcp`) edge whose dial direction requires
a `remote=` match MUST have a resolvable endpoint (declared `EndpointHost` or the target's
`PublicEndpoints`). Emit a **warning** (not a hard error — keep existing topologies compiling) when a
mimic link would render with neither a usable endpoint nor a listen port (it would shape nothing).
Add a structured code if a new one is warranted (reuse an existing mimic/endpoint code if it fits;
follow the `apierr`/validator-code registry discipline — no orphan codes).

## Definition of done

- Loopback/empty egress IP never produces a dead `local=127.0.0.1` filter (handled per policy +
  surfaced as a Node Condition).
- The filter is route-correct per the Step-0-confirmed upstream semantics (per-peer `remote=` or
  port-keyed), additive/safe for currently-working nodes.
- Compile-time guard warns on un-shapeable mimic links.
- Go test ladder green in-sandbox (`go test -race ./internal/...`); TS mirror + `internal/localcompile`
  golden contract green (`go test ./internal/localcompile/` + `npx vitest run`); `go vet` +
  `go vet -tags airgap` clean.
- `mimic.md` updated (new filter semantics, the new outcome, the residual #4 limitation).
- Independent review (4-lens + security: this renders root install-script + eBPF attach) → fixes →
  re-review → CI green (incl. the netns realtunnel job, which does not exercise mimic but must stay
  green) → merged.

## Hazard / stop-loss

This is a data-plane behavior change that cannot be smoke-tested here. If the owner's real-host smoke
(plan-3) shows ANY regression vs. the prior `local=`-only behavior, the additive `remote=` lines and
the per-peer derivation are independently revertable (Step 1 loopback-rejection is safe to keep). Do
NOT promote the beta to Latest until the owner smoke passes (gate in plan-3).
