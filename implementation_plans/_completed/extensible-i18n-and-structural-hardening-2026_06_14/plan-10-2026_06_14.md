## Principle risk (from outline Principles)

- **Key custody (HIGH):** `saveDesign()` MUST `stripPrivateKeys` before `updateTopology` (mirror
  `deploy()` at `controllerStore.ts:847-849`). A save must never put a private key on the server.
- **Mode logic never crosses the boundary (HIGH):** the switch helper and the dirty/save logic must
  re-read live `mode`/`canvasFromServer`; never persist a server-held secret mirror to `localStorage`.
- **Never drop main flow (MEDIUM):** `flushWorkspace`/`exportProject` stay usable by the gate-flush +
  pre-hydration-backup paths — gate at call sites, do not refuse the store actions.

## Implementation steps (each a commit)

### 1. Shared mode-switch helper (T1) — security parity
- Extract the controller→local decision into ONE place (a `controllerStore` action, e.g.
  `switchToLocal()`, or a shared module fn) implementing exactly LoginPage's logic:
  `const serverHeld = topo.canvasFromServer; serverHeld ? flushWorkspace() : purgeModeBoundaryState();
  clearModeNotices(); setMode('local');` Restore `uiStore.translucency` to the local default here (A3).
- Rewire BOTH `LoginPage` and `SettingsPage` to call it. Fork the confirm dialog copy on
  `canvasFromServer` (server-held → "the entire server design incl. fleet addresses/SSH will be cleared
  locally"; local-original → "your graph is kept, secrets purged"). New i18n keys; no positional text.
- Local→controller (`SettingsPage` "switch to controller" + `checkSession`): purge local-only artifacts
  (compileResult/history/validateResult) and force `hydrateFromServer` when the canvas is not
  server-held even if `wasLoggedIn` (close the stale-mirror gap A2).

### 2. `saveDesign()` Save primitive (T2) — store action
- Add `controllerStore.saveDesign()`: `const clean = stripPrivateKeys(getTopology()).topo;
  await updateTopology(cfg, JSON.stringify(clean)); setCanvasFromServer(true); set lastSyncedSnapshot +
  lastSyncedAt`. NO `stage`/`promote`. Surfacing the button is plan-11; this plan ships the action +
  its store state + tests.
- Dirty state: store `lastSyncedSnapshot` (the canonicalized server design captured at hydrate AND after
  save). Add a `isDesignDirty()` selector comparing `canonical(getTopology())` to `lastSyncedSnapshot`
  using the SAME canonicalization `hydrateFromServer` diff uses (`controllerStore.ts:535-539`).

### 3. Client-side conflict warn (T2 / A4)
- At `hydrateFromServer`, record `serverBaseSnapshot` (canonicalized server copy).
- In `saveDesign()`, before `updateTopology`: `const cur = await getTopology(cfg)` and compare canonical
  to `serverBaseSnapshot`; if changed, do NOT overwrite — surface a conflict notice offering
  re-sync-with-backup (reuse `exportProject` like `hydrateFromServer` does). On user confirm-overwrite,
  proceed. No backend version field (D13).

### 4. `clearServerCanvasAtGate` hardening (T2)
- Before flushing a server-held mirror at logout/gate, if `isDesignDirty()` back up via `exportProject`
  (pre-logout backup, mirroring the hydrate-path backup) so unsaved edits aren't silently discarded.

### 5. `deploy()` confirmed-shrink flag fix (T7)
- On the confirmed-shrink success path, `loadTopology(confirming.snapshot, true)` so `canvasFromServer`
  never mislabels a canvas that diverged from what was uploaded.

## Tests produced by this plan

Subject-scoped (frontend has no runner — verification is lint+build + reasoning/manual per
PRINCIPLES.md; add a lightweight store-logic test only if a harness exists). Document each as a
trace/manual check in the PR body:
- switch parity: controller→local with `canvasFromServer` true → `flushWorkspace`; false → purge; copy
  forks; SettingsPage and LoginPage take the identical path.
- save: `saveDesign` calls `update-topology` ONLY (no stage/promote), strips keys, marks server-held,
  refreshes `lastSyncedSnapshot`; `isDesignDirty()` true after edit, false after save.
- conflict: server changed since hydrate → warn+backup, no blind overwrite.
- gate-flush backs up a dirty mirror; deploy confirmed-shrink leaves canvas == uploaded snapshot.
