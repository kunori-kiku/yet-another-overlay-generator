## Implementation steps

### 1. Model (Go + TS + drift mirror)

`internal/model/topology.go` — constants beside `EdgeRolePrimary/Backup` (after `:163`):

```go
// Value constants for Edge.LinkDirection. An empty value is equivalent to EdgeLinkDirectionBoth.
const (
	// EdgeLinkDirectionBoth lets both sides initiate (today's behavior; the default).
	EdgeLinkDirectionBoth = "both"
	// EdgeLinkDirectionForward lets only from→to initiate (dials edge.endpoint_host);
	// the reverse peer keeps AllowedIPs but carries no Endpoint.
	EdgeLinkDirectionForward = "forward"
	// EdgeLinkDirectionReverse lets only to→from initiate (dials from's public endpoint);
	// the forward peer keeps AllowedIPs but carries no Endpoint.
	EdgeLinkDirectionReverse = "reverse"
)
```

Field after `MimicFallback` (`:203`), doc comment following its prose pattern (tri-state, `""`≡both
for back-compat/omitempty, PURE POLICY — gates only which PeerInfo gets a dial Endpoint, NEVER
touches allocation; cite `docs/spec/data-model/edge.md §Link direction`):

```go
LinkDirection string `json:"link_direction,omitempty"`
```

TS: `frontend/src/types/topology.ts` (after `mimic_fallback`, mirroring its comment):
`link_direction?: 'both' | 'forward' | 'reverse';` — and add `'link_direction'` to
`EDGE_OMITEMPTY` (`frontend/src/stores/controllerStore.ts:126`; the drift manifest's
`fe_field_lists` pins this).

### 2. Compiler gates (Go ⇄ TS, byte-exact)

Helper beside `resolveMimicFallback` (`peers.go` ~:199), mirrored in `peers.ts`:

```go
// effectiveLinkDirection resolves an edge's dial-direction policy: "forward"/"reverse" pass
// through; "" / "both" / anything unrecognized floors to EdgeLinkDirectionBoth (defensive — the
// validator rejects unknown values; the compiler never re-errors). PURE, allocation-blind.
func effectiveLinkDirection(edge *model.Edge) string
```

- **Forward endpoint** (`peers.go:756-771` ⇄ `peers.ts:627-637`): wrap the existing
  `edge.EndpointHost != ""` block so it is skipped when
  `effectiveLinkDirection(edge) == model.EdgeLinkDirectionReverse`.
- **Reverse endpoint** (`peers.go:886-898` ⇄ `peers.ts:732-756`): skip the WHOLE resolution
  (explicit-reverse-edge branch AND public-endpoint fallback) when the direction is `forward`.
- Direction reads from the link's driving edge (the Pass-2 loop var == `link.primaryEdge` on the
  producing iteration; the conflict rule guarantees single-edge pairs whenever direction ≠ both).
- **Keepalive derivation untouched** (`peers.go:773-778`, `:862-865` ⇄ `peers.ts:639-646`,
  `:715-718`) — D5. An endpoint-less peer's keepalive is inert until a session exists, then keeps
  the return path alive; renderer already omits both lines when empty/zero.
- **Allocation (Pass 1) untouched** — HIGH invariant.

### 3. Validation (Go ⇄ TS, 6 codes + NAT adjustments)

`internal/validator/code.go` — six consts in the edge block + registry message templates;
`frontend/src/compiler/validator.ts` — Code entries + registry mirrors; `en.ts` + `zh.ts` —
twelve `error.<code>` keys (gated by `TestI18nCatalogSync`/`Parity`):

| Code (`validation_edge_…`) | Stage | Rule |
|---|---|---|
| `link_direction_invalid` | schema (`validateEdgesSchema`, after the role check) | value ∉ {"", both, forward, reverse} |
| `link_direction_conflict` | semantic | direction ≠ both on an enabled primary-class edge whose pair has ANY other enabled primary-class edge (either direction); backup edges exempt (own links) |
| `link_direction_forward_no_endpoint` | semantic | forward + `endpoint_host == ""` (no dialer can exist) |
| `link_direction_reverse_unreachable` | semantic | reverse + NOT (`from.has_public_ip` && `public_endpoints ≥ 1`) — the compiler's exact resolvability condition (`peers.go:895`) |
| `link_direction_reverse_endpoint_ignored` | semantic | reverse + `endpoint_host != ""` (override would be silently unused — require-explicit-host precedent) |
| `link_direction_client_edge` | semantic | direction ≠ both on a client-touching edge |

Semantic rules live in a new `validateLinkDirection(topo, nodeMap, result)` in `semantic.go`
(called beside `validateSinglePrimaryPerPair`), skipping disabled edges like its siblings; the
schema enum check follows the `mimic_fallback` pattern (`schema.go:489-496`). TS mirrors in
`validateEdgesSchema` + the semantic pass.

**NAT adjustments** (`nat.go` ⇄ `validator.ts:1036-1090`):
- Skip the `CodeNATTargetUnreachable` warning when the edge's direction is `reverse` (the forward
  direction is deliberately passive — the warning would be noise for the feature's core use).
- `CodeNATNoOutboundToPublic`: also count an enabled INCOMING `reverse` edge (the node is the `to`
  side and the `from` node is publicly reachable) as an outbound-to-public path.
- D50 dead-link escalation untouched (rules 3/4 preempt every direction-induced case).

### 6. Unit tests

- Go `internal/compiler/peers_endpoint_test.go` — extend the `TestEndpointResolution_*` matrix:
