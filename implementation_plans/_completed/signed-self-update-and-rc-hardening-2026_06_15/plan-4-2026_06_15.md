# plan-4 — agent version reporting + build-version injection + observability (design PR-3, observable half)

See `outline.md`. Milestone: **beta.1**. Prerequisites: plan-2 (struct) + plan-3 (shares
`ControllerSettings`/settings-validation block; lands after plan-3, rebase). **D7 (ldflags injection)** and
**D9 (release.yml agent-SHA emission)** locked.

## Goal
Make every node's running version *observable* (report → store → panel badge) and make `BuildVersion` real
via build-time ldflags — plus have `release.yml` publish per-arch agent binaries + SHA-256 so beta.2's
self-update has something to fetch and pin. **No swap logic, no target/canary settings** (those are beta.2).

## Implementation steps
1. **Build version:** add `var BuildVersion = "dev"` + a `version` subcommand to `cmd/agent/main.go` (and
   `cmd/server`, `cmd/compiler` for parity). Inject via `-X main.BuildVersion=${tag}` — **EXTEND the existing
   `-ldflags "-s -w"` string** (release.yml ~`:178-181`, `Dockerfile`), do NOT replace it (D7). Verify
   `yaog-agent version` prints the tag in a release build. (`Dockerfile` is co-edited with plan-6's HEALTHCHECK
   — disjoint stages, pick an order.)
2. **release.yml SHA emission (D9):** the per-arch `yaog-agent-linux-amd64`/`-arm64` binaries ALREADY
   build+publish today (release.yml ~`:181,207-215,234`) — do NOT re-add the build matrix. Net-new = emit a
   `.sha256` per agent asset (no `sha256` step exists today) so beta.2's `AgentBinSHA256` is populatable;
   confirm asset names match the bootstrap arch table (`handler_bootstrap.go:278-285`). (Closing the
   bootstrap-TOFU *hole* itself is deferred — document in `docs/spec/controller/bootstrap.md`.)
3. **Report wire:** add `agent_version` to `reportRequestWire` (agent fills from `BuildVersion`) and to the
   server `reportRequestJSON`; persist onto `Node` (`store.go:74`) — store-interface touch, both impls.
4. **Panel:** add `agent_version` to `nodeJSON`; render a version line/badge in `NodeRegistry.tsx`; new i18n
   key in `en.ts`+`zh.ts` (**lockstep; plan-5 also touches these — plan-4 lands first**).
5. **Spec:** refresh `docs/spec/api/wire-contract.md` + `http-api.md` for the new `agent_version` field
   (Addition 3, was orphaned). Own the wire-field-parity test here.
6. Commit per sub-step; push; PR.
