# plan-9 ‚Äî agent self-update mechanism + canary-then-fleet (design PR-4, the RISKY CORE)

See `outline.md`. Milestone: **beta.2**. Prerequisites: plan-4 (version reporting must exist so a crash-loop
is fleet-visible) + plan-3 (`artifacts.json` envelope). **D2 (canary-then-fleet)** locked ‚Äî the empty-target
‚áí no-swap contract + the canary subset are the entire risk bound. This is the subject's highest-risk plan
(R1: brick-a-fleet).

## Goal
Let agents self-update their own binary, verified against the signed `artifacts.json` pin, rolled out
canary-subset-first then fleet, with a swap that cannot infinitely crash-loop a node.

## Implementation steps
1. **Settings (D2 canary):** `TargetAgentVersion`, `MinAgentVersion`, `AgentBinSHA256 map[string]Artifact`
   (`linux-<arch>`), and a **canary selector** (node-id list or a percentage) + a "promote canary ‚Üí fleet"
   action; strict validation (D8). Reuse the EXISTING `ControllerSettings.AgentReleaseBaseURL` (`store.go:294`,
   defaulted `settings.go:24`) as `artifacts.json` `agent.release_url` ‚Äî do NOT add a duplicate field (mirror
   plan-3's `FetchSettings.Mimic*` mapping). Fill the `artifacts.json` agent block from these (only for nodes in
   the active rollout set). Empty `TargetAgentVersion` ‚áí no agent block ‚áí no swap (the safety contract).
2. **`internal/agent/selfupdate.go`:** `maybeSelfUpdate(cfg, artifacts.agent)`:
   noop if `desired==running`; **anti-downgrade**: refuse `desired < AgentVersionFloor` and `desired <
   running`; build url `${GH_PROXY}${agent.release_url}/<asset-for-arch>`; download to
   `<dir(os.Executable())>/.yaog-agent.<desired>.partial` (same FS ‚áí atomic rename; cross-fs fallback
   copy+fsync+rename); **verify SHA vs signed `artifacts.json` pin** (refuse on mismatch); `exec(tmp,
   "version")` self-test (refuse if ‚âÝ desired); write `State.PendingUpdate{from,to,attempts:0}` breadcrumb
   (crash-durable); `rename(target‚Üítarget.bak); rename(tmp‚Üítarget); syscall.Exec(target,...)`.
   **Version comparator (S4):** add `golang.org/x/mod/semver` (admissible scoped dep, PRINCIPLES.md:39-42) or a
   documented+tested `Compare`, for the `desired < floor` / `< running` / `< agent.min_version` checks ‚Äî lexical
   compare mis-orders `-beta.10` vs `-beta.2`; treat empty/absent reported version as the MINIMAL sentinel
   (legacy agents are below-floor, Addition 4a). **Arch‚Üíasset (S6):** `key := "linux-" + runtime.GOARCH`,
   reconciled with the bootstrap arch table (`handler_bootstrap.go:278-285`); scope amd64/arm64 (D9), fail-closed
   on others. Always write `agent.min_version` / `AgentMinVersion`, never bare `min_version` (avoids conflation
   with the bundle anti-rollback floor `State.MembershipEpoch`).
3. **Ordering (`min_version`):** if `running < agent.min_version` ‚Üí self-update FORCED before apply (refuse
   to apply an incompatible bundle, report unhealthy); else self-update AFTER a successful apply.
4. **Startup reconcile** (`runControllerMode`, pre-loop): if `PendingUpdate` set ‚Äî `BuildVersion==to` ‚Üí
   health gate (one clean Poll+VerifyBundle) ‚Üí promote (`AgentVersionFloor=to`, rm `.bak`, clear) or
   rollback (`rename(.bak‚Üítarget)`, Exec back); `BuildVersion!=to` ‚Üí `attempts++`; `>N` ‚Üí abandon (stay
   `from`, cooldown, report unhealthy) ‚Äî **bounds the `Restart=always` loop**.
5. **Legacy agents (Addition 4a):** define empty/absent reported version as **below-floor** so legacy
   agents self-update (not frozen out); test it.
6. **PRINCIPLES.md:** amend HIGH tier with the self-update custody invariant (also in outline Principles).
7. **Docs:** create `docs/spec/controller/agent-selfupdate.md` (protocol, floor, canary, min_version policy
   D3, recovery); refresh `wire-contract.md` for any new self-update wire.
8. **Canary UI:** panel surfaces canary set + per-node "update pending/applied/failed" (built on plan-4 badge).
9. Commit per sub-step; push; PR.

## Tests produced
- selfupdate decision table (noop / downgrade-refuse / floor-refuse / hash-mismatch-refuse / self-test-fail /
  happy) ‚Äî **subject-scoped**.
- breadcrumb-reconcile (boot-as-`to`-healthy‚Üípromote, boot-as-`to`-unhealthy‚Üírollback, boot-as-`from`‚Üíretry
  then abandon at cap) + cross-fs fallback ‚Äî **subject-scoped**.
- **Perpetual** custody guard: an agent never execs a self-fetched binary unverified against the signed
  `artifacts.json` pin; never downgrades below floor (pins the new HIGH principle).

## Definition of done
A canary node self-updates + badge flips; a tampered hash is refused (keep-last-good); a crashing new binary
rolls back within the attempt cap (no infinite loop); downgrade refused; legacy empty-version updates;
documented manual recovery exists + tested; PRINCIPLES.md amended; `agent-selfupdate.md` created.

## Out of scope
Closing the bootstrap-TOFU hole (rc.2). Auto (non-canary) fleet rollout.

bootstrap renderer ‚Üí **STOP and draft plan-9.5**.
