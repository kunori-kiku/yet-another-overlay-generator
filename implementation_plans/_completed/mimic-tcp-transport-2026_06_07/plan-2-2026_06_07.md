# Plan 2 — Backend mimic support (Go PR)

Outline: [outline.md](outline.md) · Prerequisites: plan-1 merged · Branch stacked on plan-1.

## Goal

Make `transport: "tcp"` actually wrap the link with mimic: compiler marks mimic interfaces and
lowers their MTU; the install script provisions + configures + starts mimic on the egress NIC;
the validator gates preconditions; the v1.3.0 reserved warning is removed. No new model field
(keyless). Clients included.

Reads from specs: data-model/edge, artifacts/mimic, artifacts/install-script, compiler/validation,
security/security

  `docs/mimic.1.md`, and the packaged example conf (search the repo, e.g. `*/eth0.conf.example`).
  Confirm: config directive syntax, whether `mimic@<iface>` reads `/etc/mimic/<iface>.conf`, filter

- Code: `internal/compiler/peers.go` (PeerInfo, link build, DeriveClientConfigs/ClientPeerInfo),

  RenderInstallScript ~26-40/572-595), `internal/validator/schema.go` transport block,

## Implementation (4 disjoint partitions for the workflow)

(a) **Compiler** — `peers.go`: add `Mimic bool` and per-interface `MTU int` to `PeerInfo`; mark a
   link mimic when its edge `Transport == "tcp"`; set `MTU = (node.MTU>0 ? node.MTU : 1420) − 12`
   for mimic interfaces, else `node.MTU` (0 ⇒ omit, unchanged). Mirror for `ClientPeerInfo`
   (`DeriveClientConfigs`): client wg0 mimic + MTU−12 when its single edge is `tcp`. Expose, per
   node, the set of mimic listen ports (for the renderer's filters) — e.g. via the peers slice
   (each mimic PeerInfo already carries ListenPort) so the renderer derives it without a new type.
(b) **WG renderer** — `wireguard.go`: per-peer config MTU from `peer.MTU` (was `node.MTU`); client
   from `clientInfo.MTU` (already). Non-mimic output byte-identical.
(c) **Install/uninstall renderer** — `script.go`: add `HasMimic bool` + `MimicPorts []int` (or a
   `[]WgIfaceInfo.Mimic` flag) to the config; Phase 1: `ensure_pkg mimic` (apt) / `require_bin mimic`
   (other) + `modprobe mimic` & persist `/etc/modules-load.d/mimic.conf` + a kernel-eBPF check that
   fails clearly; NEW phase BEFORE wg-quick up: detect egress NIC
   (`ip route show default | awk '{print $5; exit}'`), write `/etc/mimic/<egress>.conf` with one
   filter per mimic listen port (exact directive form per mimic docs), `systemctl enable --now
   mimic@<egress>`; uninstall: stop/disable `mimic@<egress>`, remove the conf + modules-load entry,
   detach (rmmod optional/tolerant). All template lines obey existing shell-escaping; egress var
   quoted.
(d) **Validator** — `schema.go`: delete the `tcp` reserved-warning branch (keep the enum check).
   `semantic.go`: error if a `tcp` edge has an endpoint whose platform is not Linux-deployable;
   (no secret validation — keyless). + tests.

  non-mimic topology compile byte-identical (no-drift assertion alongside the perpetual gate).

  ensure_pkg/modprobe mimic, egress detection, `/etc/mimic` config write, `mimic@` enable, and the

- `internal/validator/mimic_test.go` (**subject-scoped**): tcp no longer warns; tcp on a
  non-Linux endpoint errors; tcp between Linux nodes passes.
