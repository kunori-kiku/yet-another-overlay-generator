# plan-4 — Native-XDP pre-deploy capability probe + panel warning

**Prerequisites:** plan-3 merged (deploy-time achieved-mode is the definitive signal; this plan adds
the *pre-deploy* heuristic). Branch `feat/mimic-native-capability-probe` off `main`.

**Goal (owner D-native):** the agent reports each node's native-XDP capability on its heartbeat so
the panel can warn BEFORE `xdp_mode: native` is selected on a NIC that can't do it. Definitive truth
still comes from plan-3's deploy-time condition; this is a best-effort forward signal.

**Design (D7):** heuristic primary (zero new deps, no live-NIC mutation): the agent reads the egress
NIC driver via `ethtool -i <egress>` (or `/sys/class/net/<egress>/device/driver`) + the kernel
version and classifies `supported` (ena/mlx5/i40e/ixgbe/bnxt/nfp/veth/…), `conditional`
(`virtio_net` — native depends on kernel+queue count), `unsupported` (drivers with no native XDP), or
`unknown` (no signal). Report it as an additive field on the existing `/telemetry` heartbeat. **No
cilium/ebpf dependency; no XDP attach in the primary path** (P: minimal deps + live-NIC safety). A
real brief drv-mode attach probe is the **plan-4.5** escalation (owner decision).

## Implementation steps

### 1. Agent probe (`internal/agent`)
Add `probeNativeXDP() -> {capability: "supported"|"conditional"|"unsupported"|"unknown", driver,
kernel}`: detect the egress NIC (default-route dev), read its driver (`ethtool -i` with a
`/sys/class/net/<if>/device/driver` fallback), map driver+kernel → capability via a small curated
table (documented, conservative — unknown driver → `unknown`, virtio_net → `conditional`). Sample it
on the heartbeat cadence (it's cheap + static; sample once and cache, re-probe on NIC change).

### 2. Telemetry transport (additive — don't break the `/telemetry` contract)
Add `native_xdp` (the probe result) to the heartbeat payload as an OPTIONAL field; controller ingest
tolerates its absence (older agents). Store it per-node; serve it to the panel with the rest of the
node's telemetry/conditions. Mirror the payload type in `frontend/src/…` telemetry types.

### 3. Panel surface + warning
- Node row / conditions: show a small per-node "native XDP: supported/conditional/unsupported"
  indicator (`data-testid`), muted when `unknown`.
- `NodeEditor` `xdp_mode` select: when the selected value is `native` AND the node reports
  `unsupported`, show a warning hint ("this node's NIC (`<driver>`) reports no native-XDP support —
  it will auto-downgrade to skb"); `conditional` → an info hint. Never block (advisory only — the
  deploy-time downgrade is the safety net). i18n en+zh.

## Definition of done

when native is chosen on an unsupported NIC; older agents (no field) degrade to `unknown` cleanly.

## Insertion point

## Out of scope
