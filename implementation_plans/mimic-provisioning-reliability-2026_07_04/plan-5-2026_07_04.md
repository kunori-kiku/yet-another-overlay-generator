## Implementation steps

### 1. `docs/spec/artifacts/mimic.md`
- The **two-package** install: upstream ships `mimic` + `mimic-dkms` (Provides `mimic-modules`); the
  catalog pins both per `<codename>-<arch>`; the install fetches+verifies+installs both together.
- The DKMS/headers reality: the module builds against `linux-headers-$(uname -r)`; a stale-kernel
  node whose exact headers left the repo must reboot into the current kernel — until then mimic
  cannot build and (policy=udp) the link runs as plain UDP.
- The **robust fallback** contract: which failures degrade under policy=udp vs. fail-closed, and the
  breadcrumb → Node Condition each produces.
- Native XDP: the deploy-time auto-downgrade (native→skb) + achieved-mode condition, and the
  pre-deploy capability signal (heuristic; virtio → conditional). Update the `xdp_mode` prose.

### 2. Bilingual wiki (`docs/wiki.md` + `docs/wiki-zh.md`, lockstep)
User-facing: when to use `transport: tcp`; the two-package requirement is handled by Discover
(pick both + Assist); "Fall back to UDP" vs "Fail closed" and what each means when a node can't build
the module; the stale-kernel reboot note; native vs skb + the per-node native indicator. Keep en/zh
sentence-for-sentence.

### 3. Behavioral proof
- The new plan-1 conformance golden is the artifacts.json-shape + `_MIMIC_SKIP` proof; cite it.
- Add a focused Go test (if not already in plan-1) asserting the full node install.sh for a
  two-deb+policy=udp catalog contains: two `sha256sum -c`, one `apt-get install … ./mimic.deb
  ./mimic-dkms.deb`, and the `_MIMIC_SKIP=1` degrade path; and for policy=none the breadcrumb-then-exit.
- Realtunnel: a genuine DKMS build in netns is impractical (kernel headers/build). Decide in-plan: if
  a pre-installed-mimic skb-mode netns scenario is cheap, add it as a non-blocking additive-tier
  scenario proving a mimic link forms in skb; otherwise document why realtunnel is not extended here
  and rely on the golden + Go tests. Do NOT fake it.
