# plan-3 — Native-XDP deploy-time auto-downgrade + achieved-mode condition

**Prerequisites:** plan-1 merged (robust provisioning block + breadcrumb plumbing). Branch
`feat/mimic-native-autodowngrade` off `main`.

**Goal:** a failed `xdp_mode: native` attach no longer bricks — the install rewrites the mimic config
to `skb` and retries once; the **achieved** mode (native / skb-downgraded / failed) is recorded via a
new breadcrumb outcome, classified by the agent into the mimic Node Condition and surfaced in the
chip. Both install blocks + TS mirror + goldens.

## Implementation steps

### 1. New breadcrumb outcome(s) (closed enum — coordinate model+agent+script)
`internal/model/artifact.go`: add e.g. `MimicOutcomeNativeDowngraded = "native_downgraded_skb"`
(native requested, attach failed, skb succeeded). Wire it into `MimicBreadcrumbData`
(`script.go:99-123` `newMimicBreadcrumbData`). The existing `MimicOutcomeActive` covers native-ok and
plain-skb-ok; the new outcome is specifically "you asked for native, you got skb".

### 2. Native attach with auto-downgrade (both install blocks)
In the `mimic@<egress>` start region (`script.go:856-871`), when the node's `xdp_mode` is `native`
and the first `systemctl enable --now mimic@…` (native) fails: rewrite the config's `xdp_mode` line
to `skb`, `systemctl reset-failed` + retry once. On skb success → `_mimic_breadcrumb native_downgraded_skb`
(NOT `Active`). On skb also-failing → the existing policy-aware EbpfLoad path (udp: skip+continue;
none: exit). When `xdp_mode` is already `skb`, behavior is unchanged (no downgrade branch renders).
Keep this inside the existing `{{ if eq .MimicXDPMode "native" }}`-gated template region so an
skb node's `install.sh` stays byte-identical (zero churn for the common case). Mirror in the client
block + the TS renderer.

### 3. Agent classify + chip (`internal/agent/conditions*.go`, `MimicConditionChip.tsx`)
- `classifyMimic`: map `native_downgraded_skb` → a distinct Condition reason (e.g.
  `MimicActiveSkbFallback`) with an actionable message ("native XDP unsupported on this NIC — running
  in skb mode"). Keep `active` → the existing healthy reason.
- Chip: render the downgraded state distinctly (a `data-testid`-labeled variant, not a color-only
  cue); i18n keys en+zh.

### 4. Conformance + tests
- Regen goldens if any fixture uses `xdp_mode: native` (add one if none does — a small `native` tcp
  fixture) so the downgrade branch is covered: `go test ./internal/localcompile/ -run
  TestContractGolden -update` + `go test ./internal/conformance/ -update`.
- Go renderer test: a `native` node renders the downgrade-retry + `native_downgraded_skb` breadcrumb;
  an `skb` node renders neither (byte-identical). Agent test: `classifyMimic(native_downgraded_skb)`
  → the skb-fallback condition. vitest: the chip renders the downgraded variant.
