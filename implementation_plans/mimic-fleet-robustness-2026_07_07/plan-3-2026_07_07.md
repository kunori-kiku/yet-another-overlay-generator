# plan-3 — relay-path warning + mimic_fallback split caveat

**Goal:** surface, at design time, that `transport: tcp` (mimic) needs a DIRECT L3/L4 path — a
`relay-path` edge (through an L7/UDP-accelerator relay) can't carry mimic and should be `transport: udp`.
The owner's whole rc.3-soak debugging chased exactly this; a one-line panel warning saves the next person.

Reads from specs: `docs/spec/artifacts/mimic.md`, `docs/spec/data-model/edge.md`.

## Changes
- **Validator (Go `internal/validator` semantic.go ⇄ TS `validator.ts`, byte-mirrored):** a new
  **warning** (NOT a hard error) — `validation_edge_mimic_relay_path` — when an enabled edge has
  `transport == "tcp"` AND `type == "relay-path"` (model topology.go:194 / TS topology.ts:89):
  "mimic (transport=tcp) needs a direct path; a relay-path edge won't carry the fake-TCP through an L7
  relay — use transport: udp". Emit as a WARNING (result.AddWarning, not AddError) so deploy isn't
  blocked. New code in `code.go` + message + the `error.<code>` i18n (en+zh) + drift regen.
- Validator table tests (Go + TS): tcp+relay-path → warning; tcp+direct → none; udp+relay-path → none.
- (Optional, same PR) a `mimic_fallback: udp` **doc/tooltip caveat** noting a unilateral fallback can
  split a link — the clean fix for an unbuildable node is `transport: udp` (both ends), not relying on
  the per-node runtime fallback. Wire a short EdgeEditor hint if it fits cleanly.
