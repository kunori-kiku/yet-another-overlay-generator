## Implementation steps

1. **Canonical name function:** export `SafeInstallerFileName` (and a `SafeNodeDirName` if the
   export dir should sanitize too — per Spec D decision: keep export dirs raw, ZIP installer
   entries sanitized) from ONE home — `internal/renderer/naming.go` (move from deploy.go);
   `createExportZip` uses it for the `.install.sh` entry name (D3/D32).
2. **Uniqueness validation (D5/D13/D14):** new `validateNodeNameCollisions` in semantic.go —
   errors on duplicate raw Name, duplicate `SafeInstallerFileName`, duplicate
   `wgInterfaceName(name)` among any node's peer set. Export `WgInterfaceName` from compiler
   for reuse.
3. **Remote path (D31):** deploy scripts scp to `/tmp/<node.ID>-install.sh` (IDs unique by
   existing validation) — bash and ps1 both.
4. **Frontend interface names (spec-gap):** RightPanel derives interface names from
   `compileResult.wireguard_configs` keys (`nodeID:iface`) instead of reconstructing from the
   name; delete the truncation reimplementation at :622.
5. **Deploy endpoint compiles (D36/D61):** HandleDeployScript runs the same compile pipeline as
   HandleCompile (validation errors → 422) and passes real PeerMap + BabelConfigs to
   RenderDeployScripts; delete the role-only HasBabel fallback (deploy.go:87-89) or keep it
   solely as a nil-safety with domain-aware shouldRunBabel parity.
6. Tests, commit, push, PR #4. Commit template: `fix(naming): <step> — Spec D (D3/D5/D13/D14/D31/D32/D36/D61)`
