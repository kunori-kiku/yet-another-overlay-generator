## Implementation steps

1. **TopologyCanvas onConnect (D1/D8):** stamp `endpoint_host: preferredEndpoint?.host` only;
   DELETE the `endpoint_port: preferredEndpoint?.port` line. Edge type stays 'direct'.
2. **Stale-endpoint reconciliation (D9/D19):** in the store, when a node's public_endpoints are
   updated/removed, find edges with `to_node_id === nodeId && endpoint_host === oldHost` and
   update/clear `endpoint_host` (+ clear `compiled_port`). In RightPanel edge editor, every
   `endpoint_host`/`endpoint_port`/type onChange includes `compiled_port: undefined`.
3. **Reverse-endpoint fallback (UX-2, backend):** in `peers.go` reverse-peer block (379-396),
   when no reverse edge exists OR reverse edge has empty EndpointHost, fall back to
   `fromNode.PublicEndpoints[0].Host` with port = the from-side interface's allocated port
   (alloc.fromPort/toPort per direction) — NOT PublicEndpoints[0].Port (that would recreate D1
   server-side). Gate on `fromNode.Capabilities.HasPublicIP`.
4. **CompiledPort honors override (D51):** in compiler.go write-back, prefer
   `edge.EndpointPort` when >0, else existing alloc lookup.
5. **Endpoint-mismatch warning:** in `validator/semantic.go`, warn when an edge's endpoint_host
   matches none of the target node's current public_endpoints hosts (catches stale snapshots).
6. **Examples (D21–23/D57):** remove every `"endpoint_port"` from all three examples; add one
   commented NAT-override demonstration in nat-hub's README-comment or notes field.
7. **Tests** (see below), commit per logical step, push, `gh pr create` (PR #2).
