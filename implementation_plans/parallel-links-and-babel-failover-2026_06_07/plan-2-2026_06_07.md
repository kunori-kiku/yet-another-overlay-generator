## Implementation steps

1. **Model** — `internal/model/topology.go`: `Role string \`json:"role,omitempty"\`` on Edge
   (values `primary`/`backup`/empty).
2. **linkKey** — `peers.go`: add `linkKey(edge, pairEdges) string` next to `pinKey` implementing
   the canonical-lowest-edge-ID rule. Canonicality is computed over the pair's enabled edges
   AFTER unify-rule grouping.
3. **Link enumeration (Pass 1 collect)** — replace the `seenPinKey` collapse: group enabled edges
   by pinKey; within a group, unify the roleless opposite-direction pair (legacy bidirectional
   draw) into one link whose primaryEdge keeps today's selection rule; every `role=backup` edge
   is its own link; extra roleless same-direction edges remain dropped-with-warning exactly as
   today (D71 re-scope text). One linkEntity per resulting link, keyed by linkKey.
4. **Reserve/gap-fill** — `pinnedAllocations` keyed by linkKey; pins read from each link's own
   edge; gap-fill iterates links sorted by linkKey (single-edge pairs sort identically to today).
5. **Allocations map** — re-key from `from->to` strings to linkKey; Pass-2 emits one PeerInfo per
   link per side (drop the `addedPeers` pair gate); reverse-endpoint resolution resolves the
   reverse edge OF THE SAME LINK (unified pair) — never a parallel link's edge.
6. **Naming** — `naming.go`: `WgInterfaceNameForEdge(remoteName, edgeID string, canonical bool)`
   — canonical → `WgInterfaceName(remoteName)` byte-identical; else the hashed edge-aware form.
   Call sites peers.go:391,456,504 pass canonicality from the link entity.
7. **Cost** — LinkCost resolution: explicit Priority/Weight → else `role == "backup"` → 384 →
   else 0. Constant `backupDefaultLinkCost = 384` with a comment citing babel.md.
8. **Write-back** — `compiler.go:138-184`: look up allocation by the edge's linkKey; stamp each
   parallel edge with ITS OWN six `pinned_*` + `compiled_port`.
9. **Validator mirrors** — semantic.go: pin-dedup linkID := linkKey (687-741, 581); interface
   counting per link not per pair (319-353); per-node interface-name uniqueness (new, mirrors
   naming exactly); role enum + one-primary-per-pair; equal-effective-cost warning for multi-link
   pairs; D71 message re-scoped (suggest `role: backup`); client rule (480-492) untouched.
10. **DeriveClientConfigs** — confirm the node-pair lookup still resolves (clients are
    single-link by validation; switch lookup to linkKey for consistency).
