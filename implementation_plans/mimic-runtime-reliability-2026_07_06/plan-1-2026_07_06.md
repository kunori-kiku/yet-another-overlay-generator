# plan-1 — mimic module build/load verification + honor-policy + lock cleanup + modprobe

**Goal:** make `_mimic_provision` succeed only when mimic can ACTUALLY run — the DKMS kernel module
is built + loadable — and, when it can't, honor the link's `mimic_fallback` policy with a clear
message instead of the exit-22 loop. Plus the orphaned-lock cleanup (the exit-17 secondary) and an
explicit `modprobe` on start. Fleet-critical: this lets stale-kernel nodes self-heal (udp → plain UDP;
none → clean fail-closed with a reboot hint) when the agent self-updates to rc.3.

Reads from specs: `docs/spec/artifacts/mimic.md` (install ladder, UDP fallback, breadcrumbs).

## Changes

### 1. New closed outcome + condition
- `internal/model/artifact.go`: add `MimicOutcomeModuleUnavailable = "module_unavailable"` to the
  MimicOutcome enum + the breadcrumb-data field (`MimicBreadcrumbData.ModuleUnavailable` + the
  `newMimicBreadcrumbData` wiring), mirrored on the TS side wherever the breadcrumb strings live.
- `internal/agent/mimic_condition.go`: `classifyMimic` case → `(mimicReasonModuleUnavailable,
  model.ConditionStatusWarn, "mimic kernel module unavailable for this kernel — reboot into the
  current kernel or set mimic_fallback=udp")`. Warn (the link is down/degraded, surface it).
- i18n `mimicCond.moduleUnavailable` (en + zh). (Regenerate the drift manifest if it enumerates the
  MimicOutcome/condition set; add the i18n key so `TestI18nCatalogSync`/`Parity` pass.)

### 2. `_mimic_provision` real success gate (`internal/renderer/script.go` node + client, + TS mirror)
Replace the binary-only `command -v mimic` (L593 node / mirror in client) with a genuine module gate:
- Install the matching headers non-swallowed where it matters: keep `_pm_install
  "linux-headers-$(uname -r)"`, but capture whether it (or a running-kernel-headers equivalent)
  actually succeeded; a miss is the "stale kernel / headers pruned" signal, not ignored.
- After the two-`.deb` install, ensure the module is built + loadable:
  - if `command -v mimic` is absent → still `install_failed` (unchanged).
  - `modprobe mimic 2>/dev/null` → if it loads (or `dkms status` shows `...$(uname -r)...: installed`
    and modprobe then succeeds), provisioning is genuinely OK.
  - else try to build it explicitly once: `dkms autoinstall -k "$(uname -r)"` (or `dkms install
    mimic/<ver> -k "$(uname -r)"` derived from `dkms status`), then re-`modprobe`.
  - if the module STILL won't load (headers unavailable / build failed / -cloud BTF wall) → return a
    distinct code the caller maps to `module_unavailable` (NOT the generic binary-only success).
- The function must distinguish `install_failed` (deb/download) from `module_unavailable` (deb OK,
  module not usable) so the caller writes the right breadcrumb.

### 3. Caller honors policy on `module_unavailable` (`_mimic_provision` call site, both templates + TS)
Extend the `if ! _mimic_provision; then` block (L595) so a `module_unavailable` return, like
`install_failed`, respects `mimic_fallback`:
- policy=udp → `WARNING: mimic kernel module unavailable; falling back to plain UDP` +
  `_mimic_breadcrumb module_unavailable` + `_MIMIC_SKIP=1` (bring the link up as plain UDP — closes
  the silent-no-degrade hole).
- policy=none → `ERROR: mimic kernel module unavailable — reboot into the current kernel or set
  mimic_fallback=udp` + `_mimic_breadcrumb module_unavailable` + `exit 1` (clean fail-closed, not a
  cryptic loop).

### 4. Orphaned-lock cleanup + modprobe on start (start block, L884-892 node / L1750 client + TS)
Before `systemctl restart "mimic@${IF}"`: a small helper (beside `_mimic_breadcrumb`) that clears a
wedged unit — `systemctl stop mimic@IF 2>/dev/null || true; rm -f /run/mimic/*.lock 2>/dev/null ||
true; systemctl reset-failed mimic@IF 2>/dev/null || true` (a node has exactly one mimic egress, so
clearing all `/run/mimic` locks while the unit is stopped is safe; `reset-failed` clears the
"repeated too quickly" rate-limit). Add `modprobe mimic 2>/dev/null || true` right before the restart
(belt-and-suspenders — the shipped unit's `Requires=modprobe@mimic.service` only helps once the module
is built). The native-downgrade `elif` also clears the lock before its retry. Mirror byte-exactly in
`frontend/src/compiler/renderers/script.ts` (escape `\${`, `grep -a` for the long lines).
