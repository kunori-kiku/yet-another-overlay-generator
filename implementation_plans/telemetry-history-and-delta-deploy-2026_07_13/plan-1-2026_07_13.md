# plan-1 — agent `cpu_pct` (true CPU utilization) + current-value panel row

**Goal:** the resource telemetry carries a real CPU % (what a "CPU chart" means), not just loadavg.

## Changes
- `internal/agent/telemetry_resource.go`: make `resourceSampler` stateful — keep the previous
  `/proc/stat` aggregate-cpu jiffies snapshot (total + idle+iowait) in the sampler struct
  (registered once in `BuildTelemetry`, so instance state persists across beats). Each sample:
  `cpu_pct = 100 * (Δtotal - Δidle) / Δtotal`, rounded to 0.1. First sample after start (no prior
  snapshot) and counter-wrap/Δtotal==0 → OMIT the field (charts show a gap, never a fake 0).
  New indirected `statFn` (mirror `loadavgFn`); best-effort: a /proc/stat read/parse failure omits
  cpu only, load+mem still report.
- `hostResource` gains `CpuPct *float64 \`json:"cpu_pct,omitempty"\`` (pointer so "unknown" is
  absent, not 0 — the ResourcePanel em-dash precedent). Metrics ride the opaque map — the controller
  stores verbatim (ZERO controller change), no conformance/drift impact.
- FE: `frontend/src/types/controller.ts` `NodeResource` gains `cpuPct?`; `lib/resource.ts` maps it
  (+ a severity helper consistent with `memSeverity`); `ResourcePanel.tsx` adds a CPU row (absent →
  em dash); i18n `resourcePanel.cpu*` en+zh.

## Tests
- Go: table tests with injected `statFn` fixtures — two-beat delta correctness, first-beat omission,
  wrap/zero-Δ omission, parse-failure keeps load+mem. Assert `BuildTelemetry` registers ONE stateful
  instance (two consecutive `Sample` calls on the registry produce a value on the second).
- FE: `lib/resource` unit tests (mapping + severity + absent); ResourcePanel snapshot/testid.

## Verify + branch
Go build/vet/test (default+airgap) + gofmt; FE `npm run build` + vitest + lint.
Branch `feat/telemetry-cpu-pct`.
