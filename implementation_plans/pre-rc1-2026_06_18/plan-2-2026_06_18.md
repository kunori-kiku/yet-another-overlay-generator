> Part of the pre-rc.1 program — see outline.md for mission, principles, and full sequencing.

## Prerequisites

- **plan-1 (1.1 hygiene/normalization) MUST land first.** plan-1 and plan-2 touch
  overlapping files: plan-1 owns ALL comment Englishization (the ~30 truncated godoc
  stubs, and `internal/render/render.go:1`'s CJK package doc), and plan-2 moves whole
  functions/types *with their comments verbatim*. Landing plan-1 first means plan-2's
  same-package moves carry already-Englishized comments and never collide with plan-1's
  comment-text edits. (Per the sequencing spine: plan-1 before plan-2 + plan-8 — shared
  files.) If plan-1 has not yet landed, plan-2 moves the existing comment text verbatim and
  plan-1 rebases its Englishize edits onto the symbol's new home.
- **No other plan is a hard prerequisite.** This is intra-package / edge-removing structural
  work. It is a producer FOR several siblings, not a consumer:
  - **plan-8 (1.8) line-anchored edits rebase ONTO this plan's splits** — see the explicit
    cross-ownership note in Phase 0.1 (handler_controller.go split). The shared ALLOCATION
    constants (`defaultTransitCIDR`, `backupDefaultLinkCost`, `minPinnedPort`) are NOT this
    plan's: plan-8 (1.8) single-sources them into a NEW `internal/allocconst` leaf package it
    owns; plan-2 leaves the `peers.go`/`semantic.go`/`script.go` const sites untouched (see

  - **plan-3 (1.3) ctx-plumbing** edits `controller/compile.go` bodies after this plan's
    whole-function R3 split; land R3 first or partition (signature-neutral moves rebase
    mechanically).

## Goal

Reduce the maintainability and diff-review debt concentrated in the backend's two verified
god-files and one documented naming/misplacement smell, **without changing behavior, the
public Go API contract that the frontend mirrors, or the (already-correct) package dependency
graph.** Concretely: decompose `internal/api/handler_controller.go` (~1984 lines) and
`internal/controller/compile.go` (955 lines) by concern into same-package sibling files;
hoist the two value types that are mis-homed in the leaf `internal/renderer` package up into
`internal/model` (these are unrelated value types — `Artifact`/`InstallFetch` — NOT shared
allocation constants); refresh the stale source-tree doc; and add a net-new package-doc
clarifying the orchestrator-vs-leaf split. **Single-sourcing the shared ALLOCATION constants
(`defaultTransitCIDR` and friends) is explicitly NOT this plan's work — it is owned by plan-8
(1.8) in a new `internal/allocconst` leaf package; plan-2 leaves those const sites untouched.**
The one genuinely structural-but-breaking item — renaming
`internal/renderer` -> `internal/template` — is **scoped and sequenced out of the mid-beta
window** per the skeptic's explicit anti-churn warning, with a zero-risk doc-comment
mitigation in its place.

## Implementation steps

Three design rules govern every task:

1. **Move, don't rewrite.** Decomposition is cut/paste of function bodies and type decls
   between files **in the same package and directory**. No symbol is renamed, no signature
   changes, no export-visibility changes, no comment edits (those belong to plan-1). Go links
   files within a package, so `go build ./...` is the mechanical correctness proof for a
   same-package move.
2. **Respect the already-correct dependency directions.** The one cross-package move this plan
   owns (`Artifact`/`InstallFetch` -> `model`, R4a) goes *toward the leaf everyone already
   imports*, so it can only *remove* edges, never add a backwards one. Precisely: R4a removes
   the `controller -> renderer` package edge (store.go was controller's sole renderer user) and
   the per-file renderer imports in `handler_bootstrap.go` / `release_pins.go`; it does NOT
   remove the `api -> renderer` package edge (handler.go still uses
   `RenderAllBabelConfigs`/`RenderDeployScripts`). Re-run `go list` after to confirm. (The
   shared allocation-constant single-source is plan-8's, into `internal/allocconst` — not part
   of this plan's dependency-graph deltas.)
3. **Lean on the existing test nets as the behavior-neutrality oracle.** Run the byte-/
   behavior-identity tests per phase, not just `go test ./...` (enumerated under "Tests
   produced by this plan").

Run all `go`/`gofmt` commands with the local toolchain + module mirror:
`PATH="$HOME/.local/go/bin:$PATH" GOPROXY="https://goproxy.cn,https://goproxy.io,direct" GOSUMDB=off`.
