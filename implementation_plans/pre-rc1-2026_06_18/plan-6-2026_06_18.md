# plan-6 — Local-mode rewire to the in-browser compiler  (subject S1, milestone 1.6)

> Part of the pre-rc.1 program — see outline.md for mission, principles, and full sequencing.

## Prerequisites

- **plan-4 — TypeScript compiler reimplementation** (`frontend/src/compiler/` full layering + keygen + export + `router_id`/F2). 1.6 consumes its public API and nothing else; it adds **no** new Go-mirroring TS logic of its own. plan-4's TS babel renderer MUST mirror the `babel.go:120-124` `InterfaceName` ascending sort (the consequence of C1, already merged in beta.8). `@noble/curves` + `jszip` are added to `frontend/package.json` by plan-4.
- **plan-5 — Go↔TS conformance harness** (`internal/conformance/`, fixture corpus, comparator, Go coverage floor, vitest runner) is **GREEN and a permanent required CI gate** before 1.6 starts coding. This is the load-bearing drift control: `drift_risk_adequately_mitigated` is FALSE until the harness exists and pins the duplication. The corpus must include the cross-subgraph pin-reservation path, the retroactively-pinned `normalizeEdges.ts` duplication, the **edge-reorder babeld byte-stability case** (proving plan-4's TS babel renderer mirrors `babel.go:120-124`), and a `router_id`-pinned node (F2).
- Sequencing spine: plan-3 (contract) → plan-5 (harness, built FIRST) → plan-4 (TS compiler, no TS lands until the harness is green) → **plan-6 (rewire)** → plan-7 (backend-shrink, after 1.6 soaks). plan-9 supports plan-4 by owning the `router_id` field addition to `frontend/src/types/topology.ts`.
- **C1 is NOT a prerequisite** — it is already merged in `v2.0.0-beta.8` (`babel.go:116-185`, sort at `:120-124`). The only related obligation is plan-5's edge-reorder fixture verifying the TS side matches.

## Goal

Switch YAOG **LOCAL** mode from a thin client that POSTs every compute action to the Go backend's air-gap routes, to a fully browser-resident pipeline that calls the in-browser TypeScript compiler library (`frontend/src/compiler/`, built in plan-4, drift-pinned by the plan-5 conformance harness). Concretely, rewire the four compute actions in `frontend/src/stores/topologyStore.ts` — `validate()`, `compile()`, `exportArtifacts()`, `downloadDeployScript()` — so that **in local mode** they call the TS compiler instead of `fetch('/api/validate'|'/api/compile'|'/api/export'|'/api/deploy-script')`. The seam ships **default-OFF** behind a typed env flag so merging 1.6 changes no default behavior (the server path stays the proven default), preserving the migration-isolation invariant; controller mode stays server-authoritative and zero-knowledge (public-keys-only), untouched.
