## Goal

Perform a single, isolated, behavior-preserving pass that normalizes **all** CJK source-code text in the
YAOG repository to English while preserving exact technical meaning, across two surfaces: (1) **all
source-code comments** (`//`, `/* */`, `{/* */}` JSX, and inline trailing field comments) in production and
test Go and the frontend; and (2) **all test-only diagnostic string literals** (`t.Errorf` / `t.Fatalf` /
`t.Logf` / `t.Error` / `t.Fatal` / `fmt.Errorf` / `fmt.Sprintf` failure messages in `_test.go` files —
verified **465 CJK occurrences across 31 test files**, not the lone case the draft assumed). The pass also
repairs the ~30 truncated godoc "stub" comments left by an earlier partial CJK-strip (e.g.
`internal/api/server.go:52` `// NewServer  API`, `:123` `// cors CORS`). The deliverable is a clean,
English-only comment-and-test-diagnostic layer with **no change to any rendered byte, runtime behavior,
identifier, production string literal, struct tag, JSX text, or template** — verified by the existing
air-gap byte-identity test suite. It unblocks reviewers and the downstream TS-compiler migration by
removing CJK noise from the diff surface before SUBJECT-1's structural refactor (plan-2) and the C-series
correctness fixes (plan-8) touch the same files.

## Implementation steps

**CJK definition used throughout** (the WIDENED definition — the narrow `[\x{4e00}-\x{9fff}]` the draft used
MISSES CJK punctuation entirely; verified 4 files carry CJK punctuation but ZERO ideographs:
`internal/renderer/sysctl.go`, `internal/validator/validator_test.go`, `internal/model/topology_test.go`,
`internal/allocator/ip_test.go`):

```
[\x{3000}-\x{303f}\x{4e00}-\x{9fff}\x{ff00}-\x{ffef}]
```

i.e. CJK Symbols & Punctuation (U+3000-303F, e.g. 、 。 「 」), CJK Unified Ideographs (U+4E00-9FFF), AND
Halfwidth/Fullwidth Forms (U+FF00-FFEF, e.g. fullwidth parens （ ） U+FF08/FF09, comma ， U+FF0C, colon ：
U+FF1A). NOTE: the em-dash `—` (U+2014) is **NOT** CJK and is correct to keep (134 Go files use it; never
flag it).

**Reference tokens to preserve verbatim throughout** (translate surrounding prose, keep tokens byte-exact):
dossier D-codes (`peers.go` has 10, `semantic.go` has 7, `(D40)`/`(D41)` in tests/babel), invariant
I-codes (`I1`..`I10`), `plan-N`/`plan-N.M` tags, `docs/spec/...` citations (e.g.
`docs/spec/compiler/allocation-stability.md（不变量 I10）` at `topology.go:33`), and backtick-quoted
shell/config/babel tokens (`redistribute local`, `redistribute ip ...`).

**Method:** scripted detection (the widened grep produces the per-file/per-line CJK inventory and flags
JSX / trailing-comment / template-adjacent / diagnostic-string cases), **manual translation per file** —
NO automated find-replace on comment or diagnostic bodies (machine translation drops reference tokens and
mis-renders technical intent). Leaf-first ordering (lowest blast radius first); one commit per phase, one
commit per hotspot file (`peers.go`, `semantic.go`, `script.go`, `controllerStore.ts`, `topologyStore.ts`).

**Per-phase verification gate** (Go env: `export PATH="$HOME/.local/go/bin:$PATH"`):
- Go phases: `gofmt -w` the touched files FIRST (English widths + trailing-comment column alignment differ
  → re-wrap is expected, R3), then `gofmt -l ./cmd ./internal` (must be empty), `go build ./...`,
  `go vet ./...`, `go test ./...` (incl. `internal/regression`).
- FE phases: `cd frontend && npm run build` (= `tsc -b && vite build`, the stricter project-references
  typecheck per project memory — NOT bare `tsc --noEmit`) + `npm run lint` (`eslint .`).
