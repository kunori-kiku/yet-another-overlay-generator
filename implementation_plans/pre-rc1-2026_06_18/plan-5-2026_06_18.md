## Goal

Build the drift-control apparatus that makes the owner-chosen TypeScript reimplementation of the local
compile pipeline (plan-4) safe to attempt at all: a fixed-key fixture corpus run through BOTH the Go
compiler (the authoritative oracle, frozen by plan-3) and the TS compiler in CI, asserting canonical-equal
output via a content manifest, with deterministic checked-in keys so output is byte-comparable across runs,
machines, and languages. The deliverable is a PERMANENT REQUIRED CI gate + a Go-side per-package coverage
FLOOR + structural drift guards that turn "changed Go without updating TS" into a red build mechanically.
It also retroactively pins the already-drifting `frontend/src/lib/normalizeEdges.ts` duplication (the F3
five-list / pin-key drift class on the FE side). The Go-oracle half ships standalone value as a
golden-output regression net for the Go pipeline itself — it would have caught the beta.5–beta.7 pin-drift.
This plan is the load-bearing answer to `arch_skeptic.drift_risk_adequately_mitigated = FALSE`.

## Implementation steps

> Land ISOLATED off `main` on a dedicated branch (per locked scope: the migration is isolated from the
> active PR stream). Blast radius is contained to `internal/conformance/`, the catalog-sync test, FE test
> config, `ci.yml`, and docs — NO production pipeline code. Within-plan order: 1→2→3 first (the Go
> oracle/golden ships value standalone); 4–6 in parallel after step 1; 7 after step 3 + vitest; 8 last; 9

1. **Manifest package + canonical serializer.** Create `internal/conformance/manifest.go`: the manifest
   struct + a canonical-JSON (sorted-key, LF, no trailing whitespace) serializer; the **two-channel
   verdict** type; peers keyed `LinkKey|dir`. The one canonical document per fixture:
   ```
   {
     "fixture": "<name>",
     "verdict": {
       "validator": ["<sorted validator.Code set from the 200 ValidateResponse errors[]/warnings[]>"],
       "apierr":    ["<sorted apierr.Code set from the HTTP/compile error envelope, empty on success>"]
     },
     "topology":    <canonical-JSON of the post-write-back topology>,
     "allocations": { "node_overlay_ips": {...}, "peers": { "<linkKey>|<dir>": {...} } },
     "files":       { "<relpath>": "<verbatim bytes>", ... },   // per-file by key, sorted
     "checksums":   "<bundlesig.Canonicalize output verbatim>"
   }
   ```
   Bake in the exclusion set by MIRRORING plan-3's `io-contract.md` IN/OUT list (do NOT redefine it):
   exclude `compiled_at` (`export.go` manifest), `computeChecksum` (`compiler.go:234`, C4), and the
   self-extracting tar.gz wrapper bytes (compare bundle CONTENTS + checksums only). Add a schema doc stub
   pointing at step 9. Design invariants to enforce in the serializer:
   - peers are a SET keyed by `linkid.LinkKey` (`linkid.go:53-60`), never a slice index — PeerMap append
     is edge-array order (`peers.go:570`);
   - per-file by key, never a concatenated blob — WG configs keyed `nodeID:interfaceName`
     (`export.go:85-98`), Go map iteration non-deterministic;
   - both verdict channels are sorted code sets so neither is free to drift.

2. **Go oracle + golden mode.** Create `internal/conformance/oracle.go` + `internal/conformance/golden_test.go`.
   Drive the production path through plan-3's `localcompile` façade where available:
   `localcompile.Compile(CompileRequest{Custody: AirGap, Keygen: fixed-key wgtypesKeygen, Fetch: zero,
   CompiledAt: fixed})` → `artifacts.Export` → build the manifest. If plan-3's façade is not yet merged at
   harness bring-up, fall back to the explicit `render.GenerateKeys → compiler.Compile → render.All →
   artifacts.Export` sequence, then swap to the façade once it lands (same bytes by plan-3's contract).
   Reuse the case-(a) wiring SHAPE from `equivalence_test.go:27-90` (fixed key present → derive public),
   NOT its random `mustGenerateKey`. Two modes:
   - **golden** (`-update`): Phase-0 standalone regression net — freezes today's Go bytes; valuable
     regardless of the TS port (would have caught beta.5–beta.7 pin-drift);
   - **conformance**: byte-compares the Go manifest against the TS-produced manifest with a
     first-divergence report.

3. **Seed / unify the fixture corpus.** Author or adopt ~10 fixtures with **checked-in fixed per-node
   private keys** (case (a), `render.go:106-115`), derived from existing inline topologies in
   `internal/compiler/compiler_test.go`, `parallel_links_test.go`, `allocation_stability_test.go`,
   `transit_pools_test.go`, `mimic_test.go`, `peers_endpoint_test.go`, and
   `internal/validator/validator_test.go`. **UNIFY with plan-3's `internal/localcompile/testdata/contract/`:

   MUST reference plan-3's directory, not duplicate it.** Coverage requirements:
   - every role / client / parallel-backup / pinned-unpinned;
   - delete-readd, multi-domain custom transit, mimic-tcp, endpoint-`has_public_ip`;
   - a `router_id` fixture (`topology.go:87`) so **F2 drift is mechanically visible** — the harness reds
     if the TS `Node` drops `router_id` (adding `router_id?` to TS is plan-4/plan-6's job, not closed here);
   - **≥2 fail fixtures spanning BOTH channels**: one validator failure (e.g.
     `validation_client_endpoint_host_required`) AND one apierr compile failure (e.g.
     `compile_transit_pool_exhausted` / `CodeTransitPoolExhausted` on a narrowed transit CIDR).

4. **X25519 KAT.** Create `internal/conformance/testdata/keygen_kat.json` (RFC 7748 test vectors +
   clamp-boundary cases: `key[0] &= 248`, `key[31] &= 127 | 64`) + `internal/conformance/keygen_kat_test.go`.

5. **Codes-catalog sync guard (SSOT fix).** Create `internal/i18n_catalog_sync_test.go`. Read the
   `error.<code>` key set out of `frontend/src/i18n/messages/en.ts` AND `zh.ts`, and assert that EVERY
