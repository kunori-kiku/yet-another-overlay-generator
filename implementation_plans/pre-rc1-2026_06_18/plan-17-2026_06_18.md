## Goal

Build the **device-emulation smoke layer** (viewport + touch + visual-regression snapshots) that *verifies* Subject 2's operator-surface responsive work actually landed and *keeps* it from silently regressing. This plan ships Playwright specs that drive the built panel at phone, tablet, and desktop viewports with touch emulation enabled, asserting that each phone-readiness blocker the investigation enumerated is fixed at narrow widths — plus a Playwright `toHaveScreenshot` visual-regression corpus that pins the rendered chrome so a later CSS edit that breaks the mobile drawer or re-cramps the fleet table fails CI loudly. It is **verification, not feature work**: zero production CSS/TSX. Every smoke asserts behavior crossing the agreed **`lg` = 1024px** boundary and binds to the **ARIA-first** affordance surface Subject 2 actually emits (accessible names / `role="dialog"`), never an invented `data-testid` taxonomy — so the suite cannot become a second hand-synced selector vocabulary that drifts from Subject 2.
