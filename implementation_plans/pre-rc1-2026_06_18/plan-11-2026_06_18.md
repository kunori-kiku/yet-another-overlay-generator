## Implementation steps

> Phases 0ŌĆō2 are unconditional (the primitive + contract + verification). Phase 3 is **Contingency-B only** ŌĆö skip it entirely under the recommended Contingency A.

### Phase 0 ŌĆö the single boundary hook (no visible change)

1. Create `frontend/src/lib/useMediaQuery.ts`:
   - `export const LG = '(min-width: 1024px)'` ŌĆö the SINGLE canonical Subject-2 boundary (Tailwind default `lg`, CSS-first config, no `tailwind.config.js`).
   - `export function useMediaQuery(query: string): boolean` ŌĆö `useState` initialized **synchronously** from `window.matchMedia(query).matches` (avoids a first-paint flash, like `ThemeProvider`), a `useEffect` subscribing `media.addEventListener('change', handler)` with cleanup, **directly mirroring `frontend/src/theme/ThemeProvider.tsx:26-33`** (and the `:7-8` client-only assumption ŌĆö no SSR guard).
   - `export function useIsDesktop(): boolean` ŌåÆ `useMediaQuery(LG)`.
   - **Polarity is `min-width` / `isDesktop` (true at `lg+`)** to match plan-12 (2.3)'s already-chosen `useMediaQuery('(min-width: 1024px)')` / `isDesktop`. Consumers needing "below lg" negate it (`!isDesktop`). Header comment: this is the SINGLE Subject-2 boundary that plan-10's Tailwind `lg:` classes and plan-12's gate also key off `1024`/`min-width`, so CSS and JS can never disagree.
   - *(If plan-12 lands first and has already created this file, 2.2 CONSUMES it and skips this task ŌĆö the file has exactly one author; coordinate at PR time.)*

### Phase 1 ŌĆö the shared `Drawer`/`Sheet` primitive (no consumer yet)

2. Add a `CloseIcon` (X, e.g. `<path d="M6 6l12 12M18 6L6 18" />`) to `frontend/src/components/shell/icons.tsx` using the existing `Base` wrapper (`icons.tsx:9`). **Do NOT add `MenuIcon` here** ŌĆö the hamburger belongs to whichever plan owns the sidebar consumer (added only in Phase 3 under Contingency B).

3. Create `frontend/src/components/shell/Drawer.tsx` ŌĆö props `{ open, onClose, side: 'left' | 'right' | 'bottom', ariaLabel, children }`. A **controlled** component (holds NO open-state of its own; consumer owns `open`/`onClose`). Responsibilities:
   - **Backdrop:** `fixed inset-0 z-30 bg-black/40`.
   - **Panel:** `fixed z-40`, slide-in via `transition-transform` (so `index.css:71-80`'s reduced-motion rule neutralizes it automatically ŌĆö no JS animation that bypasses it). `side` picks the translate axis + anchor: `left`/`right` = `inset-y-0` + horizontal translate; `bottom` = `inset-x-0 bottom-0` + vertical translate (= the **Sheet** shape).
   - **Theming:** panel uses shell theme tokens (`var(--surface-elevated)`, `var(--hairline)`) + the `.app-chrome` vibrancy class so it matches the token-themed sidebar/topbar.
   - **a11y:** `role="dialog"`, `aria-modal="true"`, `aria-label={ariaLabel}`; a close (X) button using the new `CloseIcon` + `FOCUS_RING` (`shell/styles.ts:5`), `aria-label={t(language, 'shell.closeDrawer')}`; **focus-trap** (capture `document.activeElement` on open, focus the first focusable in the panel, cycle Tab within, restore the captured element on close/unmount); **Esc** and **backdrop-click** call `onClose`; **body-scroll-lock** (toggle `overflow-hidden` on `document.body` while open, removed on unmount **even if closed via route change**); **route-change auto-close** via a `useLocation()` effect that calls `onClose` on pathname change (the Drawer is inside `createBrowserRouter`'s tree, so `useLocation` is available).
   - **Header comment (anti-debt):** (a) this is the canonical overlay primitive; (b) a future `variant: 'centered'` (focus-trapped, Esc/scroll-locked centered modal) is a small additive change and the migration target for the four ad-hoc `role="dialog"` overlays at `CanvasToolbar.tsx:133`, `DeployBar.tsx:335`, `AgentUpdateSettings.tsx:386`, `SettingsPage.tsx:103`; (c) the z-index contract (backdrop `z-30` / panel `z-40`, strictly below the existing `z-50` modal layer and `Shell.tsx:85`'s `focus:z-50`, so a confirm dialog always wins). NOT building the `centered` variant here.

4. Add i18n keys to BOTH `frontend/src/i18n/messages/en.ts` and `frontend/src/i18n/messages/zh.ts`: `shell.closeDrawer` (the X-button label) and `shell.dialogBackdrop` (only if a labeled backdrop is used). Parity is enforced by `zh.ts:7`'s `Record<keyof typeof en, string>` and `index.ts:17`'s `MessageKey = keyof typeof en` ŌĆö a missing zh key is a `tsc -b` error. Consumer-specific labels (e.g. `shell.openNav`) are added by the consuming plan, NOT here.
