## Goal

Build a **REQUIRED, rc.1-gating** CI tier for Subject 3 that brings up the **configs YAOG
actually generates** — per-peer WireGuard interfaces + Babel + the `install.sh` activation
sequence — inside Linux **network namespaces** on a single CI runner, and then **asserts the
overlay works**: WireGuard tunnels handshake, `babeld` converges routes to peers' overlay
IPs, an end-to-end overlay ping succeeds, and the overlay-SNAT rule rewrites transit-source
traffic to the overlay IP. This is the closest a test can get to real hardware without real
hardware, and it is the only place a class of pitfalls surfaces that **every layer above it
structurally cannot see**: the byte-correctness layers prove the *bytes* are stable; the
agent-wire tests prove the *bundle is delivered and verified*; the 3.1 Playwright rig proves
the *product UI flows* — **none of them ever runs `install.sh` against a kernel.** A config
that is byte-perfect, delivered, verified, and clicked-through can still fail to form a
tunnel, fail to converge a route, or silently drop traffic at the SNAT rule. The MANDATORY
floor (the `simple-mesh` canary, asserted green 20/20) gates the `v2.0.0-rc.1` tag; the
`realtunnel` job is wired as a required status check from the start.

**Phase 4 — netns wiring from the topology graph + endpoint rewrite.** **Before compile**,

pairs matching the topology edges) and move one veth end into each node namespace, assigning
the rewritten IP. **Verify raw underlay reachability (ping the veth IPs) BEFORE any WG
bring-up**, so an underlay miswire is distinguished from a config bug (R3). The rewrite is
**mandatory, not polish** — the shipped examples carry TEST-NET addresses that are NOT
reachable veth addresses.

**Phase 5 — Per-node activation (Option A extract-and-run; or Option B per Phase 1).** In
each namespace, as root, **execute the load-bearing command lines extracted from the node's
rendered `install.sh`** (reading the SAME files the bundle produced — substrings of the bytes
that ship, NOT a hand-port):
  - `ip link add dummy0 type dummy` + `ip addr add <OverlayIP>/32 dev dummy0` + `ip link set
    dummy0 up` (mirrors `script.go:506-512`);
  - `wg-quick up <iface>` for each `wireguard/<iface>.conf` (mirrors `script.go:730-739`;
    `Table = off` per `wireguard.go:59` means routing is babel's job, plus the
    `wireguard.go:72` client `PostUp ip route replace`);
  - apply `sysctl/99-overlay.conf` keys via `sysctl -w` (`sysctl.go`; the script form is
    `sysctl --system` at `script.go:686`);
  - install the overlay SNAT rule — nft `oifname "wg-*" … snat to <OverlayIP>`
    (`script.go:580-581`) or `iptables -t nat -A POSTROUTING -o "wg-+" -s <transit> -j SNAT
    --to-source <OverlayIP>` (`script.go:591-592`); teardown mirrors the uninstall match
    (`script.go:558-567`);
  - start `babeld -c babel/babeld.conf` as a managed child (the unit's
    `ExecStart=/usr/sbin/babeld -c …` at `script.go:754`).
  Skip ONLY the systemd plumbing (`systemctl enable` of `overlay-dummy.service`/
  `overlay-snat.service`/`wg-quick@…`/`babeld`, the unit-file heredocs). Add the
  **template-shape pin** test (Anti-drift below) so the extractor cannot silently drift from
  `script.go`.

  **Anti-drift (concrete, load-bearing — the Option-A risk mitigation):**
  1. **Extract-and-run, don't retype.** The harness reads each node's **rendered
     `install.sh`** (the same file `checksums.sha256` covers — `export.go:34-40`) and drives
     activation by programmatically extracting and executing the script's own netns-safe
     command lines. The commands the test runs are thus **substrings of the bytes that ship**,
     not a parallel transcription — avoiding the dual-maintenance drift the rc.1 program is
     fighting.
  2. **Template-shape pin (a `script_signature_test.go`-style guard).** A test greps the
     rendered `install.sh` for each load-bearing command shape the extractor relies on
     (`dummy0 type dummy`, `wg-quick up`, `-j SNAT --to-source` / `snat to`,
     `babeld -c /etc/babel/`) and **FAILS LOUD if the template changes shape** so the
     extractor no longer matches — turning silent drift into a red test that forces harness
     and template to be reconciled in the same PR.
  If a step cannot be replayed outside `wg-quick`'s wrapper, that step's exact retyped command
  is allowed ONLY with the shape-pin guard covering it, documented in the README.

**Phase 6 — Convergence + data-plane assertions (the MVV floor payload).** Gate every wait on
a **bounded poll loop** (hard timeout, e.g. 30s, fails loud) — **NO fixed `sleep`**:
  - **(a) Handshake (REQUIRED):** poll `wg show <iface> latest-handshakes` per interface until
    non-zero → tunnels formed.
  - **(b) Route convergence (REQUIRED):** poll each node's kernel table for a route to every
    other node's `OverlayIP/32` (the `AnnounceSelf` redistribute, `babel.go:153-154`) → babel
    converged.
  - **(c) End-to-end overlay ping (REQUIRED):** `ping`/`ping6` from node A's overlay IP to
    node B's, expect **0% loss** → the generated overlay actually routes packets.
  - **(d) SNAT transit→overlay rewrite (REQUIRED floor assertion):** confirm a packet sourced
    from a transit IP is source-rewritten to the overlay IP (the unique-to-netns data-plane
    check the byte layers structurally cannot see; `script.go:580-581`/`591-592`). **This is a
    REQUIRED floor assertion, not a stretch.** It is demoted to Tier 2 ONLY if Phase 5
    surfaces a concrete netns blocker (e.g. the conntrack/nat hook is unavailable in the
    runner kernel), in which case the blocker and the demotion are documented in `RC1-GATE.md`
    and the README.
  On any failure, dump `wg show`, `ip route`, `ip addr`, `iptables-save -t nat`, and a babeld
  dump into the test log.

**Phase 7 — Scenario coverage — canary floor, then additive.** The `simple-mesh` canary

  - the **C3 adversarial fixture** — a purpose-built topology: a node with non-empty

  second source of truth — the same drift class the rc.1 program is fighting).
  **Retirement trigger:** the extract-and-run mechanism is removed (e.g. the floor moves to
  Option B / Tier-2 unmodified-script-under-systemd, which eliminates the surface entirely).
  **Retirement destination:** tests/legacy/rc1/realtunnel/.
- **`test/realtunnel/` C3 adversarial fixture + scenario** (purpose-built node with
  `public_endpoints` set AND `has_public_ip=false`, no reverse edge; asserts the reverse
  `Endpoint` is empty and the handshake forms one-directional).
  **Lifetime:** subject-scoped (S3).
  **Guards:** turns compiler finding C3 (`peers.go:783-785`, "compiles clean, validator only
  warns, tunnel never forms one way") from a validator-warning into a caught regression — the
  only place that failure becomes observable.
  **Retirement trigger:** Subject 3 closes AND the C3 contract is covered by a compiler-level
  unit/conformance test that asserts the empty reverse `Endpoint` directly.
  **Retirement destination:** tests/legacy/rc1/realtunnel-c3/.
- **`test/realtunnel/` additive scenarios** (`relay-topology` relay forwarding + transitive
  reachability; optional `nat-hub` NAT hub-and-spoke forwarding — NOT C3), all behind a flag.
  **Lifetime:** subject-scoped (S3).
  **Guards:** the relay/gateway forwarding data-plane paths route packets on a kernel beyond
  the full-mesh canary.
  **Retirement trigger:** Subject 3 closes (these are additive breadth beyond the required
  floor; the load-bearing gate is the `simple-mesh` canary).
  **Retirement destination:** tests/legacy/rc1/realtunnel-scenarios/.

## Definition of done

1. `sudo go test -tags integration ./test/realtunnel/...` on Linux brings up the
   `simple-mesh` canary in network namespaces from **`cmd/compiler`-produced bundles with
   real WG keys** and asserts, green, the **full MVV floor**: (a) per-interface WG handshake,
   (b) babel-converged kernel routes to every node's `OverlayIP/32`, (c) end-to-end overlay
   ping with 0% loss, and **(d) SNAT transit→overlay source-rewrite correctness** (required
   unless a documented netns blocker demotes it to Tier 2 in `RC1-GATE.md`/README).
2. Bring-up and teardown are fully scripted and deterministic — **no fixed `sleep`**,
   convergence gated on bounded polls; teardown leaves no orphan namespaces/veths/bridges or
   temp dirs even on failure; a pre-run sweep removes orphans from a crashed prior run.
3. The tier is **test-only and isolated**: every file carries `//go:build linux &&
   integration`; the default `go test ./...`, `go vet`, gofmt, and frontend jobs are
   unchanged and green; **zero production code added** — bundles come from the unmodified
   `cmd/compiler` / `internal/artifacts` path and `checksums.sha256` verifies.

   headless, uploads failure logs, and is wired as a **REQUIRED status check** on `main`

5. The negative-proof test exists and demonstrably **fails loud** when a wire is broken

6. The canary passes the **20/20 bake-in + negative-proof (Phase 9)** — the precondition to
   wiring `realtunnel` as a required check and to cutting rc.1 (a required gate must not be

8. `test/realtunnel/README.md` documents the run command, prerequisites, the extract-and-run

## Out of scope for this plan

- Any **production-code behavior change.** The Go pipeline (`internal/compiler`,
  `internal/renderer`, `internal/artifacts`, `cmd/compiler`) is the single oracle; this tier
  only *consumes* its output. The only Go code added is **test-only** (the netns driver +
  helpers under the integration build tag).
- **mimic / `transport:tcp` real-tunnel coverage.** mimic is eBPF/XDP attached to the egress
  NIC at runtime, needs a kernel module + a distro package, and is platform-fragile; this
  tier stays `transport:udp` + babel + iptables/nft SNAT only. A Tier-2/3 stretch, deferred.
- **Controller-mode / agent self-update / keystone** flows. Those are the agent-wire tests'
  and 3.1's domain. This tier asserts the *generated overlay config behaves on a kernel*; it
  consumes air-gap bundles produced by `cmd/compiler`, not controller-staged bundles.
- **The TS-compiler conformance harness** (`internal/conformance/`, Subject 1 / plans
  1.4–1.6) — a different artifact (Go↔TS byte-equality). No overlap; no shared files. (Future
  synergy noted, not in scope: once the TS local compiler exists, this same netns tier can be
  pointed at TS-produced bundles as a *second* real-world oracle — the byte-equality harness
  proves the bytes match; this proves those bytes route packets.)
- **Tier 2 (containers, real systemd, unmodified `install.sh`) and Tier 3 (multi-host VMs,
  real NAT).** Designed and documented as the post-rc.1 upgrade path; NOT the mandatory floor.
  Tier 3 is the documented "real hardware minus the hardware" ceiling (manual/nightly at
  most). Each is a separate multi-session effort.
- **Windows / non-Linux.** Linux netns + WG is inherently Linux-only; the build tag is
  `linux && integration`.
- **The owed-smoke legs that remain irreducibly owner-on-hardware (Tier 2/3):** the real
  systemd `systemctl restart` lifecycle, the keystone-trust/reprovision custody path, the
  WebAuthn passkey leg, and real-NAT traversal. This tier discharges the *data-plane* legs of
  owed smokes #2 (NAT sticky-pin: "deployed config still forms tunnels and routes") and #6
  (keystone rotation: "trusts the fresh signed deploy → tunnels form + routes converge") — it
  discharges *"the generated overlay config routes packets,"* NOT *"real systemd + real NAT +
  passkey."* plan-22 / 4.3 may mark only those data-plane legs "now automated"; the rest stay
  "owner-run-passed."
