## Goal

Make the **default / controller** YAOG backend **controller-only** in its *reachable surface*,
while **retaining** the four air-gap compute routes in the codebase behind a `//go:build airgap`
build constraint so an `-tags airgap` build still ships them as the local-design oracle. After the
TS-migration track (plans 1.4/1.5/1.6) has shipped, the frontend's LOCAL mode runs entirely in the
browser and never POSTs to the Go backend's air-gap compute routes. This plan moves the four
anonymous air-gap compute routes (`/api/validate`, `/api/compile`, `/api/export`,
`/api/deploy-script`), their handler functions, and the three handler-local ZIP helpers **behind the
`airgap` build tag** so that the **default (controller) build does not register or even link them** —
no anonymous/unauthenticated code path reaches the keygen/allocator/compiler pipeline in the shipped
controller — while an explicit `-tags airgap` build **retains** them unchanged (the local-design
oracle + the boot target for plan-13's `--mode airgap` E2E and plan-21's `-tags airgap` DAST). It
retains `/api/health` (ungated liveness probe, **registered in BOTH builds**), `readTopology` (used
by the operator-only `HandleCompilePreview`), and `EnableStatic`/`YAOG_WEB_DIR` (serves the panel
SPA). It establishes a two-deployment story (a static SPA pinned to local mode via a
`VITE_LOCAL_ONLY` build flag, plus the controller server) and banks the security delta: in the
default controller build S1/S2/S3/B4 are downgraded from anonymous-remote to operator-or-CLI-trust
(and absent entirely from the backend-free static site), while the `-tags airgap` build deliberately
re-exposes them as the local oracle. The algorithms themselves are **not** fixed here — that is
plan-8.

**Phase 1 — Gate the air-gap routes, handlers, and helpers behind `//go:build airgap`.**

The split rule: the **un-tagged** files (`server.go`, `handler.go`) keep only what BOTH builds need;
everything air-gap-only moves into **new `//go:build airgap`** files in the `internal/api` package.
Both builds compile the same package; the `airgap` tag merely adds the gated files. `go build ./...`
(default) yields the controller-only surface; `go build -tags airgap ./...` yields the full oracle.
