## Goal

Run a **diff-aware** adversarial security pass over the post-Subject-1/2/3 frozen tree: audit **what
changed** since the captured pre-refactor baseline, and prove that **nothing** in the refactor regressed
the security fixes already shipped in beta.8 (B1 panic recovery, F1 getTrustlist keystone-sign-401, C1
babeld.conf byte-stability, S4 revoked-resurrection guard, S5 enrollment-token purge-on-revoke, S6
enrollment-token TTL cap) or the Go-side security/correctness fixes landed by **plan-8 (1.8)** (S1–S3
compile-DoS caps + ctx, S7/S8/S11 SSRF completeness, B2 fsync, the bootstrap arg-injection fix, C2/C3
compiler-correctness, single-sourced constants, and the recorded B3/S9/S10/TOFU-MITM decisions). This is
a **fresh run against the new tree, NOT a resume of the pre-refactor audit** — the threat model is the same
but the code is materially different (the TS local-mode reimpl makes the deployed backend controller-only
and plan-7 physically removes the four unauthenticated air-gap routes; the TS port is a genuinely-new
attack surface). The pass is two-pronged: a **static diff audit** (re-derive the threat for each changed
surface; verify each fix is still present AND still load-bearing) AND **DYNAMIC testing (DAST)** against a
LIVE instance driven by the Subject-3 harness (replay the original exploits at the wire and assert each is
still refused). It also verifies plan-7's air-gap-route removal actually shipped and audits the new TS-port
surface (browser keygen randomness, JSZip filename safety, TS-validator reject-path parity, conformance
gate wiring), then institutionalizes the missing CI security tooling. The deliverable is a written GO/NO-GO
verdict with a residual-risk register — the **SUBJECT-4 security sign-off** that feeds the authoritative
rc.1 gate (plan-22, criterion A2) and the plan-20 exit bar; it does NOT itself gate the tag (plan-22 owns it).
