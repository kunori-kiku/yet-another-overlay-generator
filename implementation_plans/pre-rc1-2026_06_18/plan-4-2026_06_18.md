> Part of the pre-rc.1 program — see outline.md for mission, principles, and full sequencing.

## Prerequisites

**Both hard upstreams must have LANDED, and the harness must be GREEN, before any TS code lands here. Harness FIRST.**

- **plan-5 (1.5 conformance-harness+CI) — HARNESS FIRST, the controlling gate.** plan-5 builds the Go↔TS conformance harness (`internal/conformance/`), the fixture corpus, the `code.go → codes.ts` codegen + sync-guard, the X25519 KAT vectors (`testdata/keygen_kat.json`), the transit-pool drift manifest, and the vitest runner + config. plan-5 states "Blocks 1.4 and 1.6" (`1.5-conformance-harness-ci.md:48`). **No TS module in this plan is trusted, and no layer is merged, until the harness asserts it byte-equal to the Go oracle for the fixtures exercising its branches.** This is the structural answer to `drift_risk_adequately_mitigated = FALSE`.
- **plan-3 (1.3 compiler-extraction+frozen-contract) — the contract this plan CONSUMES.** plan-3 freezes: the `internal/localcompile.Compile(req)` façade (input→output contract this plan mirrors), the frozen `model.ts` (incl. `router_id` (F2), the six `pinned_*` fields, `compiled_port`, `alloc_schema_version`), the `CodeXxx` enum source-of-truth, the three-operation `Keygen` interface (`1.3-compiler-extraction-contract.md:79-85`), the golden fixture corpus, and `docs/spec/compiler/io-contract.md`. This plan imports/extends those artifacts; it does NOT re-freeze them.
- **plan-9 (1.9 FE↔Go-type-drift) — supports this plan.** plan-9's type-drift reconciliation (notably `router_id` on the FE `Node`) is consumed here; the F2 fix is closed in the library by this plan once the frozen `model.ts` carries the field.

Spine note: plan-3 before {plan-4, plan-5}; **plan-5 before plan-4** (harness before the TS port consumes it); plan-4 + plan-5 before plan-6 (store rewire); plan-6 before plan-7 (backend-shrink removes the air-gap routes).

## Goal

Reimplement the entire YAOG local/air-gap compile path as a pure-TypeScript **library** under `frontend/src/compiler/`, mirroring the Go layering leaf-first so that local mode (design → validate → compile → export) runs entirely in the browser and the deployed server becomes controller-only. This plan ships only the pure library plus a keygen KAT — a single side-effect-free public entry (`compile()`/`validate()`/`exportArtifacts()`/`deployScript()`) that the harness (plan-5) and the store rewire (plan-6) consume — and the one permitted touch outside `frontend/src/compiler/`: retiring the duplicated `pinKey`/`linkKey` from `normalizeEdges.ts` into the new `linkid.ts`. The Go pipeline (`internal/{validator,allocator,compiler,renderer,render}` + the plan-3-frozen `internal/localcompile` façade) remains the authoritative oracle forever; the TS port's correctness is mechanically pinned by the conformance harness. **Store wiring, the `VITE_YAOG_LOCAL_ENGINE` flag, the dev canary, and the production cutover are NOT here — they are plan-6.**
