## Prerequisites

- **plan-1 (1.1 hygiene)** must land first — it owns the CJK→English comment-hygiene pass and shares
  files with this plan (`internal/render/render.go`, `internal/renderer/babel.go`,
  `internal/compiler/compiler.go` all carry CJK comments today). plan-1 lands before plan-2 and plan-8;
  this plan layers its new façade package on top of a hygiene-clean tree. (Per the sequencing spine:
  plan-1 before plan-2 + plan-8 for shared files.)
- This plan is the **hard prerequisite** for **plan-4 (1.4 TS-compiler reimplementation)** and
  **plan-5 (1.5 conformance-harness)**: it produces the *contract those plans consume*. Per the locked
  spine, plan-5 is built FIRST in the migration track and consumes this contract; plan-4 then mirrors
  the seam in TS. plan-9 (FE↔Go type-drift) supports plan-4 and **owns `router_id`** — this plan does
  NOT define or fix `router_id` semantics, it only *freezes* the current write-back behavior it observes.
- No upstream dependency on plan-2 (structure-redesign): the façade *wraps* rather than relocates, so it
  pulls toward plan-2 (single clean compile boundary) without blocking on it.

## Goal

Carve the **local compile path** — schema/semantic validation, IP allocation, capability inference,
peer derivation, the renderers, `render.All`, and `artifacts.Export` — behind a single clean Go façade
(`internal/localcompile.Compile`) with a **stable, documented, reproducible input→output contract**: a
canonical `topology-in` request and a canonical `artifacts-out` result, with every non-deterministic and
environment-coupled input lifted into explicit parameters (keygen seam, signing key, fetch settings,
clock, reserved allocations for the controller path). Decouple WireGuard key derivation from
`wgtypes`/`wgctrl` (the one true browser/WASM blocker) behind a `Keygen` seam, with a stdlib
`crypto/ecdh` X25519 path proven byte-equal to today's output. Freeze the contract as a documented
schema (`docs/spec/compiler/io-contract.md`) plus a golden-fixture corpus that pins today's exact byte
output. **C1 (babeld.conf byte-stability) already shipped in beta.8** — Phase 3 is verify-only; the
contract must *encode* the as-shipped "sort the node's peer slice by `InterfaceName` before render"
behavior as a load-bearing rule the TS port must mirror. This plan writes **no TS code and no harness
assertions** and introduces **no intentional byte change** — it is a pure wrap-and-freeze.

### Phase 1 — Define the frozen contract types + the façade skeleton (no behavior change)

1. Create `internal/localcompile/contract.go` with the request/result types. `SigningKey` is the
   `bundlesig.ConfigSigner` **interface** (NOT a pointer — bundlesig.go:116; this avoids the typed-nil
   gotcha, R6). `CompileArtifacts.Files` mirrors the per-node `bundleFiles` map `artifacts.Export`
   already builds (export.go:154-175); `Signatures`/`SigningPubPEM` mirror export.go:202-217; `Manifest`
   carries the OUT-of-conformance `CompiledAt`/`Checksum` (compiler.go:65-72):

   ```go
   type CompileRequest struct {
       Topology   model.Topology                // the only required input
       Custody    render.KeyCustody             // AirGap (local/CLI) | AgentHeld (controller)
       Keygen     Keygen                        // seam; nil ⇒ default wgtypesKeygen
       SigningKey bundlesig.ConfigSigner        // interface, nil ⇒ unsigned (NOT a pointer)
       Fetch      render.FetchSettings          // explicit, replacing FetchSettingsFromEnv
       CompiledAt time.Time                     // explicit clock, replacing time.Now() (compiler.go:225)
       Reserved   *compiler.ReservedAllocations // controller subgraph path only (compile.go:159); nil ⇒ full compile
   }
   type CompileArtifacts struct {
       Topology      *model.Topology              // pins written back (six pinned_* + CompiledPort + OverlayIP + RouterID)
       Files         map[string]map[string]string // nodeID -> (relpath -> content)
       Deploy        map[string]string            // deploy-all.sh / .ps1
       Checksums     map[string]string            // nodeID -> checksums.sha256 (via bundlesig.Canonicalize)
       Signatures    map[string]string            // nodeID -> bundle.sig (base64), present only when SigningKey != nil
       SigningPubPEM []byte                       // PKIX public-key PEM, identical per node (present iff signing on)
       Warnings      []validator.ValidationError
       Manifest      compiler.CompileManifest     // carries CompiledAt + Checksum; both OUT of the conformance byte set
   }
   ```

   Note on `Topology.RouterID` write-back: this field is *observed and re-emitted* by the contract; this
   plan does **not** define or change `router_id` semantics (plan-9 owns `router_id`). The contract only
   freezes whatever the current pipeline writes back.

### Phase 2 — Keygen seam + stdlib X25519 equivalence (both custody branches)

1. Create `internal/localcompile/keygen.go` with the `Keygen` interface + `wgtypesKeygen`:

   ```go
   type Keygen interface {
       // DerivePublic: base64 public key for a base64 X25519 private key.
       // Covers AgentHeld pub-from-private (render.go:89-93) AND air-gap
       // case-a/case-c public derivation (render.go:115,132).
       DerivePublic(privB64 string) (pubB64 string, err error)
       // Generate: fresh (privB64, pubB64) X25519 pair (air-gap case-c, render.go:126-132).
       Generate() (privB64, pubB64 string, err error)
       // ParseAndNormalize: round-trip a private key to its canonical base64 form. MUST
       // reproduce wgtypes' privateKey.String() canonicalization byte-for-byte (the air-gap
       // case-a re-write at render.go:114 persists this back onto the node), not merely validate.
       ParseAndNormalize(privB64 string) (canonicalPrivB64 string, err error)
   }
   ```

   `wgtypesKeygen` wraps today's exact calls (`wgtypes.ParseKey`/`GeneratePrivateKey`/`.String()`/
   `.PublicKey().String()` exactly as render.go:89,93,109,114,115,126,131,132). `ParseAndNormalize`
   reproduces `privateKey.String()`; `DerivePublic` reproduces `ParseKey(...).PublicKey().String()`;
   `Generate` reproduces `GeneratePrivateKey()` + both `.String()` outputs.
2. Add `ecdhKeygen` using stdlib `crypto/ecdh` X25519
   (`ecdh.X25519().NewPrivateKey(clampedBytes).PublicKey().Bytes()` + `base64.StdEncoding`). This is the
   *reference/test* implementation that proves the seam is real and gives 1.5/1.4 a stdlib-anchored,
   wgctrl-free definition (and unblocks a future `js/wasm` Go oracle). It is **not** wired into any
   production caller.
3. Add `internal/localcompile/keygen_equivalence_test.go`: for 10k random private keys, assert
   `wgtypesKeygen.DerivePublic == ecdhKeygen.DerivePublic` AND `wgtypesKeygen.ParseAndNormalize ==
   ecdhKeygen.ParseAndNormalize` (the case-a re-canonicalization), incl. the clamp-bit edge cases
   (`key[0]&=248; key[31]&=127; key[31]|=64`). If `crypto/ecdh` diverges on a clamp edge case, switch
   `ecdhKeygen` to `golang.org/x/crypto/curve25519.X25519(clamped, basepoint)` (already an indirect dep,
   per MEMORY.md cache) and re-run (R1).
4. Refactor `render.GenerateKeys` (render.go:71) to accept a `Keygen` param; replace **all eight**
   `wgtypes.*` call sites (render.go:89,93,109,114,115,126,131,132) with interface calls across both
   custody branches:
   - AgentHeld: `ParseKey` (89) + `.PublicKey().String()` (93) → `kg.DerivePublic`.
   - AirGap case-a: `ParseKey`+`.String()` (109,114) → `kg.ParseAndNormalize`; `.PublicKey().String()`
     (115) → `kg.DerivePublic`.
   - AirGap case-c: `GeneratePrivateKey()`+`.String()`+`.PublicKey().String()` (126,131,132) →

   Keep a `GenerateKeys(topo, custody)` shim that injects `wgtypesKeygen` so all existing callers
   (handler.go:145,197,267; cmd/compiler/main.go:60; controller/compile.go:143) are unchanged and

   `GenerateKeys` keeps emitting `PrivateKeyPlaceholder` (render.go:99) and clearing the node's private
   key (render.go:97); `wgtypesKeygen` never returns a real private key to the AgentHeld caller.
5. Verify: `go test ./internal/render/...` (`signing_test.go`, `custody_diff_test.go`,
