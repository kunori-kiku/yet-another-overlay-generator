## Implementation steps
1. **Dep:** `cd frontend && npm install react-router-dom` (v7). Verify lockfile updated.
2. **Theme tokens — `src/index.css`:**
   ```css
   @import "tailwindcss";
   @custom-variant dark (&:where(.dark, .dark *));
   @theme { /* fonts, radii, easing tokens */ }
   :root { --accent: <teal>; /* + surface/hairline vars */ }
   :root.dark { --accent: <graphite>; }
   ```
   Refined teal (light) / graphite (dark) accent as CSS vars; hairline + surface vars for the Apple-minimal layer (full polish in P6).
3. **ThemeProvider — new `src/theme/ThemeProvider.tsx` + `useTheme`:** state `theme: 'system'|'light'|'dark'` (persisted, see step 6); effect applies/removes `.dark` on `document.documentElement`; when `system`, subscribe to `matchMedia('(prefers-color-scheme: dark)')`. Default `system`.
4. **Shell — new `src/components/shell/Shell.tsx` (+ `Sidebar`, `Topbar`, `UserMenu`, `ThemeToggle`):** flex layout — collapsible `Sidebar` (default expanded, fold button), `Topbar` (brand + breadcrumb placeholder + **top-right** `ThemeToggle` + `UserMenu` placeholder), `<main>` with `<Outlet/>`. Sidebar nav items are static placeholders this phase (wired to routes in P2). All strings via `txt()`.
5. **Router — `src/App.tsx`:** `createBrowserRouter([{ element: <Shell/>, children: [{ index: true, element: <AppLayout-content/> }] }])` + `RouterProvider`. The index route renders **exactly today's UI** (the existing `viewMode` dispatch stays intact for now, nested in MAIN). `ReactFlowProvider` stays where it is.
6. **Persist `sidebarCollapsed` + `theme`:** add to the appropriate store partialize (reuse `topologyStore`, which already persists `language`/`showInterfaces`, OR a tiny `uiStore`) — additive, optional, non-secret.
7. **i18n:** EN/ZH for any new chrome label (collapse, theme: system/light/dark).
