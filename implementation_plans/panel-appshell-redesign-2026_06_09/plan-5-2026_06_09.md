# plan-5 — httpOnly-cookie auth (cross-origin) + CSRF + CORS

See `outline.md` (M5). One session (may split → `plan-5.5`). **Risk: HIGH — the only backend change.**

## Goal
Make operator login survive a refresh by issuing an **httpOnly session cookie** (cross-origin-capable) with **double-submit CSRF**, alongside (not replacing) the existing **Bearer/break-glass** path. JS never reads the session; login state derives from a server probe. **Agent (machine-to-machine) routes are untouched** — no cookies, no CSRF, no credentialed CORS.

## Implementation steps (backend)
1. **Set cookies** — `mintSessionResponse`: add `setSessionCookie(w, plaintext)` (HttpOnly, `Secure=h.secureCookie`, `SameSite` per origin mode, `Path=/`, `Max-Age=` session TTL, name `yaog_session`); generate a CSRF token (32 B CSPRNG, base64url), set a **readable** `yaog_csrf` cookie, and also return `csrf_token` in JSON. Keep returning `session_token` (Bearer fallback).
2. **Clear cookies** — `HandleLogout`: emit both cookies with `Max-Age=-1`.
3. **Read + CSRF** — new `cookieToken(r)` in `auth_controller.go`: extract `yaog_session`; for state-changing methods, require `X-CSRF-Token` header `== yaog_csrf` cookie via `subtle.ConstantTimeCompare`. `operatorAuth`: try Bearer → cookie → 401 (Bearer stays primary fallback). **Do NOT** apply to agent routes.
4. **Credentialed CORS** — `cors()`: if `Origin` ∈ `panelOriginAllowlist`, reflect it + `Access-Control-Allow-Credentials: true` + `Vary: Origin` + allow `X-CSRF-Token`; **never** emit `*` with credentials. Non-allowlisted/absent origin → today's behavior. Add `panelOriginAllowlist []string` + `secureCookie bool` to the struct + setters.
5. **Settings field** — `ControllerSettings.Translucency bool` (default false) in `store.go`/`settings.go`; wire `settingsJSON` + `HandleSettings` GET/POST in `handler_bootstrap.go`. **Do NOT** inject it into the bootstrap script.
6. **Session probe** — add `GET …/session` (operator-auth) returning `{operator, expires_at}` so the SPA derives login state from the cookie (reuse an existing cheap authed GET only if cleaner).
7. **Env** — `cmd/server/main.go`: read `YAOG_PANEL_ORIGIN` (comma-split allowlist) + `YAOG_SECURE_COOKIE` (default true; false for dev); pass to handler.

## Implementation steps (frontend)
8. `controllerClient.ts`: `credentials:'include'` on `request()` + bare fetches; read `yaog_csrf` cookie and send `X-CSRF-Token` on state-changing calls; prefer same-origin/relative URLs when co-hosted.
9. `controllerStore.ts`: `selectLoggedIn` ← `GET /session` probe on mount (not a stored token); `login`/`logout` stop holding plaintext where the cookie suffices (Bearer/break-glass path preserved).

## Tests produced by this plan
- `internal/api/handler_login_test.go` (extend): cookie set on success; logout clears; CSRF mismatch → 403; credentialed-CORS reflects allowlisted origin + rejects others; **break-glass header still authenticates**. **Lifetime:** perpetual (security boundary). **Retirement:** never.
- `internal/api/handler_passkey_test.go` (extend): passwordless/2FA finish sets cookies. **Lifetime:** perpetual.

## Definition of done
- [ ] Login survives refresh (same-origin Docker + an allowlisted cross-origin).
- [ ] CSRF rejects header/cookie mismatch on state-changing requests.
- [ ] Non-allowlisted origin gets no credentialed CORS; `*`+credentials never emitted.
- [ ] Agent routes unaffected (no cookies/CSRF); break-glass Bearer still works.
- [ ] No token in web storage; Go `vet`/`test` green in CI; frontend lint/build green.
