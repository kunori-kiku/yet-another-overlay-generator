## Implementation steps

1. **`AgentUpdateSettings.tsx`** (co-located with BootstrapSettings), mounted in `SettingsPage.tsx`
   controller-only fragment after `<BootstrapSettings/>` (:151-156). Mirror the parent-loads /
   `key={JSON.stringify(settings)}` remount pattern.

2. **Fields:** `target_agent_version` (text, semver hint); `min_agent_version` (optional, under an
   "Advanced" disclosure, copy: "forces an update before applying any bundle"); a **per-arch bins editor**
   â€” fixed `linux-amd64` / `linux-arm64` rows of `{asset, sha256}` via the `field()` primitive (only the
   two certified arches); a **canary multiselect** sourced from `useControllerStore` `nodes[].nodeId`;
   a **promote-fleet-wide** toggle; a **read-only** echo of `githubProxy` + the composed proxy URL
   (single-source from settings â€” do NOT add a second editable copy; the proxy itself is edited in
   BootstrapSettings).

3. **Assist from GitHub release:** a button that calls `fetchPins({kind:'agent', version:
   target_agent_version || undefined})` and pre-fills the two bins rows for operator **review** (do NOT
   auto-save). Show explicit copy: *"Fetched over the GitHub proxy as a convenience â€” review before
   saving. Trust comes from the signed artifacts.json the agent verifies against, not this fetch."*
   Strip any path-qualified asset name to the bare filename (asset pattern forbids `/`). Handle
   loading/error states via `tError`.

4. **Promote-fleet-wide confirm:** flipping the toggle ON opens the amber confirm modal (model
   SettingsPage.tsx:99-146): "This rolls the target agent version to the ENTIRE fleet on the next
   deploy." Only on confirm does the value set. OFF needs no confirm (reversible).

5. **Client-side pre-validation** mirroring `validateAgentRollout` (semver target/min; `linux-<arch>`
   key; `[A-Za-z0-9._-]+` asset; 64-hex sha256; a non-empty target REQUIRES â‰Ą1 bin). Inline hints;
   treat the **server as authoritative** (its `error.req_field_invalid {field}` localizes on save).

6. **Local error banner:** `saveSettings` only sets the (unrendered) global error â€” add a `localError`
   + `localizeError` banner in the card so save failures are visible.

7. **Save copy:** "Saved. Changes reach nodes on the next Compile â†’ Stage â†’ Promote." (persist-only â€”
   no orchestration).

8. **i18n:** add the `agentUpdate.*` keys (labels, advanced disclosure, assist states, confirm modal,
   validation hints, save notice) to BOTH `en.ts` and `zh.ts` in lockstep.
