# plan-2 — Frontend data layer: full rollout+mimic contract + drop-on-save fix

See `outline.md`. Milestone: frontend foundation. **Prerequisites:** plan-1 (`done`) — the
`fetchPins` client wrapper targets plan-1's endpoint; the TS types should match the wire it added.

## Goal
Extend the TypeScript settings contract so the panel can READ and WRITE the five agent-rollout fields
and the mimic catalog fields, and **stop the full-replace `POST /settings` from silently wiping config**.
This is the type/mapping foundation both config cards (plan-3/4) and the status chip (plan-5) depend on.
**No UI in this plan.**

## Reads from specs
`controller-operator-api`, `panel-auth`.

## Read first
- `frontend/src/api/controllerClient.ts:615-664` (the `ControllerSettings` interface, `SettingsJSON`
  wire type, `mapSettings`, and `postSettings`'s body literal — the drop site).
- `frontend/src/components/deploy/BootstrapSettings.tsx:43-51` (empty-default literal), `:124-134`

## Implementation steps

1. **Shared pin type** (`frontend/src/types/controller.ts` or co-located): `export interface AgentPin
   { asset: string; sha256: string }` (mirrors `renderer.Artifact`).

2. **Extend `ControllerSettings`** (`controllerClient.ts:615-627`): add
   `targetAgentVersion: string`, `minAgentVersion: string`,
   `agentBins: Record<string, AgentPin>`, `agentCanaryNodeIds: string[]`,
   `agentRolloutFleetWide: boolean`, and the mimic fields
   `mimicVersion: string`, `mimicReleaseBase: string`, `mimicDebs: Record<string, AgentPin>`.

3. **Extend `SettingsJSON`** (`:629-636`) with the snake_case wire fields EXACTLY:
   `target_agent_version`, `min_agent_version`, `agent_bins`, `agent_canary_node_ids`,
   `agent_rollout_fleet_wide`, `mimic_version`, `mimic_release_base`, `mimic_debs`. Keep
   `agent_path_prefix` read-only (map it in, never send it).

4. **Extend `mapSettings`** (`:638-646`) with safe `?? ` defaults (`"" / {} / [] / false`) for every new
   field so an older/partial server record hydrates cleanly.

5. **Fix the drop-on-save (the load-bearing change):** rewrite `postSettings` (`:654-664`) to build its
   body from a **spread of the full current settings** plus the edited fields, sending ALL agent +
   mimic fields (never an omit-list literal). Convert `BootstrapSettings.tsx`'s empty-default (`:43-51`)
   and onSave (`:124-134`) literals to spread round-trips `{ ...initial, ...edited }` so editing a
   bootstrap field carries rollout + mimic config through untouched. Since the POST is full-replace,
   the body must include every persisted field.

6. **`fetchPins` client** (`controllerClient.ts`): `export async function fetchPins(cfg, body:
   { kind: 'agent'|'mimic'; version?: string; base?: string; assets: {key:string;asset:string}[] })`
   over the existing `request()` helper — POST, `credentials: 'include'`, the CSRF header the other
   mutating calls send, coded-error → `ControllerError` → `tError`. Returns `{ pins: Record<string,
   AgentPin>; base: string; version: string; proxyApplied: boolean }`.

- `postSettings` and `BootstrapSettings` save paths round-trip every field (drop-on-save fixed; smoke
  proves config survives an unrelated edit).
- `fetchPins` wrapper exists and type-checks. `npm run lint && npm run build` green.
