# Plan 2 — Phase 1a: split-render + custody mode

Outline: [outline.md](outline.md) · Prerequisites: Plan 1 merged (signing primitive) · Branch stacked
on Plan 1's.

## Goal

Let the compiler render a whole fleet **from public keys only**, emitting a `PRIVATEKEY_PLACEHOLDER`
in each node's own `[Interface]` so the controller never needs (or sees) a private key. The agent
(Plan 3) splices its locally-held key into the placeholder. The air-gap path is untouched and
byte-identical. This is the code half of the zero-knowledge custody decision.

Reads from specs: compiler/peer-derivation, data-model/node, security/security, controller/signing

## Implementation steps

1. **Custody-mode flag.** Add an explicit mode to the render entrypoint, e.g.
   `render.All(result, keys, opts RenderOptions)` with `opts.KeyCustody = AirGap | AgentHeld`
   (default `AirGap`, preserving every existing call site byte-for-byte). Thread it to the WG
   renderer only.
2. **`GenerateKeys` case-b becomes mode-aware** (`render.go:55-58`): in `AgentHeld` mode, a node with
   a public key but no private key is the **normal** path — return a KeyPair with
   `PrivateKey = PRIVATEKEY_PLACEHOLDER` (a sentinel) instead of erroring. In `AirGap` mode it stays
   a hard error, byte-identical. Case-a/case-c unchanged.
3. **WG renderer placeholder** (`wireguard.go`): when the private key is the sentinel, emit the
   literal `PRIVATEKEY_PLACEHOLDER` on the `PrivateKey =` line. Everything else (peer public keys,
   transit IPs, ports, MTU, Babel, sysctl, install.sh) renders unchanged — a node's private key
   appears in exactly one place, so this is the only edit.
4. **No-private-key CI guard** (`internal/render/custody_guard_test.go`): render a multi-node topology
   in `AgentHeld` mode and assert **no** emitted file contains a parseable WG private key (44-char
   base64 in a `PrivateKey =` line other than the sentinel). This is the perpetual guard against a
   split-render regression reintroducing the key vault.
5. **Byte-diff parity** (`internal/render/custody_diff_test.go`): the `AgentHeld` output equals the
   `AirGap` output for the same topology *except* each node's own `PrivateKey` line (placeholder vs
   real). Proves the only difference is the intended one.
6. **Spec** (`security.md`, `data-model/node.md`, new `docs/spec/controller/key-custody.md`): document
   AgentHeld custody, the placeholder contract, and that I5's guarantee holds via public-key persistence.
