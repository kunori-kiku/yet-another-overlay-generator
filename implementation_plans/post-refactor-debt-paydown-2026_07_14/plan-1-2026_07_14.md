## Goal
Close a confirmed HIGH signature-bypass: the standalone/manual/deploy-script `install.sh` reads the
mimic `.deb` pins from `artifacts.json` and `dpkg`-installs them **as root** without ever checking that
`artifacts.json` is covered by the signed `checksums.sha256`. The Go agent path closed this exact hole
(`verify.go:225-229`); the shell mirror ‚Äî the *sole* verifier on the standalone path ‚Äî did not.

**Attack (verified):** `install.sh` verifies `bundle.sig` over `checksums.sha256` then runs
`sha256sum -c` (which never flags a file *not listed* in the manifest). For a legitimately-signed bundle
that carries **no** `artifacts.json` (a supported config: a `tcp`/mimic link exists but no mimic catalog
was configured, on a host where distro mimic is unpackaged ‚Äî e.g. Debian 12), an attacker who can alter
the bundle at rest / in transit adds an unlisted `artifacts.json` with an attacker-chosen
`release_url`+`asset`+matching `sha256`. `sha256sum -c` passes (file unlisted), `bundle.sig` still
validates (checksums unchanged), and `_mimic_provision` downloads + `dpkg`-installs the attacker `.deb`
as root.

## Implementation steps
1. In `install.sh.tmpl`, after the signature/checksum verify (inside the `{{ if .HasSigning }}` block,
   or unconditionally whenever `checksums.sha256` exists), add a **presence-conditional** coverage guard
   BEFORE any `artifacts.json` read:
   ```sh
   if [ -f "$SCRIPT_DIR/artifacts.json" ]; then
       grep -qE '^[0-9a-f]{64}  artifacts\.json$' "$SCRIPT_DIR/checksums.sha256" \
         || { echo 'ERROR: artifacts.json present but not covered by signed checksums; refusing' >&2; exit 1; }
   fi
   ```
   (Match the exact `sha256sum` line format the manifest uses ‚Äî two spaces. Verify against a golden's
   `checksums.sha256`.) It must fail-closed under `set -e`.
2. Fix the false comment at `install.sh.tmpl:368` ‚Äî `artifacts.json` is integrity-verified above ONLY on
   the agent path; the standalone path relies on THIS guard. Reword to state the guard enforces coverage.
3. If `client-install.sh.tmpl` reads `artifacts.json`, mirror the guard there (the framework-refactor
   plan-6 lesson: a `replace_all` skips the client template ‚Äî apply deliberately to both).
4. Add a test proving refusal: extend the renderer/golden tests (or a shell-level assertion) so an
   injected **unlisted** `artifacts.json` in an otherwise-valid bundle causes `install.sh` to exit
   non-zero before reading any pin, while a **listed** one proceeds. If a conformance/golden fixture is
   the right home, regenerate to a REVIEWED byte-diff (only the guard lines added).

## Definition of done
- The standalone `install.sh` (and `client-install.sh` if applicable) refuses a present-but-uncovered

  still installs). The false comment is corrected. A test pins the refusal. Independent review ‚Üí fixes ‚Üí
  re-review clean ‚Üí CI green.
