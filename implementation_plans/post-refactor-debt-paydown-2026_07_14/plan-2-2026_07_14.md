## Goal
The framework-refactor made Go/WASM the **default and only** in-browser local engine (plan-4 flip,
plan-5 deleted the TS twin) — but no shipped artifact contains `yaog.wasm`, so every shipped panel's
local-design mode is dead-on-arrival, and CI cannot catch it (CI *does* pre-build the wasm for the
conformance gate, proving it **correct** while nothing proves it **shipped**). Fix the ship gap, make
the load path fault-tolerant, and stand up the orphaned cross-browser soak.

**Verified:** `web/yaog.wasm` + `frontend/public/yaog.wasm` are gitignored (`.gitignore:25-26`);
`frontend/public/` ships only 2 SVGs; `build`/`build:local` never call `build:wasm`; release.yml
`build-frontend` (`:167-210`) has no `setup-go`; the Dockerfile builds the panel in `node:20-alpine`
(no Go) and `COPY`s `dist → /app/web` without the wasm.

## Implementation steps
1. **release.yml `build-frontend`:** add `actions/setup-go@v5` (with `go-version-file: go.mod`) and a
   `run: npm run build:wasm` (or `./scripts/build-wasm.sh` from repo root) BEFORE `npm run build` +
   `npm run build:local`, so both `dist` and `dist-local` include `/yaog.wasm` + `/wasm_exec.js`.
2. **Dockerfile:** build `web/yaog.wasm` in the Go/`backend` stage (`GOOS=js GOARCH=wasm go build -o
   web/yaog.wasm ./cmd/wasm` + copy the toolchain `wasm_exec.js`) and `COPY --from=backend` those into
   `frontend/public/` (or into `/app/web` after the node build) BEFORE the panel is served. Keep the
   node stage's `npm run build` but ensure `frontend/public/{yaog.wasm,wasm_exec.js}` exist at build
   time. (Choose the cleanest layering; verify the final image serves `/yaog.wasm`.)
3. **Red-build ship assertion:** after each build, assert `dist/yaog.wasm`, `dist/wasm_exec.js`,
   `dist-local/yaog.wasm` exist (a `test -f … || exit 1` step, or `if-no-files-found: error` on a
   dedicated upload) so a future regression FAILS the release instead of 404-ing at runtime (invariant
   3 — the exact gap that shipped this).
4. **Fault-tolerant load:** in `wasmEngine.ts` `ensureWasm()`, reset `loadPromise = null` when the
   promise rejects (so a transient fetch blip does not permanently brick local compute for the session):
   e.g. `loadPromise = loadWasm().catch((e) => { loadPromise = null; throw e })`. Apply the same to the
   sibling module-load promise in `lib/localEngine.ts`. Add a recovery vitest (first call rejects →
   second call re-attempts + succeeds).
5. **Soak job:** make the `YAOG_WASM_SOAK` webkit+firefox projects a real job — either a scheduled
   workflow (`workflow_dispatch` + cron) that `npx playwright install webkit firefox` then runs with
   `YAOG_WASM_SOAK=1`, or a documented-manual runbook step. Correct the "automated stop-loss for
   deleting the TS compiler" comments (`playwright.config.ts`) to match reality.
6. **Comment hygiene:** fix `wasmEngine.ts:6` (`compiler/localEngine.ts` → `lib/localEngine.ts`) and
   `:49-51` (drop "the default (TS) build never depends on them" — there is no TS build; reframe as
   "the wasm assets must be present whenever the panel is served").

## Definition of done

  re-review clean → CI green.
