## Goal
`deploy.go` (686 LOC) is an inferior *second implementation* of the node-teardown that `install.sh`
already owns — and it drifted in a HIGH-severity way: `deploy-all --uninstall` **never tears down
mimic** (leaving an orphaned root eBPF XDP/TC program + boot-persistent `mimic@` service after the
operator believes the overlay is gone), and it uses a **hard-coded `10.10.0.0/24` exact-match** SNAT
delete (`deploy.go:310`) that `install.sh` was fixed away from (D38/D39), leaving a stale POSTROUTING
source-rewrite on custom-transit or IP-changed nodes. It is also the last hand-built dual-shell
(`bash` + PowerShell) `fmt.Sprintf` string builder (~18 positional verbs) outside plan-6's
`ShellToken`/`go:embed` regime (the deferred plan-6.5).

## Implementation steps
**Part A — teardown correctness (the HIGH fix; do this even if Part B splits to plan-3.5):**
1. Add mimic teardown to the `--uninstall` builder mirroring install.sh: stop + disable every
   `mimic@<egress>` unit, detach the XDP/TC program, remove `/run/mimic/*` + the unit files. Gate on the
   node actually having had mimic (a `HasMimic`-class flag), matching install.sh's conditionality.
2. Replace the hard-coded `-s 10.10.0.0/24` SNAT delete with the transit-CIDR-aware deletion install.sh
   uses — resolve the node's transit pools via the same `NodeTransitCIDRs` path so a custom-transit or
   IP-changed node's rule is actually removed (belt-and-suspenders with the `nft delete table` line).
3. New fixture pinning the corrected teardown (mimic + per-CIDR SNAT) — a deliberate, reviewed byte
   change isolated to those lines.

**Part B — templating (finish plan-6.5):**
4. Convert the `deploy.go` `fmt.Sprintf` builders (`deploy-all.sh` + the PowerShell variant + the inline
   uninstall script) to `go:embed` `deploy/*.sh.tmpl` + `deploy/*.ps1.tmpl`, reducing `deploy.go` to a
   config-struct builder. Route every user-controlled field through `ShellToken` (`ShellQuoted` for
   quoted contexts, `ShellRaw` for validated CIDRs) — and extend `field_safety_test` to the new template
   fields. The non-teardown output must be byte-identical to today's (reviewed diff).
5. If the dual-shell (`bash` + PowerShell) seam does not fit `ShellToken` cleanly, STOP and open plan-3.5
   (characterize the PowerShell escaping context; do NOT force a lossy conversion). **Part A must not
   wait on Part B** — split if needed.
