# plan-6 — Identity reconciliation (design ↔ fleet)
<!-- drafted: 2026-06-12 -->
Outline: [`outline.md`](outline.md) · Prerequisites: plan-3 (orphan agents idle safely), plan-4 (hydration makes the panel's design view trustworthy)

## Goal

Surface and police the node-ID↔fleet binding that today silently severs: fleet rows whose
node-ID is absent from the current design get a visible "not in design" marker and a one-click
(manual, never automatic — D10) revoke from a deploy-summary orphan list; enrollment refuses a
WG public key already approved under a different node-ID (the duplicate-fleet-rows vector);
minting an enrollment token for a node-ID absent from the stored design warns (not blocks).

Reads from specs: panel-deploy-fleet, controller-agent-api, controller-operator-api

## Implementation steps

1. **Enrollment dedupe (server)**: in the enroll ceremony, before minting the API token: if any
   approved node has the same `WGPublicKey` under a DIFFERENT NodeID → 409
   `"this WireGuard public key is already enrolled as node <id>; revoke it first or reuse that node id"`.
   Same-ID re-enroll (fresh token, e.g. reinstalled host) stays allowed. Audit entry on refusal.
2. **Token-mint design check (server)**: `HandleEnrollmentToken` loads the stored topology
   (GetTopology; 404 ⇒ skip check) and, when the requested node-ID is absent, includes
   `"warning": "node-id not present in the stored design; it will be skipped at stage until added"`
   in the 200 response (warn-not-block: operators may pre-mint).
3. **Fleet markers (panel)**: NodeRegistry computes orphan-ness per row (registry node-IDs vs
   `useTopologyStore` node-IDs — both already client-side post-hydration) and renders a
   "not in current design" badge + the existing revoke affordance prominently on such rows.
4. **Deploy-summary orphan list (panel)**: after a successful deploy(), the summary panel lists
   orphaned fleet rows ("enrolled but not deployed to: …") with per-row revoke buttons reusing
   the existing `revoke(nodeId)` action + confirm. Never auto-revoke.
5. **Mint-time warning surfaced** in EnrollmentFlow (show the server warning string).
6. i18n; branch `plan-6-identity-reconciliation`; per-substep commits; PR.

## Verification

CI for the Go changes; lint+build for the panel. Manual: enroll node A; delete it from the
design; deploy → summary lists A as orphaned; Fleet shows the badge; revoke works. Enroll a
second machine reusing A's pubkey under id B → 409. Mint a token for an undesigned id → warning
shown in EnrollmentFlow.
