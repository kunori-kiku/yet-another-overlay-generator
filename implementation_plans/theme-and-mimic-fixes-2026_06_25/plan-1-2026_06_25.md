## Implementation steps

### Step 1 — BUG1: node Conditions chip tokens (`nodeConditions.ts`)

Replace the dark-only `STATUS_CLASS` (`nodeConditions.ts:13-18`) with the semantic status families,
mirroring `UpdateStatusChip.tsx:11-19`:
- `ok` → `bg-[var(--success-bg)] text-[var(--success)] border-[var(--success-border)]`
- `warn` → `--warning-*`
- `error` → `--danger-*`
- `unknown` → `--info-*`

`conditionVisual` (`:22-24`) is unchanged in shape. The conformance test
`frontend/src/lib/nodeConditions.conformance.test.ts:9-20` only asserts the 4 statuses are
non-empty/distinct/total — it stays green with NO change. Verify the four consumers
(`NodeConditions`, `MimicConditionChip`, `NodeRegistry`, `FleetNodeDetailPage`) inherit the fix
(they all funnel through `conditionVisual`).

### Step 2 — BUG3: `--cta` token family + repoint primary CTAs

Add to `frontend/src/index.css`:
- `:root` → `--cta: #0d9488; --cta-hover: #0f766e; --cta-fg: #ffffff;` (equals today's light accent
  — light mode is visually unchanged).
- `:root.dark` → `--cta: #14b8a6; --cta-hover: #2dd4bf; --cta-fg: #06241f;` (vivid teal; dark CTAs go
  grey→teal). Document the family in the token-vocabulary comment block (`index.css:17-31`).

Repoint ONLY genuine primary-action FILLS to `--cta` (leave focus rings, headings, brand, toggles,
input focus-borders on `--accent` — that is the graphite identity):
- `DeployBar.tsx:135` Deploy: `bg-[var(--accent)] hover:bg-[var(--accent-hover)] text-[var(--accent-fg)]`
  → `bg-[var(--cta)] hover:bg-[var(--cta-hover)] text-[var(--cta-fg)]` (keep `data-testid="deploy"`).
- `LocalDeploy.tsx:34` Compile (local-mode primary CTA) → same `--cta` mapping.
- Add a `BTN_CTA` const to `frontend/src/components/shell/styles.ts` next to `FOCUS_RING` and use it
  in both sites so the two CTAs cannot drift. (Structure-aware: matches how the shell centralizes
  shared class strings.)

Do a final `grep` for other `bg-[var(--accent)]` fills on genuine primary submit buttons; repoint any
that are unambiguously primary CTAs. Do NOT touch `text-[var(--accent)]` (headings) or
`border`/`ring`/`focus:` accent usages.

### Step 3 — BUG2: canvas neutral-map + ROLE_HUE dedup

Keep categorical hues (edge-type stroke, node role border/handle) — they carry meaning. Map only the
neutral chrome and make label text legible in both modes:
- `TopologyCanvas.tsx:638` `<Background color="#374151" />` → `color="var(--hairline)"`.
- `CustomEdge.tsx` label pill: bg (`:134`) → `var(--surface-elevated)`; text (`:135`) →
  `var(--content)`; keep border `colors.stroke` (`:136`); soften shadow (`:142-144`) to
  `rgba(0,0,0,0.15)`. Backup-chip text (`:189-192`) + port-chip text (`:205-207`) → `var(--content)`
  (keep the hue-tinted bg). Primary `★` (`:150-161`) → `var(--warning)` (theme-aware "gold"). The
  duplicate `⚠` chip (`:162-180`) → warning family (`--warning-bg`/`--warning`/`--warning-border`) —
  it is a status chip.
- `CustomNode.tsx`: iface chip (`:132-140`) → drive **text** from `var(--content)` (keep hue bg);
  bump light-mode role fill alpha so role tints read on light (`/15`→`/20–25`); selection ring
  (`:105`) `ring-[var(--hairline)]` → `ring-[var(--cta)]` (stronger, theme-aware).
- **ROLE_HUE dedup:** extract one shared role→hue map (e.g. `frontend/src/components/canvas/roleHue.ts`)
  consumed by `CustomNode` role border + handle AND `TopologyCanvas.tsx:663-672` MiniMap `nodeColor`,
  so the three stop drifting. This is the cheap structural win from D6.

### Step 4 — E2E + verification

- Re-grep `frontend/e2e` for any color-class locator touching the changed classes; convert to
  `data-testid` (Deploy already has `data-testid="deploy"`; add ids if a spec depends on a changed
  class). Run the relevant e2e specs locally if feasible.
- `cd frontend && npm run build` (tsc -b — the stricter gate) + `npm run lint` + `npx vitest run`.
- Manual: toggle light/dark, verify node-condition chips legible in light, canvas grid + edge labels
  themed both modes, Deploy vivid teal in dark.
