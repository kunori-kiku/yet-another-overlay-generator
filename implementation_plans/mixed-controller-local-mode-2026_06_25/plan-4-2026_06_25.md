# plan-4 — On-box kit (`yaog-agent kit`: keygen → descriptor → register → splice)

**Subject:** mixed-controller-local-mode-2026_06_25 · **Track:** Go/agent · **Depends on:** plan-2
(registration endpoint), plan-3 (downloadable signed bundle).

## Goal

The Option-C ergonomics: a single on-box command that makes deploying a manual node nearly as easy as
an agent enroll, WITHOUT installing a polling agent. It generates the keypair on-box (private key
never leaves), emits a **descriptor** the operator registers (or registers directly if given
controller creds), and splices the private key into the controller-signed bundle's
`PRIVATEKEY_PLACEHOLDER` so `install.sh` runs hands-off.

## Implementation steps

### Step 1 — `yaog-agent kit` subcommand (or a dedicated small binary)

A new subcommand on the existing agent binary (no new dependency; stdlib + wgctrl keygen already
present). Pipeline:
1. **keygen** — generate a WG keypair on-box, write the private key 0600 (reuse `EnsureKey`), print
   the public key.
2. **descriptor** — emit `{NodeID, WGPublicKey, Endpoint}` (JSON + a copy-paste line) for the operator
   to register in the panel (plan-6) OR, if given a controller URL + an operator-minted manual-node
   token, POST it directly to `POST /api/manual-node` (plan-2). Keep the token path optional —
   air-gapped manual nodes use the print-and-paste path.
3. **splice** — given the downloaded signed bundle (plan-3), splice the on-box private key over the
   `PRIVATEKEY_PLACEHOLDER` in the bundle's `install.sh` (exactly the substitution `install.sh` itself
   does for an agent at `script.go:714-742`), so a manual operator runs `sudo bash install.sh` with no
   further steps.

### Step 2 — No polling, no bearer custody

The kit MUST NOT install or start the polling agent, MUST NOT pull `/config`, and MUST NOT write a
long-lived bearer token unless the optional reporter (plan-5) is explicitly enabled. It is a one-shot
provisioning helper. Keep its surface minimal and its private-key handling identical to the agent's
(0600, on-box only).

### Step 3 — Idempotence + safety

Re-running the kit on a node must not clobber an existing private key without an explicit `--force`
(mirror the agent's key-handling discipline). The descriptor is deterministic from the on-box key.

## Tests produced

- `cmd/agent` / `internal/agent` — kit keygen writes 0600, descriptor shape is `{NodeID, pubkey,
  endpoint}`, splice replaces exactly the placeholder and nothing else, re-run without `--force` is a
  no-op — permanent.
- `internal/regression` — the kit never transmits the private key anywhere (descriptor + any direct
  register call carry only the public key) — permanent (load-bearing).

## Definition of done

- One on-box command provisions a manual node end-to-end (keygen → descriptor/register → splice) with
  the private key never leaving the box.
- `go test -race ./...` + vet green; `bash -n` on the spliced `install.sh`.
- Independent review (4-lens + security HIGH) → fix → re-review → CI green → merged.

## Out of scope

- The optional telemetry-only reporter (plan-5) — the kit only enables it on explicit opt-in.
- The panel registration/download UI (plan-6).

## Hazard

The kit handles the manual node's PRIVATE key. Adversarially verify it is written 0600, never logged,
never sent over the wire, and the splice can't be tricked into writing the key to the wrong file or
leaving it in a world-readable temp.
