# plan-3 — Manual-node signed bundle production + per-node download

**Subject:** mixed-controller-local-mode-2026_06_25 · **Track:** Go (+ FE-light) · **Depends on:**
plan-1, plan-2.

## Goal

The controller compiles AND signs a manual node's per-node bundle exactly like a managed node's
(same `CompileAndStage` pipeline, same off-host signature), and exposes an operator-gated way to
**download** that signed bundle for hand-delivery — since a manual node has no agent to pull `/config`.

## Implementation steps

### Step 1 — Render + sign the manual node's bundle

Ensure `CompileAndStage` produces a signed per-node bundle for manual nodes (now that plan-1 admits
them). The bundle is identical in shape to a managed node's served bundle: per-node configs +
`checksums.sha256` + the membership trust-list + signature, with `PRIVATEKEY_PLACEHOLDER` in
`install.sh` (the operator/kit splices the real private key on-box — custody). The bundle digest is
what plan-2's manifest binds.

### Step 2 — Per-node download endpoint

Add an operator-gated `GET /api/manual-node/{nodeID}/bundle` (or `POST .../bundle` returning a ZIP)
that returns the manual node's signed bundle as a downloadable archive (mirror the air-gap export ZIP
shape, `export.go`). Operator-authed only; never on the agent/node mux. SSRF/path-safety: `nodeID`
validated against the registry; no arbitrary path access.

### Step 3 — FE-light wiring (the API client; full UI in plan-6)

- `frontend/src/api/controllerClient.ts` — `downloadManualNodeBundle(nodeId)`.
- `frontend/src/stores/controllerStore.ts` — a thin action. (The button + manual-node UX lands in
  plan-6.)

## Definition of done

- A manual node has a controller-compiled, off-host-signed bundle whose digest is in the membership
  manifest; an operator can download it; no private key in it.
- `go test -race ./... && go test -tags airgap ./...` + vet green; `internal/localcompile` golden
  contract green.
- Independent review (4-lens + security HIGH) → fix → re-review → CI green → merged.

## Out of scope

- The on-box kit that consumes the bundle + splices the key (plan-4); the full download UI + manual
  node panel UX (plan-6).

## Hazard

The download endpoint serves a signed bundle to an operator. Verify it is operator-gated (not the
node mux), validates `nodeID`, and cannot be used to exfiltrate a managed node's served config or any
private key.
