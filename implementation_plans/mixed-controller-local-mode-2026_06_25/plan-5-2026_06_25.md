# plan-5 — Optional telemetry-only reporter for manual nodes

**Subject:** mixed-controller-local-mode-2026_06_25 · **Track:** Go/agent · **Depends on:** plan-2
(registration / a bearer for the manual node).

## Goal

Give manual nodes optional health visibility WITHOUT making them poll `/config` or auto-apply:
a stripped reporter that only POSTs `/telemetry` (conditions + metrics, bearer-authed). This closes
the "manual/unmonitored" gap (owner D3) — manual nodes can show conditions in the panel but are still
excluded from the convergence rollup (they never pull/apply, so there is no generation to converge).

## Implementation steps

### Step 1 — A reporter mode that is telemetry-only

Add an opt-in mode (a `yaog-agent report` / `--telemetry-only` flavor, or a flag on the kit) that runs
ONLY the heartbeat goroutine (`runHeartbeat`) against `/telemetry` — no `/poll`, no `/config`, no
apply, no self-update. It reuses the existing `Sampler` framework + `Telemetry()` client call; it
needs a bearer token for the manual node (issued at registration, plan-2, on opt-in only).

### Step 2 — Server accepts manual-node telemetry; convergence excludes manual

- `HandleTelemetry` already authenticates by bearer + records conditions without touching generation
  state — confirm it works for a manual node's token (no `AppliedGeneration` exists for it, which is
  correct; telemetry carries no generation). Ensure `RecordTelemetry` on a manual node updates
  Conditions/LastSeen without requiring an enrolled record.
- Convergence: the "fleet converged" rollup MUST exclude manual nodes (D3) — they have no
  desired/applied generation to compare. Make the exclusion explicit and tested, both server-side
  mapping and the panel derivation (the panel piece coordinates with plan-6).

### Step 3 — Manual node never re-pins / never self-updates

The reporter MUST NOT carry any of the apply/self-update machinery. It only samples + posts. Keep its
surface minimal so a manual node can never be pushed a config it didn't ask for.

## Definition of done

- A manual node can optionally report health via `/telemetry`; its conditions show in the panel; it
  is excluded from convergence.
- The reporter has no apply/self-update surface.
- `go test -race ./...` + vet green.
- Independent review (4-lens + security HIGH) → fix → re-review → CI green → merged.

## Out of scope

- The "manual/unmonitored" chip + the convergence-exclusion UI (plan-6 owns the panel; this plan owns
  the server/agent side + the data the panel reads).

## Hazard

A manual node holding a bearer token is a (small) new credential surface. Verify the token is scoped
to telemetry only (cannot pull `/config` or trigger apply) and is opt-in (default OFF — an air-gapped
manual node has no reporter and no token at all).
