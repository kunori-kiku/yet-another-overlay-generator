# plan-1 — Model `deployment_mode` + compiler admission

**Subject:** mixed-controller-local-mode-2026_06_25 · **Track:** Go · **Depends on:** none (foundation).

## Goal

Introduce the orthogonal `deployment_mode: "managed" | "manual"` node field and teach the compiler's
admission chokepoint to render a **manual** node from its topology-carried identity (pubkey +
endpoint) instead of an enrollment registry record — keeping its edges — without altering managed-node
behavior or zero-knowledge custody.

## Implementation steps

### Step 1 — The model field

- `internal/model/topology.go` — add `DeploymentMode string \`json:"deployment_mode,omitempty"\`` to
  `Node` with constants `DeploymentManaged = "managed"` / `DeploymentManual = "manual"` and a helper
  `(n Node) IsManual() bool` (`n.DeploymentMode == DeploymentManual`). `omitempty` + empty==managed
  preserves wire/back-compat for every existing topology (mirror `Edge.Role` empty-default at
  `topology.go:131-137`).
- Schema validation (`internal/validator/schema.go`) — accept only `""`/`managed`/`manual`.

### Step 2 — `enrolledSubgraph` admits manual nodes

In `internal/controller/compile.go:477-532`, change admission so a node is "ready" when **either**
(a) it has an enrolled registry record (`NodeApproved && WGPublicKey != ""`, unchanged) **or**
(b) `node.IsManual()` AND it carries a topology pubkey (`WireGuardPublicKey != ""`) + a usable
endpoint:
- For a manual node, stamp `node.WireGuardPublicKey` from the **topology** (not the registry) and keep
  `WireGuardPrivateKey` **cleared** (`:520` invariant holds — custody).
- Do NOT drop a manual node (`:513-522`) and do NOT drop edges touching it (`:524-529`).
- Exclude manual nodes from `SkippedUnenrolled` (`:64-66`) — they are intentionally agent-less, not
  "not yet ready".
- Structure: factor the readiness predicate into a small named helper (e.g. `nodeReadyForCompile`)
  so managed-vs-manual admission is one clear branch, not scattered conditionals.

### Step 3 — `GenerateKeysWith` manual branch

In `internal/render/render.go:159-185`, the controller path is `AgentHeld`. For a manual node, source
the pubkey from the (now topology-stamped) `WireGuardPublicKey` exactly as today, keep the
`PrivateKeyPlaceholder` + clear-private invariant (`:180-184`). The manual operator splices the real
private key by hand / via the kit (plan-4), identical to the agent splice. Hard-error remains if a
manual node has no pubkey (`CodeKeygenMissingPubkey`) — that becomes the validator rule below.

### Step 4 — Validator rule

`internal/validator/semantic.go` — a new rule: a `manual` node MUST carry a `WireGuardPublicKey`
(no enrollment to supply one), and if it must accept inbound (role/capability implies a listening
peer) it MUST carry a reachable `PublicEndpoints`/edge endpoint. Emit a structured error
(register a code in the validator/`apierr` registry — no orphan codes) so the panel can localize it.

### Step 5 — Confirm `peers.go` needs no change

Add a focused test proving a manual node with a topology pubkey + endpoint renders as a normal peer
(transit IP, allowed-IP, `[Peer]` block) in a managed neighbor's config, and the managed node renders
as a peer in the manual node's bundle. `peers.go` is downstream of the subgraph projection; once the
node + edges survive `enrolledSubgraph`, it should "just work" — assert it, don't modify it.

## Tests produced

- `internal/model` — `IsManual` + JSON round-trip (omitempty, empty==managed) — permanent.
- `internal/controller/compile_test.go` — manual node admitted from topology identity; edges kept;
  excluded from `SkippedUnenrolled`; managed nodes unaffected; a manual node with no pubkey is
  rejected by the validator (not silently dropped) — permanent.
- `internal/render/render_test.go` — AgentHeld manual branch keeps private cleared/placeholder — permanent.
- `internal/compiler` — manual peer renders bidirectionally (Step 5) — permanent.
- `internal/regression` — a zero-knowledge guard: no private key ever appears in a manual node's
  compiled/served output — permanent (load-bearing).

## Definition of done

- `deployment_mode` is a first-class orthogonal field; managed behavior byte-unchanged for existing
  topologies (back-compat test).
- A manual node + its edges survive `enrolledSubgraph` and render bidirectionally; private key never
  present; absent from `SkippedUnenrolled`.
- Validator enforces manual-node pubkey/endpoint.
- `go test -race ./... && go test -tags airgap ./...` + `go vet` (both tags) green.
- Independent review (4-lens + security HIGH) → fix → re-review → CI green → merged.

## Out of scope

- The registration ENDPOINT + custody gate relax (plan-2) — this plan only changes the COMPILER's
  treatment of a node already marked manual with a topology pubkey. How that pubkey gets there
  (operator paste / kit register) is plan-2/plan-6.
- Keystone membership scope (plan-2), bundle download (plan-3), kit (plan-4), reporter (plan-5), FE
  (plan-6).

## Hazard

The admission change is the security-critical core. A bug that admits a node it shouldn't, or that
lets a private key survive, is a custody regression. Review weight = HIGH; the `internal/regression`
zero-knowledge guard is mandatory before merge.
