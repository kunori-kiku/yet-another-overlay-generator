# plan-11 — Controller-mode boundary: UI mode-gating (both directions)

**Milestone:** M11 (see `outline.md`). **Branch:** `mode-boundary-plan-11`. **One PR, independent
review before merge.** **Depends on plan-10** (uses `saveDesign`, `isDesignDirty`, `saveConflict`).

Reads from specs: panel-shell, panel-design, panel-deploy-fleet, panel-auth.

## Goal

Every UI affordance renders only in the mode where it is meaningful — neither a local-mode
construct in controller mode nor a controller-mode construct in local mode.

## Implementation (shipped)

### T3 — Topbar I/O cluster → local; controller-mode Save UI (commit a569b1b)
- `Topbar.tsx`: gate the import/export/flush cluster to `mode === 'local'` (local file-I/O; in
  controller mode flush is a false "cannot be undone", export writes fleet IPs/SSH to disk, import
  flips the mirror to local-only).
- `CanvasToolbar.tsx`: controller-mode **Save** button (calls plan-10 `saveDesign()`), disabled
  when `!isDesignDirty`, with a live Save/Saved indicator; plus the save-conflict dialog (re-sync
  from server via `hydrateFromServer` / force-overwrite / cancel). New `canvasToolbar.*` i18n keys.

### T4 — hide local-only constructs in controller mode (commit 15ae7b2)
- `NodeEditor.tsx`: gate the "Pin private key (persist)" toggle + status panel to local mode.
- `SecurityPage.tsx`: mode-split — controller shows 2FA/Passkey/AuditLog; local shows AuditView
  ("Compile History"). (Also resolves the T5 reverse leak for 2FA/Passkey/AuditLog in local, and
  makes `clearHistory` — inside AuditView — local-only.)
- `DeployPage.tsx`: render `CompilePreview` only in local mode.
- Validate stays in both modes (D11).

### T5 — hide controller-only constructs in local mode (commit 257b2c8)
- `App.tsx`: `RequireControllerMode` route guard on `/overview`, `/fleet`, `/fleet/nodes/:id`
  (deep-link reachability now matches nav visibility; redirect to local landing otherwise).
- `SettingsPage.tsx`: gate ConnectionSettings + BootstrapSettings to controller mode (LoginPage
  still carries connection fields, so entering controller isn't stranded).
- `controllerStore.switchToLocal`: clear the persisted fleet-view cache (nodes/audit/lastDeploy)
  on the local boundary. Session preserved (switch != logout).

## Verification gate

`cd frontend && npm run lint && npm run build` green (no Go changes). Per-mode spot check: no
local-only control in controller, no controller-only control in local; Save drives `saveDesign`;
`/fleet` deep link in local mode redirects.
