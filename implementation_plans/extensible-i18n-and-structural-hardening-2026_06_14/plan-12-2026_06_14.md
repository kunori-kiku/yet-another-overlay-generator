## Problem (T6)

`/api/health`, `/api/validate`, `/api/compile`, `/api/export`, `/api/deploy-script` are registered
unconditionally on `s.mux` (the operator/panel port). In a controller deployment that port also
serves the operator routes â€” so the compute routes are an **unauthenticated compute / key-gen
oracle + DoS surface**. The server had no mode concept (the only signal is whether
`EnableController` was called, which happens *after* `registerRoutes`).

## Implementation (shipped)

### Backend â€” request-time operator-auth gate (`internal/api/server.go`)
- Add `Server.operatorAuth func(http.HandlerFunc) http.HandlerFunc` â€” nil in air-gap, armed by
  `EnableController` (`s.operatorAuth = ch.operatorAuth`).
- `gateAirgap(h)`: at REQUEST time, if `s.operatorAuth` is armed, wrap `h` with it; else
  passthrough. (Request-time because `EnableController` runs after `registerRoutes`.)
- `registerRoutes`: `/api/health` stays public (liveness); the four compute routes are wrapped
  `recoverPanics(cors(gateAirgap(h)))` so a 401/403 still gets CORS headers.
- `operatorAuth` (existing, `auth_controller.go`) accepts a Bearer token (session or break-glass,
  CSRF-exempt) OR the httpOnly session cookie + double-submit `X-CSRF-Token` for POST.

### Frontend â€” authenticate the kept Validate call (`stores/topologyStore.ts`)
- `compile`/`export`/`deploy-script` are already refused client-side in controller mode (plan-4),
  so the panel never calls them there â€” no change needed. `validate` is KEPT in controller mode
  (D11), and the route is now gated, so `validate()` sends operator credentials in controller mode:
  `Authorization: Bearer (sessionToken||operatorToken)` when an in-memory token exists, else relies
  on the same-origin session cookie + `X-CSRF-Token: csrfToken` (mirrors `controllerClient.configOf`).
  Local mode sends nothing (route is public).

## Tests produced by this plan

`internal/api/airgap_auth_gate_test.go` (subject-scoped):

- controller mode (`NewServer` + `EnableController`): each compute route is 401 without auth and
