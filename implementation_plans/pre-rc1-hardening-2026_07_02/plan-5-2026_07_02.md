# plan-5 — agent-route DoS hardening

> See `outline.md` (§2 Principle "Observability is neither a DoS vector nor a custody path", §7 plan-5).
> Four findings; build this BEFORE plan-9/plan-10 (all touch `/telemetry` + `RecordTelemetry`).

## Prerequisites
Ships toward rc.1 (after beta.17). Precedes plan-9/plan-10.

## Goal
An authenticated node cannot DoS the controller via `/telemetry` (or other agent routes); the per-beat
fsync'd whole-record rewrite is removed from the telemetry path without weakening custody durability;
per-IP limits stop collapsing behind a reverse proxy.

## Implementation steps

### F1 — per-node (bearer-identity) rate limit on the agent mux
Reuse `loginLimiter` as a fixed-window request-rate limiter (`registerAttempt` WITHOUT `succeed()`).
Add `nodeLimiter *loginLimiter` to `ControllerHandler`, constructed as `newLimiter(60, time.Minute)`
(consts `maxNodeRequestsPerWindow=60`, `nodeRateWindow=1*time.Minute` — ~10× the healthy peak: heartbeat
2/min + poll 1-2/min + deploy burst; still caps a flood to ~1 req/s). Gate in `requireNode` AFTER
`authenticateNode` succeeds, keyed `"node:"+auth.node` (survives the proxy IP collapse; isolates one
abusive node). On `!allowed`: set `Retry-After` + `writeAPIError(w, apierr.New(CodeNodeRateLimited))`.
`/enroll` stays per-IP (unchanged). **New apierr code `CodeNodeRateLimited` (429)** for precise
semantics (log/metric distinguishable) → const+registry in `apierr.go` (`TestRegistryBijection`) + drift
regen + en/zh i18n. (Zero-churn fallback if the owner prefers: reuse `CodeAuthRateLimited`.)

### F2 — bound conditions + metrics at the HTTP boundary
Shared helpers called after `decodeJSON`, before the store call:
- `validateConditions(conds)`: reject if `len > maxReportedConditions` (**32**) or any `Message` exceeds
  `model.ConditionMessageMax` runes → `apierr.New(CodeReqFieldInvalid).With("field","conditions")`.
- `validateMetrics(metrics)`: reject if `len > maxTelemetryMetrics` (**32**) or `Σ len(v) >
  maxTelemetryMetricsBytes` (**64 KiB**) → `.With("field","metrics")`.
Wire into `HandleReport` (conditions) + `HandleTelemetry` (conditions + metrics). Reuse
`CodeReqFieldInvalid` (**no new code**). Backstop: truncate to a generous ceiling (64) in
`stampConditions` so no future caller allocates unbounded (documented; handler already rejects at 32).
**Coordination (plans 9/10):** the caps must budget for the `resource` metric key (plan-10) and NEVER
drop the terminal `selfupdate:Abandoned` condition (plan-9); with a 32-cap and the agent's own
`classify()` capping conditions, a legit agent never approaches the limit — over-limit is REJECTED, not
silently truncated.

### F3 — decouple telemetry durability (no fsync per beat), FileStore only
Interface UNCHANGED; MemStore UNCHANGED. In `FileStore` add `telemetryMu sync.Mutex` + a volatile
overlay `map[TenantID]map[string]*volatileTelemetry{lastSeen,lastSeenSet,conditions,telemetry,
agentVersion,telemetrySet}`.
- `RecordTelemetry`: `os.Stat(nodePath)` existence (→`ErrNotFound` preserving the contract), then set the
  overlay under `telemetryMu` (no `writeJSONAtomic`, no `fs.mu`).
- `TouchLastSeen`: same treatment (removes fsync from `/config`+`/poll` too — same observability class).
- Merge-on-read in `GetNode`/`listNodesLocked`: after the durable read, override LastSeen/Conditions/
  Telemetry/LastAgentVersion from the overlay when set; **deep-copy** into the returned `Node`. Lock
  order: hold `fs.mu` then take `telemetryMu` (beat paths take only `telemetryMu` → no cycle).
- Coherence: `SetAppliedGeneration` keeps writing custody+conditions durably AND refreshes the overlay
  so a stale beat never shadows a fresh report.
- Optional (recommended, no new goroutine): piggyback — fold the overlay fields into any record a custody
  write already rewrites (`UpsertNode`/`SetAppliedGeneration`/`PromoteStaged`/token ops) at zero extra
  fsync, bounding cold-start staleness.

### F4 — trusted-proxy-gated real client IP
Add `trustedProxies []net.IPNet` to `ControllerHandler` (default empty = today's behavior) +
`SetTrustedProxies([]string)` (CIDR or bare IP; skip-invalid, mirror `SetPanelOrigins`). Convert
`clientIP` to a method `h.clientIP(r)`; update the 4 call sites. Logic: if `trustedProxies` empty OR
RemoteAddr not trusted → return RemoteAddr host, ignore all forwarding headers; else parse XFF
right→left skipping trusted hops, return the first untrusted (fall back to `X-Real-IP` from a trusted
peer, else RemoteAddr). Add `envTrustedProxies="YAOG_TRUSTED_PROXIES"` (comma-separated; empty=trust
none; never `0.0.0.0/0`) read in `serveController` beside `SetPanelOrigins`. **No new code.**
