# plan-1 — Pre-WASM contract hardening

**Goal:** Make `localcompile.Compile` the single heal-inclusive compile entry and restore
`internal/model` to a pure leaf, so the contract is CORRECT before Phase 3 makes it the WASM entry
point. Closes a real correctness divergence and an unsigned-file custody surface along the way.

**Prerequisites:** plan-0 (the arch-test catches the `runtimecontract` eviction not re-leaking).

## Design (assessment recap)
- **Heal is silently missing on the airgap path.** `normalize.HealCollidingPins` is a caller-applied
  decorator: `internal/controller/compile.go` heals BEFORE compiling, but `internal/api/handler_airgap.go`
  and the `localcompile` façade do NOT — so a colliding-pin topology compiles DIFFERENTLY on the
  anonymous vs controller path. The 25-topology conformance corpus has no colliding-pin fixture to catch
  it.
- **`internal/model` (a pure zero-import leaf) houses a stateful contract** — `model.Condition` is
  consumed ONLY by the agent/controller/api (never the pure compiler/renderer), a reverse
  type-placement that dodges an import cycle. **NOTE (review):** `MimicOutcome*` + `MimicBreadcrumbPath`
  are DIFFERENT — the pure-core renderer bakes them into `install.sh` (`renderer/script.go:126-134`), so
  they are legitimately model-resident and MUST stay pure-core-reachable; only `Condition` moves.
- **`51820` (the WG listen-port base) is un-hoisted** — duplicated at `peers.go` (≈1129/1361) and
  `semantic.go` (≈272), the exact comment-synced drift hazard `allocconst` exists to kill.
- **`artifacts/export.go` triplicates the bundle file-set** — `BundleFiles` parses the config keys, the
  disk-write loop re-parses them, and `allFiles` is a third hand-built list feeding the manifest; a file
  written-but-unlisted ships UNSIGNED/UNCHECKSUMMED.

## Changes
- Fold `normalize.HealCollidingPins` INTO `localcompile.Compile` (the single façade), and remove the
  ad-hoc caller heals; add a **colliding-pin conformance fixture** proving the airgap and controller
  paths now compile identically. The TS mirror still ships this phase, so mirror the heal-fold in
  `frontend/src/lib/heal.conformance.test.ts` (the gated TS heal impl already exists).
- New `internal/runtimecontract` — move **ONLY `model.Condition`** there; imported by `controller`+`agent`,
  NEVER by the pure core. **Leave `MimicOutcome*` + `MimicBreadcrumbPath` in `model`** (the renderer
  consumes them — moving them would force a pure→stateful `renderer→runtimecontract` edge and break
  invariant 7). Add `runtimecontract` to plan-0's arch-test FORBIDDEN set so a future pure-core→runtimecontract
  edge is a red build.
- Hoist `51820` → `internal/allocconst.WGListenPortBase`; `peers.go` + `semantic.go` read the constant.
- Single-source `artifacts/export.go` into one `BundleFileSet` — the write-loop, the manifest, and the
  checksum set all DERIVE from it, so written == listed == signed == checksummed structurally.

## Verify + branch
Full Go suite (default + airgap, `-race` for controller) + gofmt; **regenerate the conformance/goldens to
a REVIEWED byte-diff** (the heal-fold intentionally changes airgap-path bytes for colliding-pin inputs —
that IS the fix; verify heal idempotency: the controller path already heals, so re-healing must be a
no-op). FE build + vitest + lint. Branch `refactor/plan-1-contract-hardening`.

## Tests produced
- Colliding-pin conformance fixture (Go + TS) — **perpetual** (corpus entry). Guards airgap==controller
  compile parity.
- `BundleFileSet` unit test — **perpetual** — guards written==listed==signed.

## Invariants at risk
- **[2] allocation stability / deployable configs:** the heal-fold moves bytes only for colliding-pin
  inputs; byte-verify against the golden corpus; NEVER blind `-update`.
- **[7] quarantine:** evict ONLY `model.Condition`; keeping `MimicOutcome*`/`MimicBreadcrumbPath` in
  `model` is REQUIRED (the renderer consumes them — a move would create the pure→stateful edge invariant 7
  forbids). `runtimecontract` must be in the arch-test forbidden set so no pure-core edge re-forms.
- **[5] custody:** the single `BundleFileSet` is the load-bearing fix — do not leave a second file list.

## Stop-loss
If heal is NOT idempotent on the already-healed controller path (re-healing changes bytes), the fold is
unsafe as written — STOP; the controller call site must stop pre-healing in the SAME change, proven by
the fixture.

## Out of scope
Splitting `peers.go`/`semantic.go` (deferred to plan-5); any WASM work (plan-3).
