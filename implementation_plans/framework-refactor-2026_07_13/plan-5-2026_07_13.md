# plan-5 — Delete the TS twin + shed deps + free the pure-core splits

**Goal:** Retire the ~10.6K-LOC hand-mirrored TS compiler and its lockstep apparatus AT THE ROOT, now
that WASM==Go is gate-enforced and the soaked default. This is where the single largest strategic
duplication is eliminated.

**Prerequisites:** plan-4 (WASM the soaked default), plan-3 (the permanent gate is the oracle).
**Irreversible surface deletion — sequenced strictly AFTER WASM is proven.**

## Design (assessment recap)
The TS mirror (`frontend/src/compiler/` ~8.8K + `internal/conformance/` ~2–3K + the drift manifest + ~233
hardcoded `.go:<n>` citations across `frontend/src/compiler/` (97 in `validator.ts` alone) + the
JS-escaped `script.ts` bash twin) exists ONLY to
keep the browser compiler byte-identical to Go. With WASM running the same Go code and a permanent gate
proving it, that entire apparatus is redundant. Two guards from the judges: re-home the gates that live
INSIDE `conformance/` before deleting it, and confirm the i18n catalog-sync test is NOT in `conformance/`
(it is at `internal/` root — Go-code↔catalog, independent of the Go/TS duplication; it survives).

## Changes
- **First**, re-home from `internal/conformance/` into `internal/localcompile` + `internal/compiler`
  BEFORE deleting it: (a) the keygen KAT + the coverage-floor tests; **(b) the FAIL corpus**
  (`testdata/fail` + `testdata/golden/fail`) and its `TestGoldenFail` / `TestGoldenFail_SpansBothChannels`
  two-channel (validator + `apierr`) coverage + the `BuildManifest`/`mustManifestBytes` harness — these
  are conformance-ONLY and would otherwise be silently lost (the WASM gate covers only the success/render
  corpus, and plan-5.5's "cross-package fixture" trigger will NOT fire for these conformance-internal fail
  tests, so enumerate them HERE, not as a contingency). Confirm the permanent WASM-vs-golden gate remains
  the sole fail-closed oracle for the RENDER path; the fail/verdict path stays a Go test in `localcompile`.
- **Delete** `frontend/src/compiler/` entirely (`validator.ts`, `peers.ts`, `renderers/*`, `escape.ts`,
  `keygen.ts`, `allocator.ts`, `cidr.ts`, all `*.conformance.test.ts`) and `internal/conformance/`.
- Drop `@noble/curves`, `@noble/hashes`, `jszip` from `frontend/package.json` (subsumed by `crypto/ecdh`
  + `archive/zip` inside WASM).
- **Now** split the pure-core mirror files (no parity tax remains): `internal/validator/semantic.go` (1201)
  → `semantic_pins/_ports/_edges/_naming.go`; `internal/compiler/peers.go` (1385) → `peers_prealloc/_build.go`
  + `orientation.go` (the forward/reverse 6-tuple swap, ≈`peers.go:720/802/892` + `compiler.go:206-256`)
  + `linkgroup.go` (the "enabled edges deduped by linkKey, skip clients" loop, ≈`semantic.go:327/927/1142`).
  The 233 `.go:<n>` citations die with the deleted TS files.
- Update the branch-protection required-check DISPLAY names in the SAME PR that removes the conformance
  job (repo rule).

## Verify + branch
Full Go suite (default + airgap, `-race`) + gofmt; FE build + vitest + lint; the permanent WASM-vs-golden
gate green (now the sole oracle); confirm `i18n_catalog_sync_test.go` still runs. `npm run build` proves
no TS import dangles. Branch `refactor/plan-5-delete-ts-twin` (deletion PR, then the pure-core split PRs).
