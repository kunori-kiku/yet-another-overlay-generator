# plan-10 — FE polish + drift lock-in

**Goal:** Finish the frontend axis work and convert the LAST ungated hand-mirrored contracts into red
builds — closing the silent-drift class that has bitten the team repeatedly.

**Prerequisites:** plan-2 (stores/services slicing started), plan-7 (error framework), plan-5 (FE deps
shed). Can run in parallel with plan-8 (disjoint surfaces).

## Design (assessment recap)
The same form-field markup + class string appears 51× across 10 files with no `Field` primitive. The
controller wire DTOs are defined 3× under 3 naming conventions (agent `...Wire`, api `...JSON`, controller
unsuffixed) with NO drift gate — they round-trip fine today (identical json tags) but nothing catches a
future field rename; the `omitempty` semantics are hand-re-implemented inside the store
(`isEmptyVal`/`dropOmitempty`), and the pin/omitempty field sets are hand-mirrored across `PIN_FIELDS` /
`ALLOCATION_PIN_FIELDS` / five `*_OMITEMPTY` lists (memory records repeated bites: "a new omitempty node
field MUST go in `NODE_OMITEMPTY`", the `EDGE_OMITEMPTY` `mimic_fallback` gap). Dead code accreted across
6+ areas reads as load-bearing.

## Changes
- New `frontend/src/ui/Field.tsx` — one primitive; land it across the 51 form sites.
- Finish the `stores/controller/*` slicing + `services/*` per-domain split begun in plan-2.
- **The drift gate** — a shared struct via codegen OR a `drift_manifest` entry + test covering the
  controller wire DTOs (`...Wire`/`...JSON`) AND the `omitempty` / `PIN_FIELDS` / `*_OMITEMPTY` field
  lists; a Go field rename or a missing omitempty entry becomes a RED BUILD, not a runtime failure. This
  EXTENDS (never weakens) the existing conformance/drift fail-closed guarantee.
- **Dead-code sweep** — `deriveAllowedIPs` (`peers.go`, zero callers), `hasCatalog`
  (`render/artifacts_json.go`, test-only), `apierr.WithStatus` (dead), `edgecase` exported
  `Compile`/`CompileAndRender` (test-only in production source), the dual-keyed allocation map aliases,
  the retained FE air-gap fetch branches.
