# plan-6 — Go-only renderer hygiene (ShellToken seam)

**Goal:** Now that the renderer has no TS twin to keep byte-identical, extract the install/deploy bash
into embedded templates and convert the root-shell-safety invariant from per-site CONVENTION into a
typed, machine-checked seam.

**Prerequisites:** plan-5 (the TS `script.ts` mirror is gone — the backtick ban + the byte-identical-mirror
tax are lifted).

## Design (assessment recap)
`internal/renderer/script.go` (1999) is 95% two near-identical verbatim bash blocks; the same SNAT rule
is encoded ~4× (≈`script.go:256/724/748/779`). Shell-safety is enforced by per-site convention: `shq` is
applied at only ~20 hand-picked sites and there is a dangerous raw/quoted twin field
(`NodeName` vs `NodeNameQuoted`, ≈`script.go:15-21`) where using the wrong one is command-substitution
injection ONE identifier away — a root-RCE-class seam with zero structural guard. `text/template` is
context-BLIND, so the honest guarantee is "this value passed through `shq`," not "the template escapes it."

## Changes
- `go:embed` the install / uninstall / deploy templates as `internal/renderer/templates/*.sh.tmpl`;
  reduce `script.go` + `deploy.go` to config-struct builders that render the embedded templates. Restore
  `shellcheck` on the generated root-install path (drop the backtick ban that blocked it).
- Introduce a **`ShellToken` newtype**: type every render field that reaches a root shell. Keep TWO
  constructors — `ShellQuoted` (shell-arg positions) and `ShellRaw` (validated heredoc/comment positions)
  — replacing the `NodeName`/`NodeNameQuoted` twin with a type, NOT a naive deletion.
- Extend `field_safety_test` to assert EVERY template-consumed string is a `ShellToken` (enumerate the
  template fields; a bare `string` reaching a template position fails the build).

## Verify + branch
Full Go suite; **regenerate the renderer goldens to a REVIEWED byte-diff** (watch the `{{ end -}}`
right-trim-eats-indent trap — shellcheck + goldens catch byte drift); `shellcheck` green on the emitted
scripts; `field_safety_test` green. Branch `refactor/plan-6-shelltoken-templates`.

## Tests produced
- `field_safety_test` (extended) — **perpetual** — guards the no-unescaped-user-text-to-root-shell seam
  structurally. Retirement: never.
- `shellcheck` CI on the generated scripts — **perpetual**.

## Invariants at risk
- **[3] no unescaped user text to a root shell:** STRENGTHENED but subtle — the guarantee is "value passed
  through `shq`," enforced by the type + `field_safety_test`; do NOT overclaim structural context-awareness
  `text/template` cannot provide. Every existing `shq` site must survive the template move intact.
- **[2] deployable configs:** goldens byte-verified; the emitted bash must stay `bash`/`wg-quick`-valid.

## Stop-loss
If embedding the templates changes ANY generated byte that is not a deliberate, reviewed improvement,
STOP — the extraction must be byte-preserving except where a fix is intended and golden-diffed.

## Out of scope
The error framework (plan-7); the Store core (plan-8); any renderer LOGIC change beyond the extraction +
typing.
