### 1 — SSRF SIIT `::ffff:0:0/96` (LOW, NEW) → FIXED

`embeddedIPv4` unwrapped 6to4 / NAT64 / IPv4-compat but missed the SIIT IPv4-translatable prefix
(`::ffff:0:169.254.169.254` → loopback/RFC1918/metadata all passed `isPublicUnicastIP` as "public").
Blast radius near-zero on the shipped posture (operator-auth-gated route; literal bracketed-IPv6 base
URLs already rejected by `validateReleaseURL`; a stock kernel does not route `::ffff:0:0/96` to internal
v4 without a configured SIIT translator). Still a real completeness gap relative to its sibling unwraps,
so **fixed**: a `::ffff:0:0/96` branch in `embeddedIPv4` + four `release_pins_test.go` table cases (SIIT
loopback/metadata/RFC1918 → refused; SIIT of public 8.8.8.8 → still allowed, no over-block).

### 2 — TS exporter zip-slip (LOW, NEW, N1 parity) → FIXED

The plan-4 TS exporter dropped Go's export-time `validateSafeName` guard: `export.ts` used `node.name` as
a JSZip directory unsanitized, and the name-charset validator allows `.`, so a node named `..` passed
validation. Bounded + operator-local (export hard-refuses in controller mode; the ZIP never reaches
controller storage), but it violated the **N1 invariant** (the browser exporter must not be
security-weaker than Go). **Fixed**: ported `validateSafeName` into `export.ts` (rejects ``/`.`/`..`/
path-separators/`..`-containing names), with `export.zipslip.test.ts` proving the charset-valid-but-unsafe
names (`..`, `.`, `a..b`) are rejected and safe names still export.

### 3 — S3 compile-DoS gap-fill quadratic (LOW) → ACCEPTED RESIDUAL + charter correction

The 4.1 charter's baseline row recorded S3 as "DONE plan-8 (oracle): per-pool cursor + ctx in peers.go
gap-fill." **That guard does not exist** — `internal/compiler/peers.go` takes no `ctx`, and
`gapFillTransitPair`/`gapFillLinkLocalPair`/`lowestFreePort` restart their scan from 0 per link (O(L²)).
A valid 9999-edge topology with a `/12` transit CIDR pins a core ~17s. The codebase's OWN DoS oracle
agrees (`internal/edgecase/dos_repro_test.go`: "S3 hardening is post-rc.1 roadmap (4.3:48)"; it asserts
only boundedness, not a cap). So the charter's disposition was wrong; this verdict corrects it (and the
charter's S3 row is amended to match).

**Why it is an accepted residual, not a blocker:**
- The anonymous `/api/compile` route is `//go:build airgap` — **route-absent (404) on the shipped
  controller** (verified, Task 6). The only unauth reach is the `-tags airgap` oracle / `cmd/compiler`: a
  single local user, self-inflicted, LOW.
- On the shipped controller, the compile path is **operator-authenticated** (`HandleCompilePreview`/
  `HandleStage`) and `enrolledSubgraph` filters edges to enrolled-node pairs — a trusted actor, bounded.
- S3 is on the **post-rc.1 hardening roadmap** by the program's own design; the cap (per-pool cursor +
  ctx threading through `peers.go`) is a non-trivial algorithmic change beyond this sweep's
  report-don't-rewrite scope. Recorded for the owner; not a tag blocker.
