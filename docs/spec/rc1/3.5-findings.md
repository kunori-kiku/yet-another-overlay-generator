- **canvas-touch** — the read-only canvas pans by drag below lg with NO node mutation.

## Selector contract (ARIA-first, no parallel testid taxonomy)

Specs bind to Subject 2's accessible affordances, never an invented testid vocabulary:
- hamburger → `getByRole('button', { name: 'Open navigation' })` (`shell.openNav`)
- nav / off-canvas overlays → `role="dialog"` (the shared `Drawer`)
- CanvasGate → its `canvasGate.title` text ("Designing needs a larger screen")
- fleet rows/cards → the node's `getByRole('link', { name: <nodeId> })` + the `<table>` element
- the no-overflow invariant → `document.documentElement.scrollWidth <= clientWidth` (helper `expectNoHorizontalPageOverflow`)

No `data-testid` was added (the ARIA surface sufficed for every smoke).
